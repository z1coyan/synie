# ADR：初始化向导与初始化分层

2026-07-18，拷问访谈定案。

- **初始化分层**：`seeds.exs` 收缩为环境层（存储接入点、编号规则等无需人决策的数据）；业务初始化（首个管理员、首个公司、首选语言）归初始化向导（/setup）；内置角色、CNY 由迁移种子。首个管理员创建移出 seeds，plan 004 的随机口令逻辑随之退役（向导里操作者当场自设口令，无「打印一次」问题）。否决双轨并存（两条建管路径口令来源不一、「首个管理员是谁」语义永久模糊），否决向导吞并 seeds（存储根目录等运维配置被迫进业务向导，CI/demo 失去一条命令初始化能力）。
- **显式完成旗标**：`sys_setting` 单行表（同 acc_setting/sal_setting 模式）记 `setup_completed_at`，空 = 未初始化。迁移对存量库回填（已有用户即已初始化，永不进向导）；旗标落库后 /setup 页面与 setup 接口永久关闭。否决「无用户即未初始化」（建完用户断线即卡死）与多条件派生态（正常运维删光公司会让向导幽灵复活）。
- **编排式执行**：新增后端面只有两块——「建首个用户」（未认证可调，门面内按「未初始化且无用户」门控，幂等，直接返回登录态）与「完成」（已认证，写首选语言+落旗，同事务）。公司创建、科目表初始化、货币列表全部复用既有已认证 mutation，第一步后向导自动登录、以超管身份走正常校验与审计。否决单事务一次提交（已有动作要在 setup 上下文平行复制一套、审计 actor 为空，与留痕到人的惯例相悖）。每步幂等可续，断线重进从第一个未完成步骤继续。
- **首个用户 = super_admin 旗标**：与现状行为一致，旗标无常态界面入口、永远留兜底救火账号（角色可停用可删，旗标不可经界面触碰）。术语表「超级管理员」相应修订为「仅初始化向导设置」。否决「首个用户仅持内置角色」（须给角色叠一堆保护规则，且旗标机制在新老环境分叉）。
- **内置 admin 角色迁移种子**：角色行 + 全域通配 `*` 授权由迁移统一插入（CNY 先例），新老环境一致；界面按设计不写通配授权，只在向导种子的老环境将永远补不出全权角色。后端强制内置角色不可改、不可删、授权只读；分配给用户照常。
- **首选语言挂用户**：`sys_user.preferred_language`，i18n 落地前纯占位。否决系统级默认语言字段（概念缓做，届时登录页可用浏览器协商兜底），否决 localStorage（不落库等于没做）。
- **向导预置约 20 种常用货币**：进入公司步按 iso_code 幂等补齐，新环境专属，老环境货币列表自维护不强塞；本币下拉选择默认 CNY。科目模板三选一、必选不可跳过（常态科目页对零科目公司本就有同一初始化入口，想手工建账的极端情况初始化后再删改）。
- **不留脚本化建管通道**：CI 不跑 seeds、无测试依赖预置 admin；无人值守场景控制台 `mix run -e` 建超管永远可达。等自动化部署需求真实出现再补，非不可逆决策。
