# ADR：打印模板全资源自动接入（字段清单内省派生）

2026-07-23，拷问访谈定案。本 ADR **修订** [2026-07-23-print-template.md](2026-07-23-print-template.md) 的字段清单部分；初版其余条款（上传即校验、版式主权在 Excel、一条引擎两个产物、批量不碰 PDF 合并、降级切割、模板全局主数据不留痕、权限与入口复用标准动作）全部继承，不再复述。

## 背景

初版每接入一个资源要手写两处：字段清单逐字段抄一遍 + 装配器逐资源写取值子句。系统约 103 个 Ash 资源，逐个手写不现实，新单据上线即被打印能力卡住。访谈结论：全部资源零代码接入，资源级定制一概不要，定制只发生在 xlsx 模板文件上。

## 决策

- **字段清单由 Ash 内省自动派生，权限目录全部资源零代码接入**：`FieldCatalog` 从手写清单改为派生模块（公开接口不变），`DocBuilder` 从逐资源子句改为通用装配器；资源码 → 模块的映射复用权限目录枚举（`permission_actions` 非空的资源）。拒绝一切资源级定制——不隐藏字段、不改字段名、不加打印专用派生字段；防止 103 份例外清单替代 103 份手写清单。
- **引擎内置三条统一语义规则（全部资源一刀切，不算定制）**：belongs_to/has_one 关联按 `关系.字段` 展开一层（目标侧仅公开标量属性，不再下钻）；枚举值统一经枚举类型的 `description` 渲染中文标签（本仓库枚举统一 `values: [atom: "中文"]` 写法，标签随类型自带、无独立映射表）；资源具 `party_type`/`party_id`（PartyType 枚举）时派生 `party.name`，按 Party 四类映射取主数据名称。
- **占位符一律路径式，旧拍平键不兼容、无迁移层**：`${company.name}` 取代 `${company_name}`，与循环区语法统一为「关系.字段」；存量模板须重新上传——派生清单不再含旧键，上传校验自动以「未知头字段」点名，迁移通道即校验本身（打印是瞬时动作，重传即恢复，不值得维护别名层）。
- **循环区按关系名寻址，同一模板允许多个顺序循环区**：每个 has_many 都是一个循环区（v1 的 `${items.*}` 写死只是特例），BOM 这类多子表资源（配料/工艺路线/副产品）一单打全；行复制、`_seq`、0 行删行、mergeCell 顺移语义逐区适用。**嵌套循环明确不支持**（循环区目标资源自身的 has_many，如报价条目内套价格档），行复制引擎结构上做不到行内套行，上传校验点名拒掉。
- **技术列不进清单**：主键、外键 `*_id`（含 party_id/voucher_id 等多态引用列）、`inserted_at`/`updated_at` 排除；公开无参计算字段与公开聚合进清单（如 `gross_total`，领域口径而非打印专用）。关系不区分 GraphQL `public?` 标记（打印为后端口径，BOM 子表等未公开 GraphQL 的 has_many 照样可循环）；属性/计算/聚合仍按 `public?` 过滤（防口令散列等敏感字段经路径展开外泄）。
- **字段标签=路径名本身**：不维护字段级中文标签（前端表单/列表的中文标签散落各页面文件、无可抓取的统一字典，逐字段抄一遍正是被否决的手工活）；资源名用权限目录现成中文标签，模板作者对着自描述的路径名写模板。
- **many_to_many 不进清单**（本期无打印诉求，留跟进）；布尔值显示 是/否。

## 明确排除（v2）

嵌套循环、多层关联下钻、字段级中文标签、旧拍平键兼容/迁移工具、资源级字段定制、图片类占位符（沿 v1）、批量打开各资源打印入口（入口仍逐资源走标准动作既有启用机制）。

## 后果

- 新增 Ash 资源（声明了权限码）自动获得模板上传/打印/导出能力，打印侧零代码；入口按钮按资源启用 `print`/`export`/`batch_print` 动作即用。
- 装载口径为「全量加载」（所有一层关联+循环区+计算聚合，不论模板是否用到），受批量上限 100 条约束；按模板占位符裁剪加载列为后续优化项。
- 词条「明细模板行」泛化为「循环区」，「模板占位符」「打印模板」词条同步改写（见 CONTEXT.md）。
