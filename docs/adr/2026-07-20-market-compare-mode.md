# 行情图对比模式：绝对价 / 涨跌幅归一化与跨口径同图

行情页图区新增**对比模式**切换：**绝对价**（默认，纵轴为价格，勾选仍限同币种同单位）与**涨跌幅**（纵轴为窗内涨跌幅 %，各序列以窗内首个有效点为 0% 基准，无量纲、可跨币种/单位同图，配 0% 虚线参考，tooltip 同时给出涨跌幅与原始价）。这推翻了此前「行情页合并」ADR 中被否的"指数归一化跨单位对比"——当时顾虑是解释成本高、工厂盯盘不必要；现用户（业务方）明确提出两点需求：绝对价悬殊的品种（沪铜 ~10 万 vs 沪铝 ~2 万）同图时低价品种波动被压平，以及希望沪银（元/千克）与铜铝（元/吨）同图。以显式模式切换承载（绝对价仍是默认），解释成本由口径标识「涨跌幅 % · 基准=窗内首点」承担，原同口径绝对价叙事不受影响。

切回绝对价时，跨口径勾选自动收敛到首个品种所在口径组并 toast 提示；模式与勾选、价类、时间窗一并本机持久化。

实现上的关键约束：后端时序查询 `price_series` 仍强制单组同口径（`same_scale?`），契约不改——前端在涨跌幅模式下按（币种, 计量单位）将勾选分组，并发请求后合并序列并保持勾选顺序；绝对价模式勾选必然同组，仍单请求。归一化在前端做（首点为基准折算 %），后端与领域模型零改动。

被否：改后端放开 `same_scale?` 校验（契约放大、校验语义变复杂，且分组请求缓存粒度不变）；双 Y 轴（仅解两品种、三品种以上失效，解释成本仍在）；归一化基准取窗内末点或固定日期对齐（首点基准与 KPI「窗内涨跌」同叙事，最易解释）；跨口径时放开 6 条上限（线图可读性边界与口径无关）。
