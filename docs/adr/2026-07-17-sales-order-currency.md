# 销售订单双币：货币挂头一单一币，金额链换算，统一双套落库

销售订单引入原币/本币双币口径：货币与汇率挂**订单头**（一单一币，行金额跟随），本币立为**公司必填属性**（`bas_company.base_currency_id`，迁移时库中无 CNY 则自动建「人民币/CNY/￥」并回填全部存量公司）；汇率在订单头**手工填写**（不建汇率主数据表，留将来财务模块），原币即本币时系统强制为 1，外币必填且大于零。换算采用**金额链**：原币金额=数量×原币单价（2 位）→本币金额=原币金额×汇率（2 位）；本币单价=原币单价×汇率（4 位）仅作展示参考，不参与金额计算——保证行内恒有 本币金额≡round(原币金额×汇率)，双币口径对账不撕。落库**统一双套**：本币单照常写满原币列+本币列（汇率 1、两套同值），报表与聚合一律读本币列，无 null 分支；「本币单不显复杂」只是展示层行为，且仅在单订单上下文（订单抽屉）生效——条目视图是跨订单混合行，固定全列展示（币种、本币单价、原币单价、本币金额、原币金额）。草稿期间改头上币种/汇率，全部行本币列在同一事务内重算；审核锁单即冻结币种与汇率。被否方案：行级各自货币（同单混币报价怪，头层原币聚合失义）；单价链（本币金额=数量×本币单价，大数量/多小数位时与 原币金额×汇率 差分厘，两币口径对不齐）；本币单只存一套、外币列置空（后续一切查询/聚合/下游单据永久背 coalesce 分叉）；全局本币或写死 CNY（多公司系统，将来外币记账主体容不下）；汇率主数据表（v1 范围膨胀，且订单场景手填够用）。
