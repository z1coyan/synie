# ADR:角色授权保存改为后端事务 sync

2026-07-21,拷问访谈定案(配套角色权限配置面板重构)。

- **保存由前端逐条 diff 改为后端单次事务 sync**:前端传目标权限码集合,后端在事务内对比存量增删。旧方案逐条并发 create/destroy,部分失败会留下半完成的授权状态,且无事务性;权限点规模增长后 diff 量也增大。
- **sync 只同步当前目录(permission catalog)内的具体码**:存量中的通配行(`资源:*`/`域.*`/`*`)与不在当前目录的陈旧码**原样保留**,不做全量替换。全量替换语义更直白,但会把界面不可写的通配行静默清掉,也会随权限点下线误删历史授权——fail-safe 方向与系统整体一致(新码默认不授权,旧码默认不收回)。
- **界面仍只写具体码、不写通配**:行/列/域三级全选一律展开为当前目录的具体码保存,资源日后新增动作不会自动授权(fail-closed)。通配写入继续仅限迁移种子,由内置角色守卫锁死写面(同 2026-07-18 通配 ADR 的既定边界,本次只是重申)。
- **被拒的备选**:前端继续逐条 diff(无事务,问题原样保留);sync 全量替换(危险,见上);引入可写通配授权(新动作自动继承是安全隐患,取消单动作须拆通配,心智负担大于收益)。
