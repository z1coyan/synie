# 费用报销：发票挂账、报销单核销的两步模型

费用报销是增值税发票第四类型（员工对手）的接入。决策：报销拆成两个时点的事实、两张单据——**费用报销发票审核即完成含税支出挂账**（借费用/税额、贷其他应付款-员工，记账终点，不生成任何下游单据）；**报销单是对员工付款的核销单据**（挂票行引用「已审核未报销」的发票 + 无票行手填非税支出，审核借其他应付款/费用科目、贷付款科目）。

理由：保住发票模块「审核即过账」不变量，发票侧零新机制；挂账与付款本是两个时点的事实，与应收应付体系（发货挂账 → 发票重分类 → 付款）同构；员工欠款经「其他应付款」角色进应收应付报表净应付，负债口径完整。

## Considered Options

- **报销单汇总多张发票、审核时联动审核名下发票**（报销单作唯一过账触发点）——双单据强耦合，报销单行混「引用发票」与「手工无票」两类行源，行模型拧巴；被否。
- **发票审核自动生成 1 票 1 单的「含税报销单」**——报销单退化为发票影子，无独立价值；被否。
- **发票各自审核 + 报销单仅校验已审并过无票行**——存在「发票审了、报销单没审」的中间态，合计口径悬空；被否。

## Consequences

- 科目角色扩为往来/费用两族：往来角色加「其他应付款」，费用角色五枚举（差旅费/办公费/业务招待费/交通费/其他费用）。
- 应收应付报表净应付公式 += 其他应付款；对手类型扩员工（`PartyType` 加 `employee` → `hr_employee`）。
- GL「角色科目分录必带对手」规则收窄为往来角色（费用角色科目分录不带对手）。
- 报销审批流、部分付款、报销冲抵员工借款（`hr_employee_loan`）留跟进。
