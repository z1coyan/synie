# ADR：销售对账单——两类型状态机与开票触发的应收重分类

2026-07-21，拷问访谈定案。

- **定位：业务枢纽单，自身不过往来重分类**：销售对账单（`sal_reconciliation`）是发货与开票之间的勾稽层。否决「纯业务台账不过账」（未开票应收只进不出，应收应付报表口径必烂）与「对账审核即重分类」（客户确认≠债权性质改变，且与订单/报价「审核不派生会计后果」风格冲突）——采纳**开票触发**：常规单的「未开票应收→应收账款」迁移由发票审核关联时发生。术语上与银行流水勾稽的「对账」切割：本域一律称**销售对账单**，银行侧术语正名为银行对账记录。
- **数量模型：行内拆分 + 受控投影，生效才消耗**：否决「整行勾选」（发 1000 对 600 是常态，把拆分压力推给发货录单不现实）。发货条目冗余 `reconciled_qty`（默认单位），机制照抄已发数量先例；剩余可对账 = 发货 base − 已对账。草稿不占量——常规单**确认时**、赠送/样品单**结单时**逐行校验剩余量并累加，撤回/作废回滚（否决草稿即占用：确认前多轮修改是常态，草稿不该成为稀缺资源）。存在非零已对账的发货条目，其发货单不可作废（同订单作废先例，下游勾稽锁上游）。
- **类型二分：手选锁死，约束不对称**：`reconciliation_type` = 常规 / 赠送样品，手选必填、保存锁死（换类型作废重开，同订单类型先例），共用编号与权限码。否决系统按勾选条目自动推型（常规单混赠品行推不出）。条目约束不对称：常规单**禁勾**样品订单来源与零金额行（永远开不了票的行不该进开票链路）；赠送/样品单**不限来源**——它兼任**超发尾差核销通道**（发 1020 客户只认 1000，余 20 以此单结清），这是它存在的核心理由，不是纯标签。
- **常规单：三态 + 发票一对一关联**：草稿→客户已确认→已结单。否决独立「已开出」态（确认前必多轮修改，草稿即承担）；确认可撤回（前提未关联发票）。发票（开出方向）草稿上选关联对账单，审核校验：对账单=客户已确认、公司/对手一致、**价税合计相等、一对一**。否决多对多+余额投影（拆票场景舍弃，换来「对账单即开票依据」的硬校验）。发票作废/红冲**自动**解除关联、对账单退回客户已确认——票没了结单事实不成立；已结单对账单无独立作废入口，纠错走发票侧。
- **关联发票过账：正常三行 + 冲回组（五笔）**：否决纯过桥分录（只借应收/贷未开票应收——销项税永远进不了 GL，发货贷方成糊涂账）。采纳：发票照常过 借应收账款（价税合计）/贷金额科目（未税）/贷税额科目（税额），**加冲回组** 借发货贷方口径科目/贷未开票应收科目（价税合计）。净效果：未开票应收清零、收入按未税、税额落账，无论发货贷方当初挂收入还是过渡科目都正确。两个科目槽放**对账单头**（建单按公司默认过账科目的发货借方/发货贷方代入，可改），发票模块不加字段。
- **赠送/样品单：两态，结单即过账**：草稿→已结单，审核过账 借费用/损失科目（单头手选）/贷未开票应收，冲掉该批条目在发货时挂的未开票应收；金额为零跳过分录（同发货先例）。已结单可作废（回滚分录组与已对账数量）——与常规单的不对称是故意的：它没有发票侧纠错通道。
- **金额算法：数量×单价金额链，行各带源订单汇率**：对账行金额=对账数量×快照原币含税单价（2 位）→本币=×源订单汇率（2 位），否决按比例摊（账单必须呈现数量×单价形态）。单头不设汇率（多来源订单汇率可不同）。分次部分对账的分位尾差不配平，由赠送/样品单核销兜底。
- **条目池与 IA**：同公司+同对手（客户/内部公司都纳入，内部公司同样走对账→对向发票链路）+单内同币种+剩余可对账>0，不做发货日期强约束。权限 `sales.reconciliation`（工作流动作：确认/撤回确认/作废），编号挂编号规则两类共用，菜单供应链销售组、对账条目/对账单双 tab。应收应付报表零改动——角色迁移有真实分录后口径自动正确。
