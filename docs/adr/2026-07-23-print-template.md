# ADR：打印模板与单据打印/导出管线

2026-07-23，拷问访谈定案。

- **Excel 模板填充 + LibreOffice 转 PDF，一条引擎两个产物**：打印=填充后 xlsx 经 LibreOffice headless（`soffice --convert-to pdf`）转 PDF 交浏览器；导出=同一份填充结果直接下载 xlsx。拒绝「打印走前端 HTML 版式、导出走 Excel」双版式体系——同一单据维护两套模板必漂移；拒绝前端 SheetJS 渲 HTML 打印——丢打印区域/页眉页脚等关键版式。
- **填充由 Elixir 自写（zip+XML 替换占位符、复制明细行），LibreOffice 只做哑转换器**：拒绝 LibreOffice UNO/宏填充（编程与进程管理复杂）与 Python sidecar openpyxl（给 Elixir 项目引入第二运行时）；团队已有 xlsx 解析经验（银行流水导入），占位符替换是确定性文本处理。
- **批量不碰 PDF 合并**：批量打印=N 条记录的模板块顺序铺进同一个 xlsx（块间分页符）一次转换出单 PDF；批量导出=单 xlsx 多 sheet。拒绝引入 qpdf/pdfcpu 等 PDF 合并依赖（Elixir 无成熟 PDF 库），且比转 N 次再合并更快。上限 100 条同步处理。
- **版式主权完全在 Excel 模板**：纸张/方向/页边距/打印区域由模板页面设置承载，系统不建字段、渲染不覆盖不自行添加页边距（转换必须严格尊重模板 page setup，列为验收点）；显示格式靠单元格 Excel 格式，引擎不发明格式化语法。拒绝系统层重复建模纸张/页边距字段（两个事实源必不一致）；用户忘设页面设置靠上传校验警告提示，不强制。
- **模板是全局主数据，打印不留痕**：`sys_print_template` 全局共享不分公司（公司抬头走占位符），多模板+单默认，可删无引用约束；打印/导出是瞬时动作，不落库不留痕不追溯。
- **上传即校验，错模板活不到打印时**：只收 .xlsx（写端不支持老 .xls）；提取全部 `${...}` 对照资源字段清单，未知字段拒绝保存并点名。空值填空、0 条目删明细行等运行时容错从宽。
- **降级切割**：导出 Excel 不依赖 LibreOffice，PDF 转换服务不可用时打印明确报错、导出照常；soffice 路径走配置；并发转换用独立临时 user profile 目录。Docker Linux 镜像内置 LibreOffice + Noto CJK 字体。
- **权限与入口复用既有机制**：标准动作 `print`/`export`/`batch_print` 直接启用（首期销售订单/销售发货单），不新增 `batch_export`；模板管理为 `sys.print_template` 资源；CSV 列表导出保留，与模板版式导出并存切割。
- **明确排除（v1）**：模板内图片内容（图纸/Logo）、.xls 写支持、异步大批量、系统层纸张/页边距字段、"下载 PDF" 按钮（浏览器自带）。Excel 公式在转换时的重算行为与行复制后公式引用调整为实现期验证项。
