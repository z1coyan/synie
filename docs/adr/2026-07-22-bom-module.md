# ADR：BOM 模块

2026-07-22，拷问访谈定案。

- **BOM 挂物料、单级嵌套，不建 BOM 树实体**：BOM 是物料的单层配方主数据（一物料一张、以物料为唯一键、无独立编号、跟随物料全局共享不分公司）；多级结构靠半成品物料（滤网筒、嵌件）各自挂 BOM 嵌套形成，树展开只是查询视图（检测环并截断）。拒绝真多级树：与「物料主数据为核心」架构一致，避免改一处联动整树。
- **工艺路线 = BOM 私有子表 + 可编辑工艺模板**：路线逐行添加（行=工序引用+工艺要求文本+外协布尔标记），只管「怎么做」，不带数量、不驱动执行、无库存/财务后果。工艺模板是独立主数据，建路线时复制带入，此后模板再改不影响已建 BOM——拒绝共享路线引用，与系统快照哲学（改默认不追溯、订单条目冻结快照）一致。
- **工序为全局主数据**：冲网/分切/焊接/CNC 等稳定词汇收口为主数据，路线行引用、被引用后不可删；拒绝自由文本（将来按工序统计无从收口）。
- **配料行 = 净用量 + 可空损耗率，平铺清单**：数量语义钉死为「每 1 默认单位母物料的净消耗」；不记投料工序（执行层需要时再补列）。
- **副产品行**：BOM 上声明联产出（铜屑/网屑+单位产出量），本期纯声明，作为将来执行层扣料/完工入库/成本分摊的锚点；拒绝工序级归属（本期是假精确）。
- **单份可改无版本**：换配方直接改，历史靠审计日志；拒绝版本管理——本期无下游引用，正确引入时点是「有单据引用 BOM 且需按历史配方还原」时。
- **明确排除**：生产工单执行层（扣料/报工/入库/成本归集）、外协加工单据流（铜屑返工暂走手工出入库+采购线）、条件分支路线，均不在本期范围。
