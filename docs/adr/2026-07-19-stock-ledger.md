# ADR：库存分录与出入库/调拨单据

2026-07-19，拷问访谈定案。

- **分录＝派生事实表，照 GL 先例**：库存分录（StockLedgerEntry）用户无直接写入口，全部由来源单据审核时派生；只追加不可改，作废＝标记 `is_cancelled` 不删行；库存余额＝分录聚合，报表只查分录不查单据。「分录＝唯一事实表，单据＝事实来源」一套语义与 GL 两处复用；将来采购入库、销售发货、盘点都写同一张表，主干不为它们改。
- **一行＝叶子仓×物料的一次数量变动**：一单据行派生一条，不合并（保留行级追溯）。数量用带符号单字段（入正出负、check 非零）——不学 GL 借贷双列，库存没有「方向即科目属性」语义，`sum(quantity)` 即余额。字段：公司、叶子仓、物料、数量、来源单据多态引用（`voucher_type/voucher_id/voucher_no`，同 GL poly-ref）、业务日期、`seq`、`is_cancelled`。不做批次/序列号/保质期/库位维度，连占位字段都不留（将来要加是纯加列，不推翻既有行）；客户料不设独立维度——客户专属是物料主数据的属性，库存按物料自然分开。
- **分录只认默认单位**：表上无单位字段，数量恒为物料默认单位口径、6 位小数；余额、校验、报表全在单一单位口径下跑。录入灵活性放单据行：单位限默认单位或转换单位（销侧先例），保存时折算并双份落库——录入单位＋录入数量（快照，展示对单用）与折算数量（系统算、6 位小数、不可手改，照订单金额链先例）。顺势堵洞：**物料存在库存分录后默认单位不可改**（与「存在转换行不可改」并列），否则改单位＝篡改全部历史分录语义。
- **v1 来源单据＝手动入库单＋手动出库单＋调拨单**：手动两张单无上游、**无对手字段**（「货从哪来/到哪去」摘要承担，加对手字段纯属空转；采购/销售线的专用单据自带对手，各管各的）。头＝公司（不可换）＋单头一仓＋业务日期（默认当天，可改即补录）＋摘要（带入分录 remarks）＋备注；行＝物料＋单位＋录入数量＋折算数量＋行备注，随行冻结物料/单位快照。状态机 草稿→已审核→（已作废）：仅草稿可改可删、无反审核、无关闭态（审核即履行完毕，无履行周期）。期初建账＝手动入库单，不开专用通道。范围取舍：调拨进 v1（在途是真实物理事实，种子仓已备）；盘点后置（要先有可用余额视图）；采购/销售接入留给各线。
- **调拨＝一单两动作走在途**：头挂调出/调入/在途三仓——在途仓默认带种子在途仓、可改、限本公司叶子仓，**不给在途仓发明保护标记**（守仓库 ADR「种子仓无特殊保护」）。状态机 草稿→已发货→已收货：发货审核写「调出仓负＋在途仓正」，收货确认写「在途仓负＋调入仓正」，分录如实反映在途停留。收货按行确认实收（默认＝已发，可改小不可改大），差额留在在途仓由手动出库单清理；一次收讫不分批；已发货不可作废（路上有货不能当单据不存在，纠错走反向调拨）；限同公司（跨公司调货是买卖，走销售/采购线）。
- **负库存＝仓级硬校验**：审核与作废同一口径——按（叶子仓×物料）「Σ 未作废分录 ＋ 本单变动 ≥ 0」，任一行不过则整单拒；`allow_negative` 逐仓显式跳过（占位字段就此生效，默认否＝最严方向）；草稿不校验；**只校验当前总余额，不做逐时点重放**（补录历史单的穿零问题留跟进，同 GL「报表只认截至日汇总」取舍）；作废会致负同样禁止作废（库存与 GL 的关键差异：作废入库单会减库存）。并发抢货由实现层兜底（审核事务内对（仓, 物料）键咨询锁），不入领域模型。
- **v1 数量账：不记金额、不派生 GL 分录**：没有成本来源——采购模块只有供应商主数据，手动单成本全靠手填，在手填成本上跑移动平均/FIFO 是全额复杂度买数字游戏；GL 派生缺对方科目——手动入库的贷方无业务语义，要么发明科目映射配置层，要么会计手选，都是半个存货核算模块。金额列将来纯加列（历史行空＝未估值），估值方法与存货记账等采购线带来真实成本时再定案；仓库关联科目继续空转，其「改动估值口径」跟进项届时才有意义。
- **余额＝纯查询聚合，不养投影表**：`sum group by (仓, 物料)` 带 `is_cancelled`/日期过滤直连分录表；`BillHolding` 投影是为段级复杂语义（区间折叠重建），库存是单纯聚合，v1 没有理由养一张投影（将来量大再上物化，口径届时已被报表验证）。两个只读视图：**库存余额表**（公司下仓×物料，支持截至日回溯、仓树/物料筛选、隐藏零行；在途仓作为普通仓自然呈现「在途库存」）与**库存分录流水**（只读列表，可跳来源单据）；物料视角＝余额表排序筛选，不发明第三界面；报表复用 read 权限码，不新设权限点。
- **不做红冲**：红冲解决的是「既成事实不能被简单抹掉」（会计更正轨迹、报表区间锁、下游引用），库存手动单一样都不占。纠错＝作废（过负库存校验）＋重开，审计日志＋作废标记＋新旧两单轨迹已完整；`is_reversed/is_reversal` 机制不复制到库存域。待采购/销售线专用库存单据带下游勾稽时再议。
- **仓库/物料约束收口**：存在库存分录（含已作废——作废分录仍是历史引用）的仓禁删、不能改非叶子；被分录引用的物料禁删（物料删除先例里「将来有库存流水后会加约束」即此）；仓停用升级为**拦新不拦旧**（新单据用仓保存时校验启用；已发货调拨的收货不校验，在途必须能收尾）；物料停用维持先例不拦截、选料器只过滤——仓与物料各按各的既定方针，不互相看齐。
- **留跟进汇总**：逐时点负库存重放校验；估值方法与 GL 存货记账（采购线落地时定案）；盘点单；调拨分批收货；物料/批次等扩展维度（用户明示后期要「库存单位」维度，届时加列）；改关联科目的估值口径；多级成环检测。

## 补充（2026-07-19，定稿后重构）

- **入/出库单合并为「手工出入库单」**（`inv_stock_doc` + `direction` 字段，创建后锁死）：两型 100% 对称（字段/状态机/校验一致，只差分录符号），合并消真重复；代价是编号系列合一（RK/CK 两前缀并为 CRK 单一系列），用户确认可接受。权限码随之并为 `inv.stock_doc`。
- **调拨单更名「手工调拨单」**：仅显示名（菜单/权限标签/流水来源链接），资源标识 `inv.stock_transfer` 不动——「手工」前缀与将来业务线驱动的库存单据区分。
