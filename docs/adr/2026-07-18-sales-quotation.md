# 销售报价单：价格清单无数量，行级定价模式+价格档，无汇率无双币

销售报价单立为**纯价格承诺清单**：条目只有物料/单位/单价没有数量——数量与供货承诺是销售订单的事,两单据以此划界。定价形态做成**行级 `pricing_mode` 枚举 + 独立档表**：`fixed` 固定价走行上含税单价;`qty_tiered` 数量梯度行单价强制空置(避免「按哪个价」歧义,DB check 兜底),价挂 `sal_quotation_tier` 价格档。梯度语义定为**起量价阶梯**:每档 =(起订量, 含税档价),订购量 ≥ 起订量即适用该档价,档间上界由下一档隐含,起订量同条目内唯一(即天然严格递增),低于首档起订量 = 无报价;档价不强制随量递减(涨价梯度合法)。币种挂头一单一币默认公司本币,但**不挂汇率、不做双币**——报价无数量即无金额,无金额链可换算,本币单价除展示无任何用途;将来报价转订单时汇率按转单时刻行情填订单头(报价至成单间汇率本就会变,报价时刻汇率无意义)。税口径与订单完全一致:单价与档价均为含税价,行带税率(默认 13%,可手改)仅作口径标注不参与计算。状态机为订单减配版:草稿→已审核→已作废,无「已关闭」(报价到期天然失效,无「履行中提前终止」概念),作废兼承担录错与提前撤回(过期后仍可作废);**已过期是派生展示态**(已审核 且 今天 > 截止日,截止当日仍有效),不落库不跑定时任务;报价延期不放开「已审核改截止日」的口子——延期是一次新的报价决策,复制重报留痕更干净。(物料, 单位) 报价单内唯一(唯一索引兜底);物料快照沿用订单条目机制(共享 `Sales.SnapshotMaterial`/`MaterialUnitAllowed`,无图纸挂接);审核门槛 = 行数 ≥ 1 且每个梯度条目至少一档;条目从梯度切回固定价时同事务清档(同「改头清行」先例)。**期货梯度报价**下期做、本期零预建:不建任何期货字段/空表,预留仅靠 `pricing_mode` 可扩值+档位数据在独立子表(新模式=加枚举值+新档表,头与条目结构不动);「期货梯度」的业务含义待与业主确认——候选解读 A:按交货期梯度(贸易口语「期货」=远期交货,档位键从数量换成交货期,与数量梯度同构);候选解读 B:挂钩期货盘面的基差/点价模式(报「合约+升贴水」,涉及合约主数据与点价动作,复杂一个量级),下期开工前必须先问清再动手。被否方案:区间报价(from-to 两头填,有缝隙/重叠校验负担且无业务增益);整单模式二选一(同单混报固定价与梯度是真实需求);统一档表建模、固定价=单档(基础报价 UI 与查询都被迫绕档表,退化情形反成负担);报价挂汇率落双币(无金额可换算,纯负担);过期落库+定时任务翻状态(日期比较即得,落库有时钟边界问题);放开已审核改截止日(破审核锁死原则)。
