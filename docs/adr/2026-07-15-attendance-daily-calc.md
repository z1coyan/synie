# ADR：考勤计算规则（日考勤）

2026-07-15，拷问访谈定案（设计先行，未实现）。

- **纯计时模型，无班次无迟到早退**：工厂计时工资，干多少算多少；日考勤只产出工时/加班/工日，不判迟到早退旷工，不建班次资源。
- **12 点切分四段模型**：按本地时区自然日，<12:00 为上午桶、≥12:00 为下午桶，各取最早卡=上班、最晚卡=下班。min/max 天然吸收考勤机连按，导入 ADR 留给计算层的连按收敛问题就地消解。切分点写死 12:00——午休跨 12 点会把去吃饭的卡认成下午上班，当前厂里午休不跨，变了再改常量。
- **不做跨零点处理**：晚班最迟 23 点前收工，自然日切分即可；将来出现整班倒夜班时此模型需重设计，不预留。
- **段工时 30 分钟向下取整，上下午各自取整**：与简道云现状一致；加班阈值在取整后判。
- **上午封顶 4h，下午超 4h 部分=加班工时**：标准半天 4h；早来不算工时也不算加班，正常工时日封顶 8h、正常工日封顶 1，加班只可能来自下午段。只上下午的人同规则（下午 6h=4 正常+2 加班），接受此推论。
- **奖励工日按日且封顶 0.5**：单日加班（取整后）≥3.5h 额外奖励 0.5 工日，再多也 0.5，不满为 0。它是奖励不是折算，与月末加班补贴（按加班小时，工资侧计价）并行。
- **缺卡计 0 + 异常标记**：桶内只有 1 张卡该段计 0 工时并标「缺卡」，不猜不兜底，逼出补卡动线；桶内 0 卡=该段无出勤（半天班常态），不是异常。
- **补卡单=一单一人一天多卡的虚拟卡**：原始打卡不可改（承接导入 ADR），修正只在计算层；补卡时刻与真实卡合并进桶取 min/max。保存即生效、删单即撤销，均自动重算当天；无审批流，靠权限码+审计留痕。
- **日考勤是派生持久行，人工不可直改**：(员工, 日期) 唯一，唯一修正入口=补卡单；导入执行/撤销、补卡增删自动重算受影响 (员工, 日) 集合，另留手动按区间重算兜底。全天无卡不生成行。
- **工日在月层核算**：日层存正常工时/加班工时/奖励工日；月工日 = Σ正常工时÷8 + Σ奖励工日，日工日仅作展示派生（÷8）不另设舍入。月汇总供工资，暂不锁定——工资模块落地时再补月结锁定。
- **周末/节假日同公式**：纯打卡推导，无工作日历、无加班审批，在厂即算；接受「多待就是加班」的磨洋工风险。
- **规则常量集中 Rules 模块，不立配置表**：12:00/30min/4h/8h/3.5h/0.5 六个常量多年稳定，为其立单行配置表不值；真要改再提升成配置。
- **权限前缀 hr.attendance_day 与 hr.attendance_correction**：承接导入 ADR 预留的独立前缀，统计、原始打卡、补卡可分别授权。
