# ADR：发货/入库公司默认过账科目，及草稿科目必填

2026-07-21，拷问访谈定案。

- **公司默认与全局设置分表**：发货/入库借贷默认按公司维护，落独立资源「公司默认过账科目」（一公司一行、四槽：发货借/贷、入库借/贷，均可空）。否决塞进 `sal_setting`（该表是全局单行标量，无公司维）与挂 `bas_company`（供应链过账配置不属于公司主数据）。`sal_setting` 继续只承载样品/零星上限与超发/超收比例。设置 UI 仍在「供应链 → 设置」销/采 Tab：先选公司再编本侧两槽，本 Tab 保存只 upsert 本侧、不覆盖对侧。权限仍 `sales.setting`。
- **默认可选、单据必填**：四槽均可空，无默认不挡建单——默认是便利预填，不是建单许可证。发货/入库头上借贷科目**草稿保存即必填**（后端 create/update 同卡）。科目表按公司隔离，**换公司时借贷两科整组用新公司默认覆盖**（无默认则清空），避免他司科目残留；打开已有草稿且未改公司则保留单上科目，不重灌。
- **零金额仍跳过总账，但科目不许空**：整单本币过账金额为零时继续跳过总账、不造名义分录（沿用销售发货 ADR）；与「科目可空」脱钩——科目必填是录入纪律，过账与否仍看金额。否决「零金额也强制过账」。
- **校验与单据同口径**：设置侧选单与保存校验与单据头一致——发货借强制未开票应收角色、入库贷强制未开票应付角色，另一侧本公司启用非汇总、角色不限（不收紧为收入/存货角色）。改默认不追溯已有单据。
- **表单布局**：借贷科目置于条目表下方并排；备注等业务头字段仍在表头。产品文档见 [销售发货](../产品文档/销售发货.md)、[采购入库](../产品文档/采购入库.md)。
