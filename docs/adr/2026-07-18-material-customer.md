# ADR：物料客户约束

2026-07-18，拷问访谈定案。

- **双字段 `is_customer_material` + `customer_id`**：显式布尔 + 可选客户 FK；互斥校验（是→客户必填，否→客户必空）。UI 开关与落库同构，不靠 `customer_id` 空非空反推（避免「空客户」被误读为未设）。
- **销侧硬约束，采购不校验**：报价/销售订单/销售发货只能挂通用料或本客户料；内部公司对手只能通用料。采购/库存可能先买后卖定制件，本期不挡。
- **有引用锁死客户字段**：报价条目、订单条目（含已作废）任一引用则不可改两字段；无引用可改。有物料引用的客户禁止删除。
- **`customer_part_no` 仅客户料可填**：改为通用料时清空；与客户 FK 语义对齐（对方料号属于该客户），通用料禁止残留文本。
- **v1 一料一客**：单 FK，不建客户×物料对照表；多客户需求出现再升级。
