# ADR：物料主档与单位转换

2026-07-16，拷问访谈定案。

- **全局共享，不挂公司**：承物料分类 ADR 同一逻辑（主数据集团统一）；库存/成本等公司维度留给将来的库存资源。
- **编号 seed 规则「分类编号-4 位序号」，per_company=false**：走通用 Numbering（计数范围=非序号段文本，每叶子分类天然各自计数）；留空自动、手填保留、换分类不改号（取号即固化）。
- **名称/规格不做组合唯一**：规格是自由文本，空格/全半角差异让硬唯一形同虚设，还挡「同名同规格不同客户料」的真实场景；防重靠录入前搜索。
- **客户方产品编号单字段**：当前业务定制件为主、一料一客户；一料多客户对照表（客户×物料×对方料号）需求出现时再升级，单字段可迁。**后续**：2026-07-18 起挂真·客户 FK（`is_customer_material`+`customer_id`），`customer_part_no` 仅客户料可填，见 [物料客户约束 ADR](./2026-07-18-material-customer.md)。
- **单位转换子表不限单位类型**：同类型换算全局 ratio 已有（`bas_unit`），但箱/包等包装单位换算只能物料级表达，禁同类型会堵死该场景；用（物料, 单位）唯一＋单位≠默认单位兜底，不与全局 ratio 交叉校验（录错靠人）。
- **有转换行禁止改默认单位**：系数全以默认单位为基准，改基准时静默保留是留毒数据、自动清空是静默销毁；报错让改动者先删转换行直面后果。
- **图纸=附件槽位 `drawing`，仅图片、可多张**：图纸多页/多视图/新旧版并存是常态；走附件槽位而非文件 FK，沿用员工证件照的裸文件下载授权链先例。
- **单位转换无独立权限点**：视为物料维护的一部分，读跟随 `inv.material:read`、写跟随 `inv.material:update`（照 UserRole 先例）。
- **分类反向硬约束**：下有物料的分类不能删除、不能改为非叶子（与「有子分类不能删/不能改叶子」同一味药）。
