# ADR：物料分类

2026-07-15，拷问访谈定案。

- **全局共享，不挂公司**：物料主数据集团统一才能跨公司采购/调拨对上号；科目挂公司是会计准则差异所迫，物料无此差异。
- **树形无限层级**：复用科目树全套先例（parent_id 自引用 + children_count + DataGrid tree 懒加载），限层反而要解释为什么。
- **is_leaf 显式存库且立刻硬约束**：分类编号要拼进物料编号，挂错层级代价高；派生方案区分不了「空的中间分类」和「叶子」。注意与科目 `is_group` 语义相反，勿混。
- **编号手填+全局唯一，不自动**：分类编号承载分类学语义（01=原材料），流水号无语义，人定。
- **编号/上级可改，物料号不追溯**：与自动编号模块语义一致（取号即固化）；锁死会导致录错只能留死数据。
- **新建 inv 库存域**：物料主档、库存、出入库将来同族同域；scm 是前端菜单聚合概念，采购/销售已各有域。
