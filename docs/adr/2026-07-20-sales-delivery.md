# ADR：销售发货单——履约出库一体写库存与未开票应收

2026-07-20，拷问访谈定案。

- **单据＝履约出库事实，不是纯物流单**：销售发货单（`sal_delivery`）承接销售订单的实物履行。审核在同一事务内：(1) 按行写负向库存分录；(2) 累加订单条目已发数量；(3) 本币过账金额大于零时写总账两行。否决「只记物流、库存/账另单」「只确认应收、库存后置」——与订单「审核不动库存与账、由下游履约」及应收应付 ADR「发货进未开票应收」对齐。
- **行必挂订单条目，可跨单合发**：每一行引用一条已审核未关闭/未作废的订单条目；物料必须与该条目一致。一张发货可挂多张订单，但全部行订单的公司、对手、原币必须一致（混币否决——过账虽用本币，档案展示与对账不能糊）。同一订单条目允许一单多行（分仓出库）。无订单自由行本期不做。
- **数量：部分发 + 可配置超发比例**：行录业务单位数量，折算默认单位 `base_qty`（库存与已发一律 base）。部分发货允许。超发挂 `sal_setting.delivery_overship_ratio`（默认 0%、范围 0～100%），审核时 `累计已发 ≤ 订购 base × (1+比例)`；草稿不硬卡、审核硬拦，与库存负库存校验同档。否决任意超发与固定写死比例。
- **已发数量＝订单条目冗余投影**：`shipped_qty` 落在订单条目上，审核加、作废减；事实来源仍是已审核发货行。否决纯 calculation 聚合——订单条目列表要筛/展剩余可发，热路径不宜每次扫发货行。
- **仓：行仓必填，头仓可空仅预填**：有效出库仓在行上；头仓可选，作新建行预填，不参与「行空则用头」的解析（与「头默认+行覆盖」的宽松形态相比更严——避免审到一半某行无仓）。仓限本公司启用叶子；负库存规则同库存域。
- **总账 v1：两科确认未开票收入/应收，不做存货成本**：库存分录 v1 无金额，发货**不**做借主营业务成本/贷存货。有金额时：借方科目**强制**角色=未开票应收，贷方科目自选（不强制收入角色）；金额=各行「订单条目快照本币含税金额 × (本行 base / 快照订购 base)」汇总配平；借方行带对手，贷方不带。整单金额为零（赠品/零单价）**跳过总账**、科目可空，仍出库存与已发——总账校验要求单边大于零，造 0.01 名义分录否决。发货不拆销项税（税在发票）。
- **日期双轨**：发货日期→库存分录业务/过账日；过账日期→总账（有金额审核必填，默认=发货日期）。头上仅备注（无独立摘要）：库存分录 remarks 带入备注，总账 remarks 留空（发票先例）。
- **状态机照库存业务单**：草稿→已审核→（已作废）；无反审核、无红冲、无关闭态。作废＝库存分录组作废+总账分录组作废+回滚已发（作废仍过负库存）。有下游开票勾稽后再议作废限制与红冲。
- **与订单关闭/作废交叉**：仅已审核订单可发；关闭后禁发、不改历史；有已审核未作废发货则订单不可作废；关闭后仍可作废发货纠错。草稿发货不挡关单，作废订单时若仍有草稿发货则拒绝并提示先删。
- **内部公司只记卖方**：对手可为内部公司，不自动在买方生成入库/应付（无对向发货）；集团无买卖调货走调拨单。
- **快照**：行保存冻物料口径（无图纸复制）与订单条目价税数量口径；过账用快照金额比例，不依赖事后 join。
