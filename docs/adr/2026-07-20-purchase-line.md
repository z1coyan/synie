# ADR：采购线——报价/订单/入库全镜像销售线，及三个差异定案

2026-07-20，拷问访谈定案。

- **全镜像销售线骨架**：采购报价（`pur_quotation`）→ 采购订单（`pur_order`）→ 采购入库（`pur_receipt`）的结构、状态机、双币金额链、Party 对手、条目快照（含图纸挂接）、编号规则、双 tab 列表 IA 全部照搬销售报价/订单/发货三单先例。否决另起形态——同构单据两套写法是纯成本，阅读与维护都按一套心智模型走。对手范围换为供应商/内部公司（内部公司同样不能是单据公司自身）。
- **订单分型＝常规＋零星**：常规订单逐行强制引用**有效采购报价条目**（口径同销售常规订单：报价已审核、订单日期在 [报价日期, 截止日] 内、公司/对手/币种一致；固定价带入、数量梯度按行数量套档、低于首档起订量视为无报价）；**零星订单**（spot）自由录价，单行数量 ≤ 供应链设置 `spot_item_max_qty`（默认 100，按行录入数量直接比、不做单位换算）。否决「单型全部强制报价」——零星紧急采购要求先录报价不贴业务；否决「不限量自由单」——无约束的自由单会绕开报价体系走大宗采购。供应链设置照先例在 `sal_setting` 加字段，不另建表。
- **采购入库＝履约入库一体写库存与未开票应付**：审核同一事务内 (1) 按行写正向库存分录；(2) 累加订单条目**已收数量**（`received_qty`，冗余投影，审核加、作废减）；(3) 本币过账金额大于零时写总账两行——**贷方强制「未开票应付」角色科目、借方会计自选**（存货/费用等），金额＝各行订单条目快照本币含税金额 ×（本行 base 占订购 base 比例）汇总配平，贷方行带对手、借方不带；整单金额为零跳过总账、科目可空。与销售发货逐点对称（借贷镜像）。行必挂已审核未关闭订单条目、可跨单（公司/对手/原币一致）、行仓必填（本公司启用叶子仓）、头仓可空仅预填、无自由行。不拆进项税（税在发票）。
- **入库超收比例**：`sal_setting.receipt_overreceive_ratio`（默认 0＝禁超收，范围 0～100%），入库审核时 累计已收 ≤ 订购 base × (1+比例)；草稿不硬卡、审核硬拦。否决「不限制超收」——默认禁超收与超发对称，供应商多送货走配置开口。
- **库存分录仍只记数量（重申推迟）**：采购入库带来真实价格，但库存 ADR「估值方法与存货记账等采购线带来真实成本时再定案」本期仍不重启——一旦分录记金额就必须同时回答出库计价（移动平均/FIFO）、销售发货成本分录、盘点差异金额，是半个存货核算模块；只记入不记出等于半成品。入库价款只走总账（借方科目会计手选），库存金额账、估值方法整体留跟进。
- **内部公司只记买方账**：采购入库不自动在卖方公司镜像出库/应收（无对向入库单），与销售发货「只记卖方账」对称；集团内买卖由双方各自开单。
- **交叉与作废**：关闭订单后不可再入库；有已审核未作废入库时订单不可作废（须先作废入库），草稿入库先删再作废订单；入库作废＝库存分录组作废（仍过负库存校验——入库作废减库存可能致负）＋总账分录组作废＋回滚已收。无反审核、无红冲，有下游发票勾稽后再议。
- **客户物料约束不延伸**：采购/库存侧维持术语表既定口径——选料不校验客户物料，任何物料均可采购。
- **IA**：菜单 供应链→交易 加「采购报价」、采购订单落既有占位项；供应链→库存 加「采购入库」与销售发货并列。权限码 `purchase.quotation` / `purchase.order` / `purchase.receipt`。三单均挂编号规则（留空自动取号可手填）。产品文档见 [采购报价](../产品文档/采购报价.md)、[采购订单](../产品文档/采购订单.md)、[采购入库](../产品文档/采购入库.md)。
- **行情挂钩定价模式**：销售报价侧该模式尚未实现（数据模型已预留），采购报价同步预留不同步实现；两侧将来一起落地。
