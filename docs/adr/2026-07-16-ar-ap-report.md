# ADR：科目角色与应收应付报表

2026-07-16，拷问访谈定案。

- **报表纯 GL 口径**：应收应付报表只聚合 `acc_gl_entry`（截至日余额，按对手×角色轧差），不从业务单据推导。账面即报表；当下没有发货单模块，未开票应收靠手工凭证过账产生，将来发货单接 GL 后自动进报表。
- **科目角色以科目字段承载**：`bas_account.role` 可空枚举，否决独立映射表（新资源/权限/页面一整套）与公司默认科目字段（加角色要改 schema）。代价是 acc 域概念侵入 base 域主数据，接受。
- **六角色**：未开票应收/应收账款/预收账款/未开票应付/应付账款/预付账款。含预收预付才能算净敞口；其他应收/应付对手维度与客商报表对不齐，缓做。
- **只挂叶子科目**：分录只能记叶子，圈叶子即完备，免递归展开子树；多明细科目挨个挂，实际往来科目就几个。
- **角色科目过账强制对手**：`GL.validate` 新规——行科目带角色必须填对手（增量生效）；报表对存量无对手行出「未指定对手」兜底行，列合计始终对平科目余额。
- **不区分币种**：现状分录单金额无折算，全 CNY；外币科目禁挂角色兜住口径，汇率机制届时一起设计。
- **两 tab 不做矩阵**：应收/应付同页分 tab（承兑 IA 先例），纯客户行的应付列全 0 视觉冗余，否决单表六列。
- **下钻跳分录页**：报表数字点击跳 `finance/entries` 带预置筛选（对手+角色科目集+截至日），不在报表页内嵌明细。
