# 销售订单分型：常规单强制有效报价来源，样品单自由录价但限量

销售订单立**订单类型**（`order_type`：regular/sample，必填、保存后锁死不可改——类型决定条目的整套校验规则，允许改类型等于绕过校验，换类型只能作废重开）。**常规订单**条目逐行强制引用**有效报价条目**（`quotation_item_id` 非空，可追溯成交价出处，报价事后作废/延期不联动订单）；有效判定 = 报价单已审核 + 订单日期 ∈ [报价日期, 截止日]（用订单日期而非今天，补录历史单不被「已过期」挡）+ 公司/对手（客户与内部公司一视同仁）/币种均一致（报价无汇率，跨币种折算等于让报价承诺了它没承诺过的价）。单价与单位随报价锁定：固定价直接带入；数量梯度按行数量自动套档、低于首档起订量视为无报价保存报错；改价必须重报价，下单环节不议价，仅税率带入后仍可手改（税率本只是口径标注）。**样品订单**反向：任选物料手填单价、必无报价引用，单行数量 ≤ 销售设置上限（按行录入数量直接比，不做单位换算——防「按箱绕过」的换算严谨不值那点复杂度）。新建 `sal_setting` 单行配置表存 `sample_item_max_qty`（默认 100），仿 `acc_setting` 先例：种子行恒在、只读改、配置一律加字段不另建表。前端条目子表在订单类型/客户/公司/日期选齐前 disabled，改此四项或币种即清空子表（币种入列因报价匹配含币种；汇率不在列，它只影响本币列重算）。后端两道闸兜底：草稿头改客户/公司/日期/币种且存量条目 ≥ 1 直接报错（保「每行恒与头匹配」不变量）；审核逐行复核（常规：引用有效性+单价与报价一致；样品：数量 ≤ 当前配置上限），审核是承诺时点作最后防线。被否方案：整单绑定一张报价单（报价延期重报后新旧两单无法共存于一张订单，逐行跨单挑才贴合业务）；单价带入作默认但可议价（破「常规单只认报价」的纪律，等于没改）；配置改小时回头批量校验存量草稿（动作太大，审核闸已够）；编号按类型分流（编号规则引擎加一段类型字段段即天然分流，无需代码）；留「存量行免报价引用」迁移后门（无线上数据，`order_type` 默认 regular 一刀切，老开发数据直接清）。连带：状态机、权限码、双币金额链、条目快照机制全部不动；样品单行为即现状逻辑原样保留。
