# ADR：其他库存单信息架构

2026-07-19，拷问访谈定案。

- **纯 IA，不动领域模型**：手工出入库单 / 手工调拨单 / 库存盘点单三资源、权限码、状态机、编号规则、审核/发货/收货动线全部不变；合并仅限菜单入口与页面壳。三单状态机差异大（调拨两段式且已发不可作废），强行统一会开大坑。
- **菜单名「其他库存单」**：相对将来采购入库/销售出库等业务线单据的剩余类分组壳；单据正式名/权限标签/审计日志仍用「手工出入库单」「手工调拨单」「库存盘点单」。「其他」是前瞻命名——业务线单落地前名称略悬空，接受换菜单成本低于以后再改对仗结构。
- **三 tab 子路由**（承兑汇票/销售订单先例）：`/scm/other-stock` + `docs` / `transfers` / `counts`；tab 短标签出入库/调拨/盘点；旧三路径重定向保书签。无对应 `read` 的 tab 隐藏；默认 tab 按出入库→调拨→盘点回落。
- **公司改列仅三张手工单**：列表 `companyId` 首列可筛（对齐会计凭证/总账分录），去掉顶部全局公司选择器；建单时公司为表单头必填（createOnly），仓候选绑表单公司。默认值：列筛唯一公司 → 唯一授权公司 → 空。库存余额/分录本期仍顶部选公司（余额 API 按公司入参、分录另开对齐轮次）。
- **不做**：第四种单据资源、壳级权限码、统一编号/状态机；余额与分录页公司交互本期不动。
