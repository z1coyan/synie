# ADR：采购对账单——两类型状态机与收票触发的应付重分类

2026-07-21，拷问访谈定案。全镜像 [销售对账单 ADR](./2026-07-21-sales-reconciliation.md)，差异点见末节。

- **定位：业务枢纽单，自身不过往来重分类**：采购对账单（`pur_reconciliation`）是入库与收票之间的勾稽层。否决「纯业务台账不过账」（入库挂的未开票应付只进不出，应收应付报表口径必烂）与「对账审核即重分类」（供应商确认≠债务性质改变，且与订单/报价「审核不派生会计后果」风格冲突）——采纳**收票触发**：常规单的「未开票应付→应付账款」迁移由开入发票审核关联时发生。
- **数量模型：行内拆分 + 受控投影，生效才消耗**：否决「整行勾选」（入 1000 对 600 是常态）。入库条目冗余 `reconciled_qty`（默认单位），机制照抄已收数量/已对账数量先例；剩余可对账 = 入库 base − 已对账。草稿不占量——常规单**供应商确认时**、赠送/样品单**结单时**逐行校验剩余量并累加，撤回/作废回滚。存在非零已对账的入库条目，其入库单不可作废（同订单作废先例，下游勾稽锁上游）。
- **类型二分：手选锁死，约束不对称**：`reconciliation_type` = 常规 / 赠送样品，手选必填、保存锁死（换类型作废重开，同订单类型先例），共用编号与权限码。否决系统按勾选条目自动推型。条目约束不对称：常规单**禁勾零金额行**（永远开不了票的行不该进开票链路）；赠送/样品单**不限来源**——它兼任**超收尾差核销通道**（入 1020 供应商只开 1000 的票，余 20 以此单结清），这是它存在的核心理由，不是纯标签。
- **常规单：三态 + 发票一对一关联**：草稿→供应商已确认→已结单。否决独立「已收票」态（确认前必多轮修改，草稿即承担）；确认可撤回（前提未关联发票）。发票（开入方向）**必须**关联对账单，审核校验：对账单=供应商已确认、公司/对手一致、**价税合计相等、一对一**。否决多对多+余额投影（拆票场景舍弃，换来「对账单即收票依据」的硬校验）。发票作废/红冲**自动**解除关联、对账单退回供应商已确认——票没了结单事实不成立；已结单对账单无独立作废入口，纠错走发票侧。
- **关联发票过账：正常三行 + 冲回组（五笔）**：否决纯过桥分录（只贷应付/借未开票应付——进项税永远进不了 GL，入库借方成糊涂账）。采纳：发票照常过 借金额科目（未税）/借税额科目（税额）/贷应付账款（价税合计，带对手），**加冲回组** 借对账单借方科目（未开票应付，带对手）/贷对账单贷方科目（入库借方口径，不带对手），金额=价税合计。净效果：未开票应付清零、入库借方口径冲回、税额落账，无论入库借方当初挂存货还是费用科目都正确。两个科目槽放**对账单头**（建单按公司默认过账科目的入库贷方/入库借方代入，可改；借方强制「未开票应付」角色），发票模块不加字段。
- **赠送/样品单：两态，结单即过账**：草稿→已结单，审核过账 借对账单借方科目（未开票应付，带对手）/贷对账单贷方科目（收益类，手选，本公司启用非汇总），金额=勾选条目价税合计（本币），为零跳过分录（科目仍必填，同入库先例）。已结单可作废（回滚分录组与已对账数量）——与常规单的不对称是故意的：它没有发票侧纠错通道。
- **金额算法：数量×单价金额链，行各带源订单汇率**：对账行金额=对账数量×入库条目快照原币含税单价（2 位）→本币=×源订单汇率（2 位），否决按比例摊（账单必须呈现数量×单价形态）。单头不设汇率（多来源订单汇率可不同）。分次部分对账的分位尾差不配平，由赠送/样品单核销兜底。
- **条目池与 IA**：同公司+同对手（供应商/内部公司都纳入，内部公司同样走对账→对向发票链路）+已审核未作废入库+剩余可对账>0+单内同币种（快照订单原币），不做入库日期强约束。权限 `purchase.reconciliation`（工作流动作：供应商确认/撤回确认/结单/作废），编号挂编号规则两类共用，菜单供应链交易组「采购对账」、对账条目/对账单双 tab。应收应付报表零改动——未开票应付→应付账款的角色迁移有了真实分录，口径自动正确。
- **与销售侧的差异点**（全镜像之外的三处有意偏离）：
  1. **对手范围**：供应商/内部公司（销售侧为客户/内部公司）。
  2. **常规单条目约束**：只禁零金额行，无「样品订单来源」禁用——采购订单无样品类型（常规/零星两型，见 [采购线 ADR](./2026-07-20-purchase-line.md)），零价赠送行走赠送/样品单。
  3. **借贷方向镜像**：入库审核为 借入库借方(不带对手)/贷未开票应付(带对手)，故对账单头借方强制「未开票应付」角色（默认代入入库贷方）、贷方=入库借方口径（赠送/样品单为收益类，默认代入入库借方）；发票冲回组随之反向（借未开票应付/贷入库借方口径）。
