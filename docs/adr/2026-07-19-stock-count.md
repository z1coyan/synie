# ADR：库存盘点单

2026-07-19，拷问访谈定案。

- **盘点单＝校正账实的来源单据，全照手工出入库单先例**：单头一仓（公司创建后不可换＋启用叶子仓＋业务日期＋摘要＋备注），行＝要盘的物料清单（物料＋账面数量＋实盘数量＋行备注，行冻结物料/单位快照）；允许部分盘点——未列入的物料不受影响，盘点单只对其列出的行负责。状态机 草稿→已审核→（已作废），仅草稿可改可删、无反审核、无关闭态；编号规则、权限码 `inv.stock_count`、行单位折算双份落库全部照搬。库存分录表零改动——盘点只是第三种来源单据（`voucher_type='inv.stock_count'`），库存分录 ADR「盘点都写同一张表」就此兑现。
- **账面数量＝显式快照，不冻结、不审核时重取**：行存取数时刻的余额快照（创建/整仓带出时落行，可整单刷新，刷新不动已填实盘数）；差异＝实盘−快照，审核按快照派生分录——差异非零行派生一条带符号分录（盘盈正、盘亏负），零行不落（分录 check 非零正好）。取舍：冻结出入库救不了补录历史单（业务日期在过去，与锁仓无关）却买一整层锁仓复杂度；审核时实时取数则口径不可见、不可打印、差异随世界变化无法解释；快照把时点管理变成一个显式按钮。
- **兜底校验替代冻结**：快照后该仓分录有新增或作废（按分录创建/作废时间判定，业务日期补录照样命中）→ 审核拒绝，须刷新重取。推论：审核时点余额恒等于快照、调整后恒等于实盘（≥0），差异分录永不致负——负库存校验只在作废时有意义（撤销盘盈会减库存，照先例拦截）。同仓并发盘点不设硬约束：校验天然串行化审核，先到者落差异、后到者刷新即收敛。
- **未填行整单拒**：审核要求至少一行、逐行实盘数已填且 ≥ 0，空行报错（补数或删行）；不做静默跳过——「我以为盘了」比「没盘」糟，盘点单内容须等于用户明确意图。整仓带出口径＝账面非零行（同余额视图默认隐藏零行），账面零的物料手工加行（实物有账面无的盘盈场景）。
- **作业纪律入文档不入系统**：取快照前确认该仓单据已录齐；盘点前收尾相关在途调拨（物理到货未收货＝假盘盈）；取快照到审核之间该仓尽量不动货。在途仓不发明禁令（系统无标记可判，守「在途仓无特殊保护」先例）。
- **留跟进**：盘点审核后补录历史单造成的再次账实不符（只能再盘一次或手工出入库调整，属补录固有限制，同「逐时点穿零」一类）。
