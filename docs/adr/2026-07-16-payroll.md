# ADR：员工薪资（工资单/发放/借款）

2026-07-16 定案（后台自主设计，重大取舍已逐条列出，供复核）。

- **工资公式**：应发 = 月工日 × 日薪 + 补贴 + 奖金 − 罚款 − 借款抵扣。基本工资 = 工日×日薪，四舍五入到分；应发持久落库（可筛可排可聚合），改单即重算。
- **工资单 = (员工, 月) 唯一持久快照**：生成动作按考勤月汇总批量建单，工日/加班工时/缺卡天数快照自月汇总、日薪/补贴快照自员工档案；无考勤不建单。已存在的 (员工, 月) 跳过不覆盖——重复生成不会冲掉手工调整。
- **快照可改 + 重取兜底，仅限待发放**：待发放态所有金额字段（含工日/日薪）可手改（月中调薪、考勤机故障手调），`refresh` 动作按当前考勤/员工档案重取快照；已发放态全字段锁死，差错走补发，不回头改单。
- **两态状态机，由发放记录驱动**：待发放 → 已发放。首笔发放记录自动翻转为已发放；发放记录全部删除自动翻回待发放。不设「部分发放」——实发合计（聚合列）已表达进度，状态只回答「发过没有」。
- **发放记录多笔（一员工一月多条）**：类型自动判别不让用户选——翻转那笔=发放，已发放后追加=补发。金额允许为负（多发冲回）、禁止为零。发放记录不可改，只可删了重录（审计口径干净）；删除受工资单联动约束（见状态机）。
- **发放动线一键化**：主动线是行动作「发放」与表格多选「批量发放」，直发未发差额——`pay_remaining` 金额不收前端，事务内锁单按 应发 − 已发 权威计算，差额 ≤ 0 拒绝（前端差额可能过期，以锁内为准防重复发放），复用 create 码。抽屉表单保留兜底（自定义金额/日期/备注/负数冲回）。
- **月发放总和两级可见**：单行级=工资单上实发合计 sum 聚合列；月度级=`month_stats` 汇总（应发/实发/工资单数/未发放数），复用 read 码不新设权限点（照考勤 `month_summary` 先例）。
- **借款台账 = 员工级流水，不做单笔核销**：借款/归还两类流水行，余额 = Σ借款 − Σ归还（员工级）；不建借款↔工资单核销明细表。员工现金还款=手工归还行。余额汇总 `balances` 出 generic action，复用 read 码。
- **借款抵扣与台账自动联动**：工资单翻转已发放时自动生成一条归还行（挂 `payroll_id` 反链、日期取发放日），翻回待发放时自动删除；翻转时校验抵扣额 ≤ 该员工当时余额，不足则整笔发放失败。自动行（`payroll_id` 非空）禁手改手删，只随工资单联动。竞态窗口（校验与并发手工归还）以工资单行锁串行化发放侧，台账侧极窄窗口在厂规模下接受。
- **不设单据编号**：(员工, 月) 是天然业务标识，工资单/发放记录均不接 AutoNumber，避免为编号而编号。
- **全局不挂公司**：照 hr 域先例（员工/考勤全局）。工资从哪个公司走账属 GL 侧问题，待员工挂公司/接凭证时一并演进。
- **加班补贴暂并入奖金手填**：考勤 ADR 预留的「月末加班补贴（按加班小时，工资侧计价）」不单列计价字段——计价规则未定案，先由奖金字段人工承载；工资单快照展示加班工时供参考。规则定案后再立字段与公式项。
- **月结锁定这次仍不落**：考勤 ADR 预留「工资模块落地时再补月结锁定」，拷问后降级——快照模型天然免疫后续考勤重算，锁定只防「生成后考勤又变而未刷新」，该风险由 refresh 动线 + 补发动线覆盖；真出事故再立锁。
- **不接 GL（跟进项）**：工资费用凭证、代发银行流水匹配留待后续。
- **权限前缀三枚**：`hr.payroll`（create/read/update/delete；generate 复用 create、refresh 复用 update、month_stats 复用 read）、`hr.payroll_payment`（create/read/delete，无 update——不可改先例）、`hr.employee_loan`（create/read/update/delete；balances 复用 read）。
- **三资源全挂审计**：generate 逐行创建留痕（厂规模员工数百，逐行可接受，不走绕过审计的 atomic 批量）。
