# ADR：考勤打卡导入

2026-07-15，拷问访谈定案。

- **两段式但暂存行不落库**：月导约 1.5 万行（60-70 人 × 日 6-7 次冗余打卡），银行式暂存行会每行写两遍，且行编辑能力对打卡无意义；预览的真实价值只是摘要（未匹配编号清单/重复数/坏行数），存批次头即可，执行时重新解析直写。
- **attendance_no 加全局唯一约束**：导入按编号匹配员工，不唯一则匹配语义模糊；多台考勤机同编号视为同一人（编号即全公司考勤号）。
- **打卡记录全局不挂公司**：跟随员工（hr 域全局主数据先例）；将来员工挂公司时一起补。
- **(员工, 时刻) 唯一 + 静默跳过**：.dat 区间重叠重导是常态、员工连按考勤机是常态；唯一键天然幂等，跳过计数上报。隔几秒的连按保留原样，收敛交给将来的考勤计算。
- **自动建员工须兼有 hr.employee:create**：自动建是真实的员工创建，fail-closed，不借导入码扩权；`[未知]` 员工身份落实后直接改名，不做合并动线。
- **只取编号+时间，其余列不存**：状态键（上/下班）出名不可靠（员工不按），机号/验证方式无业务用途；原始 .dat 挂批次留痕，将来要列可重导。
- **打卡不可改、撤销只整批**：打卡是客观事实，单条改删等于伪造原始记录；导错文件靠 destroy 批次级联删纠正（审计日志留痕），不设 canceled 状态壳。补卡/异常单是考勤计算模块的正道，不在原始层伪造。
- **权限前缀 hr.attendance_punch 而非 hr.attendance**：给将来日考勤/月统计留独立前缀，避免「只授统计不授原始打卡」时拆码迁移。
