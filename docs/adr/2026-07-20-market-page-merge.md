# 行情页合并：单工作台 + 同口径多序列图

将「行情品种」「行情价点」收成基础数据下唯一入口**行情页**（`/base/market`）：上图（多选对比 + KPI）+ 下表（价点 / 品种 Tab）。领域模型不变（品种主数据 + 不可变价点）；权限资源码仍分 `base.market_instrument` / `base.market_price`，页面按能力拼装，不另建 `base.market`。图区只叠**同一币种且同一计量单位**的启用品种（默认勾最大同口径组，上限 6），避免沪银（元/千克）与铜铝（元/吨）硬叠；整图统一价类（默认结算价，禁止跨价类冒充）、统一时间窗（默认近 30 日）。图与 KPI 用专用时序只读查询（多品种 × 价类 × 区间，仅未作废点），与 DataGrid 列表职责分离；图上 chips 的品种投影挂价点读权限，使「能看盘不必能改主数据」。图勾选与表弱联动；图状态本机持久化。系统未上线，旧双路由直接替换、不做兼容 redirect。

被否：双 Y 轴/指数归一化跨单位对比（解释成本高、工厂盯盘不必要）；图强绑表格筛选（绑架补录动线）；合并单一权限码（粒度变粗、与后续「能取价不能改品种」冲突）；图数据复用分页 list（易漏点、与维护列表耦合）；固定只画最新价或只画结算价（盘中与复盘互斥）。
