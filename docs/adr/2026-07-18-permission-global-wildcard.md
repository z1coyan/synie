# ADR：权限通配加全域级 `*`

2026-07-18，拷问访谈定案（初始化向导的配套决策，见同日 ADR「初始化向导与初始化分层」）。

- **通配梯级加一档**：权限匹配候选链由「自身 → `资源:*` → `域.*`」扩为「自身 → `资源:*` → `域.*` → `*`（全域）。改动限 `Permission.candidates/1` 追加一级；`Registry.granted_codes` 是过滤式展开，全域授权自然展开为全部码，无需另改。
- **动因是内置 admin 角色的「永真全权」语义**：`域.*` 覆盖不了将来的新域（每加一个域就要记得补授权，必然漂移）；逐码全量快照随新权限点上线 fail-closed，admin 角色名存实亡。全域一条 `*` 零维护，新域新码自动覆盖。
- **`*` 仅内部种子可写**：界面本来就自律不写通配授权（角色矩阵只逐码授予），全域行只由迁移种子进 `sys_role_permission`，配合内置角色守卫（授权只读）无法经界面增删；授权行入库、审计可见，无隐藏旁路。
- **风险自知**：通配匹配是安全核心路径，误配 `*` 的爆炸半径大于 `域.*`。缓解：写面被迁移+守卫锁死（业务代码无第二条写入路径），匹配语义与既有梯级同构（候选链追加而非新机制），permission/registry/authz 三层测试覆盖。
