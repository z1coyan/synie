# ADR：仓库模块

2026-07-18，拷问访谈定案。

- **按公司，不全局共享**：仓库要关联科目，科目按公司；库存是公司的资产事实，公司授权的数据权限天然适用。物料全局共享的理由（跨公司采购/调拨对上号）不适用于仓——两公司各有一个「成品仓」互不相干。
- **无仓库号，名称公司内唯一**：定案时砍掉了编号字段——仓库数量少、编号不承载分类学语义（不像分类编号要拼进物料编号），纯负担。名称成为唯一人类标识，故公司内唯一（物料「名称不唯一靠录入前搜索」的先例不适用于这么少的档案）；种子仓命名带公司编号前缀（`XX - 默认仓库`），引导全限定名风格，跨公司选择器可辨。
- **树语义整体照搬物料分类**：`parent_id` 自引用 + `is_leaf` 硬约束（非叶子纯归集，将来只有叶子持有库存）+ 多根并列 + 前端懒加载，一套语义两处复用，不发明第二棵树的规则。「所有仓库」种子根不特殊、非强制总根。多级成环检测留跟进（同分类）。
- **种子三仓、无保护、显式入口**：新建公司（初始化向导与公司管理页两个入口，前端编排）自动建 `{公司编号} - 所有仓库`（非叶根）/`- 默认仓库`/`- 在途`。走 `AccountInit` 同款显式幂等动作而非数据库级钩子；种子仓不做 is_default/is_system 等保护标记——「默认」语义等库存单据落地时再钉，现在钉容易钉错。
- **关联科目可空，限本公司非汇总本币**：前哨战阶段强制挂科目等于逼用户先配科目；但约束（本公司 + 非汇总 + 币种为空，与 `AccountRoleGuard` 同思路）现在就要严，错数据以后是迁移 pain。多仓可共用一科目（同角色先例）；可改不追溯，库存落地后改科目的估值口径留跟进。
- **外协仓/负库存纯占位入库**：`is_outsourced`、`allow_negative` 只落字段不关联逻辑，避免二次迁移；负库存默认否——将来校验落地时默认最严方向，要负库存逐仓显式打开。
- **存量不补偿，reset 重走 setup**：开发阶段无生产数据，不为种子写存量迁移；直接重置整库重走初始化向导验证全链路。
- **留跟进汇总**：多级成环检测；有库存后的叶子切换/删除硬约束；停用拦截业务引用；改关联科目的估值口径；外协仓与负库存的实际逻辑。
