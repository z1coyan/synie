# 术语表

- **物料分类**（MaterialCategory，`inv_material_category`）：物料的树形分类学，全局共享不分公司。
- **物料**（Material，`inv_material`）：全局共享的物料主数据（不分公司），只能挂叶子分类；名称必填不唯一，防重靠录入前搜索。
- **物料编号**（`inv_material.code`）：物料唯一标识，留空按「分类编号-4 位序号」自动取号（每叶子分类各自计数），手填原样保留；改挂分类不追溯改号。
- **默认单位**（`default_unit_id`）：物料计量与单位转换的基准单位，必填、类型不限；存在转换行时不可修改。
- **单位转换**（MaterialUnit，`inv_material_unit`）：物料级换算事实，一行＝「1 默认单位 = x 该单位」；（物料, 单位）唯一，单位不限类型但不能是默认单位，与全局单位 ratio 互不交叉校验。
- **图纸**：物料的图片附件槽位（`drawing`，仅图片、可多张）；其他文件走默认槽位。
- **叶子分类**（`is_leaf`）：可挂物料的末级分类；叶子不能再有子分类。与科目的 `is_group` 语义相反。
- **分类编号**（`code`）：人工定义、全局唯一的分类学编码（如 01=原材料），将来作为物料编号前缀；可改，已生成的物料编号不追溯。
- **打卡记录**（AttendancePunch，`hr_attendance_punch`）：考勤机原始打卡事实（员工+时刻），导入是唯一入口，不可改；(员工, 时刻) 唯一。
- **考勤导入批次**（AttendanceImport，`hr_attendance_import`）：一次 .dat 导入的留痕单据，只存解析摘要不存暂存行；destroy 即撤销（级联删其打卡）。
- **考勤机编号**（`attendance_no`）：员工在考勤机上的编号，非空时全局唯一；.dat 行凭它匹配员工。
- **[未知] 员工**：导入勾选自动创建的占位员工（name=[未知]，attendance_no 回填），身份落实后直接改名，无合并动线。
- **日考勤**（AttendanceDay，`hr_attendance_day`）：按 (员工, 自然日) 派生的考勤结果行（四段时刻+工时+加班+奖励工日），随打卡/补卡自动重算，人工不可直改。
- **补卡单**（AttendanceCorrection，`hr_attendance_correction`）：计算层的虚拟打卡单据（一单一人一天多卡），重算时与真实打卡合并取值；原始打卡永不修改。
- **缺卡**：某半天桶内只有一张卡，该段计 0 工时并标异常；桶内无卡是「无出勤」不是缺卡。
- **加班工时**：下午段（取整后）超 4 小时的部分；上午封顶 4 小时，不产生加班。
- **奖励工日**：单日加班 ≥3.5 小时额外奖励 0.5 工日（日封顶 0.5），与月末加班补贴并行，不是小时折算。
- **工日**：8 正常工时=1 工日；月工日 = Σ正常工时÷8 + Σ奖励工日，在月层核算。
- **工资单**（Payroll，`hr_payroll`）：(员工, 月) 唯一的薪资快照单据；应发 = 工日×日薪+补贴+奖金−罚款−借款抵扣。待发放可改可删，已发放锁死。
- **发放记录**（PayrollPayment，`hr_payroll_payment`）：一次实际发钱的留痕，一张工资单可多条；首笔翻转工资单为已发放，后续为补发；金额可负（冲回）。
- **补发**：工资单已发放后追加的发放记录（考勤漏算等差额），不回头改单。
- **借款台账**（EmployeeLoan，`hr_employee_loan`）：员工借款/归还流水，余额=Σ借款−Σ归还（员工级，不做单笔核销）；工资单发放时按借款抵扣自动生成归还行。
- **科目角色**（AccountRole，`bas_account.role`）：标记科目在应收应付口径下的用途，可空；只能挂叶子（非汇总）科目，外币科目不可挂；同公司多科目可挂同一角色，报表自动合并。
- **六角色**：未开票应收 / 应收账款 / 预收账款 / 未开票应付 / 应付账款 / 预付账款；枚举写死在代码里，新增角色即加枚举值。
- **应收应付报表**：纯 GL 时点余额视图——截至日前（含）未作废分录按对手×角色轧差；没过账就不存在，不区分币种。
- **净应收/净应付**：净应收 = 未开票应收 + 应收账款 − 预收账款（净应付对称）；各角色按自然方向轧差，反向余额显示负数不搬列。
- **未指定对手**：挂角色科目上无对手的存量分录在报表中的兜底行；新过账已强制角色科目分录必须填对手。
