# 术语表

- **BOM**（物料清单）：挂在物料上的单层配方主数据，跟随物料全局共享不分公司；一物料至多一张、以物料为唯一键、无独立编号，单份可改可删（历史靠审计日志，版本管理待有下游引用时再议）；配料为子物料行，多级结构靠半成品物料各自挂 BOM 嵌套形成，系统不建 BOM 树实体，树展开只是查询视图；配料子物料不能是母物料自身（保存校验），多级间接成环不硬校验、树展开检测环并截断提示。菜单「生产」组（BOM/工序/工艺模板），权限码 `mfg.bom` / `mfg.operation` / `mfg.route_template`。
- **工艺路线**（生产工艺单）：挂在 BOM 上的工序子表（每 BOM 私有，逐行添加，行=工序引用+工艺要求文本+外协标记），描述「怎么做」（工序顺序与要求），不带生产数量、不驱动执行、不派生库存/财务后果；生产工单执行层不在本期范围。
- **工序**（Operation）：全局共享的工序主数据（如冲网/分切/焊接成筒/CNC），不分公司，编号挂编号规则；工艺路线行引用工序，行内工艺参数/要求走自由文本；被路线引用后不可删除。
- **工艺模板**：独立可维护的工艺路线模板主数据；建 BOM 路线时选模板**复制带入**为 BOM 私行，此后模板再改不影响已建 BOM（快照语义，同系统改默认不追溯惯例）。
- **配料行**：BOM 的子物料行——子物料+单位（限默认单位或其转换单位）+单位净用量（每 1 默认单位母物料的净消耗）+可空损耗率（理论耗用=净用量×(1+损耗率)，空即无损耗）；平铺清单，不记录投料工序（投料时点待执行层需要时再补列）。
- **副产品行**：BOM 上声明的联产出行——物料（如铜屑/网屑，须为可入库存的物料）+单位产出量（每 1 默认单位母物料）；本期仅作声明，是将来执行层扣料/完工入库/成本分摊的锚点。
- **外协标记**：工艺路线行上的布尔标记，表示该工序通常外发加工；不指定供应商（发哪家是执行时决定），不产生任何单据；外协加工单据流（发料/入库/加工费）不在本期范围。
- **物料分类**（MaterialCategory，`inv_material_category`）：物料的树形分类学，全局共享不分公司。
- **物料**（Material，`inv_material`）：全局共享的物料主数据（不分公司），只能挂叶子分类；名称必填不唯一，防重靠录入前搜索；被库存分录引用（含已作废分录）后不可删除。
- **客户物料**（`is_customer_material`）：物料是否专属某一客户；为真时必挂一个客户（v1 一料一客），为假时客户为空（通用料）。销侧单据（报价/销售订单/销售发货）只能使用通用料或本客户料；内部公司对手只能用通用料。采购/库存本期不校验。
- **物料编号**（`inv_material.code`）：物料唯一标识，仅按编号规则自动取号（初始化完成时预置：分类编号 + 可选客户编号 + `-` + 不补零序号；字段空则省略，故通用料与客户料共用一条规则、按前缀分桶计数；例：分类 `F(P)` 下通用料 `F(P)-1`、客户 77 料 `F(P)77-1`），不接受手填、创建后不可修改；改挂分类/客户不追溯改号。
- **默认单位**（`default_unit_id`）：物料计量与单位转换的基准单位，必填、类型不限；存在转换行时不可修改；存在库存分录后亦不可修改（分录只存数量，改单位等于篡改历史）。
- **单位转换**（MaterialUnit，`inv_material_unit`）：物料级换算事实，一行＝「1 默认单位 = x 该单位」；（物料, 单位）唯一，单位不限类型但不能是默认单位，与全局单位 ratio 互不交叉校验。
- **图纸**：物料的图片附件槽位（`drawing`，仅图片、可多张）；其他文件走默认槽位。
- **叶子分类**（`is_leaf`）：可挂物料的末级分类；叶子不能再有子分类。与科目的 `is_group` 语义相反。
- **分类编号**（`code`）：人工定义、全局唯一的分类学编码，作为物料编号前缀；初始化向导完成时种子两级：大类单字母（`F` 产品 / `P` 包材 / `E` 设备工量具 / `M` 劳保耗材 / `S` 服务，非叶子），叶子为 `大类(子类)` 形态（如 `F(P)` 客户产品成品、`P(W)` 木箱；已有分类则跳过）；可改，已生成的物料编号不追溯。
- **仓库**（Warehouse，`inv_warehouse`）：按公司的树形仓库主数据；无仓库号（不接编号规则），名称必填且公司内唯一；新建公司（含初始化向导首个公司）时自动种子三仓——`{公司编号} - 所有仓库`（非叶子根）、`{公司编号} - 默认仓库`（叶子）、`{公司编号} - 在途`（叶子），种子仓无特殊保护，可改可停可删（受常规约束）。
- **叶子仓库**（`is_leaf`）：树语义照搬物料分类先例——非叶子仓只做层级归集、不持有库存，只有叶子仓能发生库存；上级只能选非叶子仓、可空（多根并列，「所有仓库」非强制总根）、不能选自身（多级成环留跟进）；新仓默认叶子、默认启用；有下级的不能改叶子/不能删；存在库存分录（含已作废）的仓不可删除、不能改为非叶子（分录只落叶子仓，改则历史与树语义矛盾）。
- **仓库关联科目**（`account_id`）：仓库指向本公司存货科目的可空引用，为将来出入库记账找科目用；限本公司、非汇总（叶子）、本币（币种为空）科目；多仓可共用一科目；随时可改不追溯（库存金额账落地后改科目的估值口径问题留跟进）；被引用的科目不可删除；种子三仓一律留空。库存 v1 只记数量，此字段继续空转。
- **仓库启停**（`active`）：默认启用；停用**拦新不拦旧**——新单据用仓（手工出入库单头仓、调拨三仓）保存时校验必须启用，已发货调拨单的收货不再校验（在途必须能收尾）；不级联下级；列表行操作切换，不进编辑表单。
- **外协仓**（`is_outsourced`）：仓库占位标记（默认否），语义为货物存放在协作方处的仓；本期纯占位——不校验、不联动、无报表口径，仅列表可筛。
- **允许负数库存**（`allow_negative`）：仓库标记，默认否（最严方向）；库存分录落地后生效——审核与作废按（叶子仓×物料）校验「Σ 未作废分录 + 本单变动 ≥ 0」，任一行不过则整单拒（草稿不校验）；打开此标记的仓跳过校验；仓级口径、不跨仓相抵；只校验当前总余额，补录历史单的逐时点穿零留跟进；作废会致负同样禁止作废。
- **其他库存单**：库存菜单下的 IA 分组入口（非领域实体、无独立资源/权限码），承载无业务上游的库存来源单据——页内三 tab：出入库 / 调拨 / 盘点，分别对应手工出入库单、手工调拨单、库存盘点单；列表公司为首列可筛（无顶部全局公司），建单时公司在表单头选择。相对将来采购入库/销售出库等业务线单据而言的「其他」。
- **手工出入库单**（StockDoc，`inv_stock_doc`）：仓管无上游单据直接录入的库存来源单据，入库/出库一体——`direction` 字段区分、创建后锁死，编号单一系列；头=公司（创建后不可换）+单头一仓+业务日期（默认当天，可改即补录）+摘要（带入库存分录 remarks）+备注（内部长文本）；行=物料+单位（限默认单位或转换单位）+录入数量+折算数量（系统折算默认单位、6 位小数、不可手改）+行备注，随行冻结物料/单位快照；状态机 草稿→已审核→（已作废），仅草稿可改可删、无反审核、无关闭态（审核即履行完毕）；审核派生分录（入库为正、出库为负），作废过负库存校验；期初建账走入库方向，不开专用通道。菜单入口在「其他库存单 → 出入库」。
- **库存余额**：分录的只读聚合视图——按（仓×物料）`Σ 未作废分录`，支持截至日回溯（业务日期 ≤ 某日）；纯查询聚合直连分录表，无投影表。呈现为两个只读界面：**库存余额表**（公司下仓×物料，仓树/物料筛选、隐藏零行，在途仓自然呈现）与**库存分录流水**（按仓/物料/日期/来源单据筛选，可跳来源单据）；物料视角=余额表排序筛选，不单设界面；报表复用分录 read 权限码。
- **手工调拨单**（StockTransfer，`inv_stock_transfer`）：同公司两仓间移货的单据，头=调出仓+调入仓+在途仓（默认带种子在途仓，可改，限本公司叶子仓）；状态机 草稿→已发货→已收货——发货审核写「调出仓负+在途仓正」分录，收货确认写「在途仓负+调入仓正」；收货按行确认实收（默认=已发，可改小不可改大），差额留在在途仓由手工出入库单（出库）清理；一次收讫不分批；已发货不可作废（纠错走反向调拨）；限同公司，跨公司调货是买卖，走销售/采购线。菜单入口在「其他库存单 → 调拨」。
- **库存盘点单**（StockCount，`inv_stock_count`）：核对账实并校正库存的来源单据；单头一仓（公司创建后不可换+启用叶子仓+业务日期默认当天可改+摘要带入分录+备注，同手工出入库单先例），行=本次要盘的物料清单（物料+账面数量+实盘数量+行备注，行保存冻结物料/单位快照），允许部分盘点——未列入的物料不受影响，盘点单只对其列出的行负责。账面数量=取数时刻的库存余额快照（创建/整仓带出时落行，可整单刷新，刷新只更新账面数量列、已填实盘数保留）；整仓带出口径=账面非零行（同余额视图默认），账面零的物料手工加行（实物有账面无的盘盈场景）。实盘数限默认单位或转换单位录入、系统折算双份落库（同手工出入库单行先例），必填且≥0。差异=实盘−账面，审核时差异非零的行派生一条带符号库存分录（盘盈正、盘亏负），零行不落；审核要求至少一行、逐行实盘数已填，有空行整单拒（补数或删行），不做静默跳过。审核兜底校验：快照后该仓分录有新增或作废（按分录创建/作废时间判定，补录的历史单照样命中）则拒绝审核，须刷新重取；不做冻结，取快照到审核之间该仓尽量不动货、取快照前确认单据已录齐（作业纪律）。单头仓限本公司启用叶子仓（在途仓不另设禁令，作业上盘点前先收尾相关在途调拨，否则物理已到货未收货会盘出假盘盈）；不做同仓并发盘点硬约束——兜底校验天然串行化审核，后到者刷新即收敛。状态机 草稿→已审核→（已作废），仅草稿可改可删、无反审核、无关闭态；差异分录永不致负（校验通过时审核时点余额恒等于快照、调整后恒等于实盘≥0），作废照常过负库存校验。编号挂编号规则（留空自动取号可手填），权限码 `inv.stock_count`。菜单入口在「其他库存单 → 盘点」。
- **库存分录**（StockLedgerEntry）：库存领域唯一事实表，只追加不可改；由来源单据审核时派生，用户无直接写入口（同 GL 分录先例）；库存余额=分录聚合，报表只查分录不查单据；作废=标记不删行；**不做红冲**——更正=作废（过负库存校验）+重开，待采购/销售线单据有下游勾稽再议红冲。一行=一次「叶子仓×物料」的数量变动（一单据行派生一条，不合并），数量带符号（入正出负、非零），恒为物料默认单位口径（表上无单位字段，6 位小数）；字段含公司、叶子仓、物料、数量、来源单据多态引用（voucher_type/voucher_id/voucher_no，同 GL poly-ref）、业务日期、seq、is_cancelled；不做批次/序列号/保质期/库位维度，客户料不设独立维度（物料主数据已区分）。数量账来源含手工出入库/调拨/盘点、**销售发货**（发货审核写负向分录）与**采购入库**（入库审核写正向分录）；v1 分录本身只记数量不记金额——销售发货的应收、采购入库的应付过账走总账分录，不在库存分录写金额；采购线已落地但估值方法仍未定案（见采购线 ADR 重申推迟：记金额须连同出库计价/发货成本/盘点差异一起设计），届时纯加列，历史行金额空=未估值）。
- **打卡记录**（AttendancePunch，`hr_attendance_punch`）：考勤机原始打卡事实（员工+时刻），导入是唯一入口，不可改；(员工, 时刻) 唯一。
- **考勤导入批次**（AttendanceImport，`hr_attendance_import`）：一次 .dat 导入的留痕单据，只存解析摘要不存暂存行；destroy 即撤销（级联删其打卡）。
- **考勤机编号**（`attendance_no`）：员工在考勤机上的编号，非空时全局唯一；.dat 行凭它匹配员工。
- **[未知] 员工**：导入勾选自动创建的占位员工（name=[未知]，attendance_no 回填），身份落实后直接改名，无合并动线。
- **日考勤**（AttendanceDay，`hr_attendance_day`）：按 (员工, 自然日) 派生的考勤结果行（四段时刻+工时+加班+奖励工日），随打卡/补卡自动重算，人工不可直改。
- **补卡单**（AttendanceCorrection，`hr_attendance_correction`）：计算层的虚拟打卡单据（一单一人一天多卡），重算时与真实打卡合并取值；原始打卡永不修改。
- **缺卡**：某半天桶内只有一张卡，该段计 0 工时并标异常；桶内无卡是「无出勤」不是缺卡。
- **加班工时**：下午段（取整后）超 4 小时的部分；上午封顶 4 小时，不产生加班。
- **奖励工日**：单日加班 ≥3.5 小时额外奖励 0.5 工日（日封顶 0.5），与月末加班补贴并行，不是小时折算。
- **工日**：8 正常工时=1 工日；月工日 = Σ正常工时÷8 + Σ奖励工日，在月层核算。
- **工资单**（Payroll，`hr_payroll`）：(员工, 月) 唯一的薪资快照单据；应发 = 工日×日薪+补贴+奖金−罚款−借款抵扣。待发放可改可删，已发放锁死。
- **发放记录**（PayrollPayment，`hr_payroll_payment`）：一次实际发钱的留痕，一张工资单可多条；首笔翻转工资单为已发放，后续为补发；金额可负（冲回）。
- **补发**：工资单已发放后追加的发放记录（考勤漏算等差额），不回头改单。
- **借款台账**（EmployeeLoan，`hr_employee_loan`）：员工借款/归还流水，余额=Σ借款−Σ归还（员工级，不做单笔核销）；工资单发放时按借款抵扣自动生成归还行。
- **科目角色**（AccountRole，`bas_account.role`）：标记科目用途的可空标记，分两族——**往来角色**圈定应收应付报表口径，**费用角色**（即报销类型）供报销类单据带出费用科目；只能挂叶子（非汇总）科目，外币科目不可挂；同公司多科目可挂同一角色，报表自动合并。
- **往来角色**：应收应付报表口径的七个角色——未开票应收 / 应收账款 / 预收账款 / 未开票应付 / 应付账款 / 预付账款 / 其他应付款；枚举写死在代码里，新增角色即加枚举值。挂往来角色科目的分录必须填对手（红字行豁免）。
- **应收应付报表**：纯 GL 时点余额视图——截至日前（含）未作废分录按对手×往来角色轧差；没过账就不存在，不区分币种。
- **净应收/净应付**：净应收 = 未开票应收 + 应收账款 − 预收账款；净应付 = 未开票应付 + 应付账款 + 其他应付款 − 预付账款；各角色按自然方向轧差，反向余额显示负数不搬列。
- **未指定对手**：挂往来角色科目上无对手的存量分录在报表中的兜底行；新过账已强制往来角色科目分录必须填对手。
- **对手**（Party）：单据上交易对手的多态引用（类型+ID，无真外键），取值供应商/客户/内部公司/员工四类（员工 = `hr_employee` 主数据）；类型与对手同填同空，保存时校验存在。挂往来角色科目的分录必须填对手。
- **基准单位**：某单位类型下换算比例恒为 1 的单位，每类型至多一个；同类型其余单位按换算比例折算到它。
- **换算比例**（ratio）：单位折算到同类型基准单位的比例，必须大于 0（如 kg 为基准时 g=0.001）。
- **总账分录**（GLEntry，`acc_gl_entry`）：全系统唯一财务事实表，只追加不可改；由来源单据审核时派生，报表只查分录不查单据。作废=标记 `is_cancelled`，不删行。
- **红冲**（红字冲销）：对已入账分录组的更正方式——原分录逐条金额取负生成红字分录组对冲，原组标记「已被红冲」；不删不改，名下分录合计归零。
- **汇总科目**（`is_group`）：科目树中只做层级归集、不可入账的节点；与物料「叶子分类」语义相反。
- **银行账户**（BankAccount，`acc_bank_account`）：公司自有银行资金账户主数据；别名/账号同公司唯一，可绑定一个会计科目；退出方式为停用而非删除。流水上的对方户名/账号只是文本，对手方不是主数据。
- **银行流水**（BankTransaction，`acc_bank_transaction`）：银行对账单的电子档案行，数据以银行为准（余额取银行快照不推算）；收/支恰一项大于零；不参与记账，不直接派生分录。
- **流水导入模板**（BankImportTemplate，`acc_bank_import_template`）：某银行账户导出文件的列布局解析配置（列号+格式+起始行）；一个账户可配多份。
- **流水导入批次**（BankImport，`acc_bank_import`）：一次上传解析的留痕单据，状态已解析/解析失败/已导入；已导入只读不可删，暂存行随批次级联清除。
- **银行对账记录**（BankReconciliation，`acc_bank_reconciliation`）：流水与已审核凭证的 m-n 金额勾稽；同一对流水-凭证仅一条，调整=解除后重建。流水的**对账状态**（未对账/部分对账/已对账）按已对账合计自动推出。
- **承兑票据**（Bill，`acc_bill`）：承兑汇票票面主档，票号全局唯一、跨公司共享；建档随接收交易，无独立新建入口。
- **子票段**：等分化票据包的金额区间 [子票起, 子票止]，1 元=100 子票；交易与持有均精确到段，段金额=（止−起+1）÷100。
- **承兑交易**（BillTransaction，`acc_bill_transaction`）：接收/转让/兑付/贴现/调拨五类单据，草稿→已审核→已作废，是 GL 凭证来源；调拨为同公司跨银行账户的持有段迁移，不生凭证。
- **持有承兑**（BillHolding，`acc_bill_holding`）：「公司—银行账户」粒度的段级库存只读投影，由已审核交易按票全链重放、整删整建；无手工入口。
- **库存重放**（BillLedger）：按票把全部已审核交易按（发生日期, 审核时间）折叠重建持有段的机制；「后续动过不可作废」由此成立。
- **开票方向**：发票的开入（进项）/开出（销项）属性，决定过账分录的借贷安排。
- **对向发票**：内部公司互开发票时，同一张纸面发票在双方公司台账中互链的记录（一方开出↔另一方开入）；保存后可一键生成对方草稿。
- **半天桶**：以 12:00 把一天切成上午/下午两桶；桶内最早卡=上班、最晚卡=下班，段工时按 30 分钟向下取整。
- **系统设置**（`sys_setting`）：系统级全局配置单行表（同 `acc_setting`/`sal_setting` 先例，种子行恒存在、只改不增删，配置统一加字段不另建表）；含初始化完成时刻（空即未初始化、向导开放），以及**行情拉取**定时开关/最新价间隔/结算自动补与上次运行摘要。
- **权限码**（权限点）：按「域.资源:动作」组织的操作能力，由代码派生不入库；授权支持整资源/整域/全域（`*`）通配。通配码仅迁移种子可写，界面授权一律写具体码。
- **权限目录**（permission catalog）：全部权限码的代码派生清单，按「域 → 资源（prefix + 中文标签 + 动作集）」组织，是角色权限配置界面与授权 sync 的共同基准。
- **标准动作**：权限动作的常规十件套（create/read/update/delete/print/import/export/batch_delete/batch_update/batch_print），角色权限矩阵中平铺为列。
- **工作流动作**：标准动作之外的资源特有动作（审核/关闭/作废/发货/收货/对账等），矩阵中收进行内「更多」展开区。
- **初始化向导**（Setup）：全新部署首启时的一次性业务初始化——建首个超级管理员（含其首选语言）、建首个公司（从预置的约 20 种常用货币选本币、必选一套科目表模板；预置货币默认全部停用，选定本币后仅本币启用，其余须事后手动启用）、落初始化完成旗标并幂等种子内置存储接入、编号规则、物料两级分类与机加工常用计量单位。可选**示例数据**起步：完成时为首个公司写入覆盖全业务链的示例数据——客商/物料/员工等主数据、销售与采购全链单据（报价→订单→发货/入库→对账→发票，各一条链走到发票结单）、库存出入库/调拨/盘点、工序/工艺模板/BOM，以及银行账户与流水、手工凭证、报销单、工资单与销/进项发票（电气/机加工演示场景，近 3 个月日期分布；以客户编号 `C01` 幂等）。空库只需迁移即可启动，无需再跑 seeds；旗标落库后入口永久关闭。常用货币与内置角色只随初始化/迁移出现，老环境不强塞。开发/测试可用 `mix synie.demo` 一键等价初始化（管理员 admin/admin123、公司 JT 台州京泰电气有限公司 + 示例数据）。
- **内置角色**：随迁移统一种子的预置角色（当前仅 admin，持全域通配 `*` 授权，新老环境一致）；不可删、不可停用、授权只读，分配给用户照常。
- **超级管理员**：用户级旗标，绕过一切功能与数据权限检查；仅初始化向导设置，常态界面无入口。
- **首选语言**（`sys_user.preferred_language`）：用户级界面语言偏好，由初始化向导为首个用户设置一次；i18n 落地前纯占位、无任何界面效果。
- **外观模式**（Appearance）：本机界面色阶偏好（仅浏览器本地持久化，不落用户表、不跨设备同步），三选一——**白天**（强制浅色）、**黑夜**（强制深色）、**系统**（跟随操作系统 `prefers-color-scheme`，OS 变更时即时跟随）；默认未选时为系统。首屏在 hydrate 前按本机键解析并挂到文档根，避免反色闪烁；同源多标签页经本机存储变更即时同步。**品牌仪式面**（启动幕布、App Shell 左侧模块图标栏、登录页左栏品牌区、向导同类品牌块）恒定深色不跟随；**业务/表单面**（内容区、二级菜单、顶栏、登录右栏、业务页）跟随色阶。切换入口为三选一控件（白天 / 系统 / 黑夜），置于用户菜单；登录页与初始化向导提供同款弱化入口。打印走独立浅色文档窗，不在外观模式范围内。纯外观色阶，不是品牌皮肤，不按公司/角色分主题。
- **公司授权**：用户级数据权限，显式授权、默认不可见（fail-closed）；挂用户不挂角色。
- **审计日志**（AuditLog，`sys_audit_log`）：数据写操作的字段级留痕（旧值→新值），只增不改不删；敏感字段不落值。
- **编号规则**（NumberingRule，`sys_numbering_rule`）：绑定单据类型的自动取号规则，内容为固定文本/记录字段/序号的有序段组合，每单据至多一条启用；序号按段渲染结果（及公司）隐含的范围独立计数，无独立重置周期。字段段取值为空时省略该段（不报错）；序号 `padding` 为 0 表示不补零，1..12 为补零位数。
- **存储接入点**（StorageEndpoint，`sys_storage`）：文件字节的存放位置（本地/对象存储）；全局唯一默认，新上传落默认点。
- **文件挂接**（Attachment，`sys_attachment`）：文件与业务记录的关联；有挂接的文件不可删，下载授权按挂接宿主可见性判定。
- **销售订单**（SalesOrder，`sal_order`）：公司向客户承诺供货的订货单据，纯业务承诺——审核不派生任何会计或库存后果；实物出库与未开票应收由**销售发货单**承载，开票仍由发票模块承载；对手限客户/内部公司两类（多态引用，同 Party 机制）；一单一币：货币与汇率挂订单头，默认公司本币，全部条目跟随换算；状态机 草稿→已审核→（已关闭/已作废），仅草稿可改可删，已审核锁死无反审核，关闭不可逆。已关闭订单不可再发货；存在已审核未作废发货引用其条目时订单不可作废（须先作废相关发货）。
- **订单条目**（SalesOrderItem，`sal_order_item`）：销售订单的商品行（物料、单位、数量、原币含税单价、税率、行备注）；行单位限该物料默认单位或其单位转换单位，金额四列全部系统算不可手改（金额链，见「汇率」），随单级联删除；另冗余**已发数量**（默认单位），由销售发货审核/作废同步加减。
- **订单类型**（`order_type`）：销售订单的分类——常规/样品两类，必填，保存后锁死不可改（换类型只能作废重开）；两类共用状态机、编号规则与权限码，系统层面不区别对待。
- **常规订单**（regular）：条目逐行强制引用**有效报价条目**（`quotation_item_id` 非空）；单价与单位随报价锁定——固定价直接带入，数量梯度按行数量自动套档（低于首档起订量视为无报价、保存报错），税率带入后仍可手改；客户/公司/日期/币种有存量条目时不可改（前端改即清空条目），审核时逐行复核报价引用与价格。
- **样品订单**（sample）：条目任选物料、手填单价，不带报价引用（`quotation_item_id` 必空）；单行数量 ≤ 供应链设置的样品上限（按行录入数量直接比，不做单位换算），审核复核。
- **本币**（`bas_company.base_currency_id`）：公司（记账主体）的记账货币，公司必填属性；单据双币换算的目标口径，报表聚合一律读本币口径。须为启用中的货币；被引用为本币的货币不可停用。
- **原币**（交易货币）：单据对交易对手报价所用的货币，一单一币挂单据头，默认取单据公司本币；原币即本币的单据照常落双套金额（汇率 1、两套同值），仅展示层简化。新建单据仅可选启用中的货币。
- **货币启停**（`bas_currency.active`）：币种主数据的启用状态；新建默认启用。停用**拦新不拦旧**——新单据币种与公司本币只能选启用币种，历史引用不受影响；被任一家公司引用为本币的不可停用。初始化向导预置常用货币时全部停用，选定本币后仅本币启用，其余须在货币管理页手动启用。
- **汇率**：单据头上原币→本币的换算率，手工填写（无汇率主数据表）；原币即本币时强制为 1，外币必填且大于零。金额链：原币金额=数量×原币单价（2 位）→本币金额=原币金额×汇率（2 位）；本币单价=原币单价×汇率（4 位）仅展示参考，不参与金额计算。
- **订单条目快照**（`material_code/material_name/material_spec/customer_part_no/unit_name`＋图纸挂接）：行保存时冻结的物料/单位口径，随审核锁死；主数据后续变更不回溯，订单上下文展示一律以快照为准。图纸为挂接复制（共享不可变文件），非字节复制。
- **已关闭**（closed）：销售订单的终止态——单据曾生效、经人工标记不再履行剩余（含部分已发后关单）；与已作废之别在于作废表示单据不该存在（有已审核发货时不可作废），关闭表示生效后提前终止剩余。关闭后不可再开销售发货；关闭不改已发数量与历史发货，已关闭订单上的发货仍可作废纠错。
- **销售报价单**（SalesQuotation，`sal_quotation`）：公司向客户/内部公司承诺价格的单据——纯价格承诺清单，条目只报单价没有数量，不构成数量/供货承诺（那是销售订单），不派生会计或库存后果；对手同 Party 机制限客户/内部公司；一单一币挂头默认公司本币，无汇率无双币（报价无金额可换算）；状态机 草稿→已审核→已作废（无「已关闭」，无反审核），作废兼承担录错与提前撤回。
- **报价条目**（SalesQuotationItem，`sal_quotation_item`）：报价单的价格行（物料、单位、定价模式、含税单价、税率、行备注），同一报价单内（物料, 单位）唯一（同物料不同单位是两种成交口径，允许并存）；行单位限物料默认单位或其单位转换单位；物料信息快照口径同订单条目（无图纸挂接）。
- **定价模式**（`pricing_mode`）：报价条目的计价形态，逐条目选择——固定价（行上一口含税价）、数量梯度（价在价格档上，行单价空置），以及**行情挂钩**（承诺计价公式，行上不含死价）；枚举可扩，新模式=加值+必要的子表/字段。
- **行情挂钩**（定价模式）：报价条目承诺「按某行情品种某价类 ± 调整项」的计价规则，不在报价单上冻结金属基价；销售订单（及采购订单）引用该条目时，按点价日取行情价点算出含税单价并冻在订单行。与固定价/数量梯度互斥（同行只选一种模式）。该模式尚未实现，两侧报价均在数据模型预留。
- **点价**：将行情挂钩公式结算为具体含税单价的动作；v1 默认在成单时（订单日期）发生，单价一旦写入订单行即冻结，盘面后续变动不回溯。
- **价格档**（QuotationTier，`sal_quotation_tier`）：数量梯度条目的起量价阶梯——每档 =（起订量, 含税档价），订购量 ≥ 起订量即适用该档价，档间上界由下一档隐含，低于首档起订量视为无报价；起订量同条目内唯一，档价不强制随量递减。
- **供应链设置**：供应链配置入口（菜单「供应链 → 设置」），页内分 **销售** / **采购** 两个 Tab。**全局标量**落 `sal_setting` 单行表（种子行恒存在，只读改不增删，同 `acc_setting` 先例）——销售：样品订单单行数量上限、**发货超发比例**；采购：零星订单单行数量上限、**入库超收比例**。**按公司的默认过账科目**落独立资源**公司默认过账科目**（不塞进 `sal_setting` / 公司主数据）。权限资源码仍为 `sales.setting`。
- **公司默认过账科目**（CompanyAccountDefault，一公司一行）：各公司在发货/入库表头预填用的借贷科目默认值——四槽均可空：发货借方、发货贷方、入库借方、入库贷方；校验口径与对应单据头科目完全一致（发货借强制「未开票应收」角色、入库贷强制「未开票应付」角色，另一侧本公司启用非汇总、角色不限）。设置页先选公司再编：销售 Tab 只维护发货两槽、采购 Tab 只维护入库两槽（本 Tab 保存 upsert 本侧两槽，不覆盖对侧）。无默认不挡建单；改默认不追溯已有单据。被引用的科目不可删除（同其它科目引用）。
- **已发数量**（`shipped_qty`）：订单条目上按物料**默认单位**累计的已履约出库量；事实来源为已审核未作废的销售发货行，条目列是受控投影——发货审核加、作废减；剩余可发 = 订购数量（折默认单位）− 已发数量（容差内超发时可负）。
- **发货超发比例**（`delivery_overship_ratio`）：供应链设置中的全局容差（小数，如 0.05=5%），默认 0（禁超发），合法范围 0～100%；发货审核时要求该订单条目「累计已发（含本单）≤ 订购 base × (1 + 比例)」。
- **销售发货单**（SalesDelivery，`sal_delivery`）：对客户/内部公司的**履约出库**单据——审核同一事务内写负向库存分录、按行累加订单已发数量，并在本币过账金额大于零时按头上借贷科目生成总账分录（借方强制「未开票应收」角色科目、贷方自选；金额按订单条目快照本币含税金额 × 本行 base 占订购 base 比例汇总）；金额为零（赠品等）则**跳过总账**，但仍要求头上借贷科目已填（录入纪律，不造零金额名义分录）。头=公司+对手（客户/内部公司）+发货日期+过账日期（有金额审核时必填，默认发货日期）+头仓（可空，仅新建行预填）+备注+单号+**借贷科目（草稿保存即必填，表单位于条目表下方；新建/换公司时按该公司默认过账科目整组覆盖代入，无默认则清空；打开已有草稿且未改公司则保留单上科目）**；行=必挂订单条目+物料（须与条目一致）+单位/数量/base_qty+行仓（必填，本公司启用叶子）+行备注+物料快照+订单条目快照。可跨多张订单，但全部行订单的公司、对手、原币必须一致；同一订单条目可在一单多行（如分仓）。状态机 草稿→已审核→（已作废），仅草稿可改可删，无反审核、无红冲、无关闭态；作废回滚库存分录、总账分录与已发数量（作废仍过负库存校验）；任一条目存在非零**已对账数量**（被生效中销售对账单消耗）则整单不可作废，须先撤回/作废相关对账单。只记卖方公司账，不向内部公司对手自动镜像入库。菜单在供应链/库存下，与其他库存单、库存余额、库存分录同组（业务线履约出库，相对「其他库存单」）；列表双 tab（发货条目默认 / 发货单）。权限码 `sales.delivery`。
- **发货条目**（SalesDeliveryItem，`sal_delivery_item`）：销售发货单的商品行；单位限物料默认单位或其转换单位，系统折算默认单位 `base_qty`（6 位）供库存与已发累计；行保存冻结物料快照（编号/名称/规格/客户料号/本行单位名，并复制物料 `drawing` 图纸挂接到行）与订单条目快照（订单号、订购量与订购 base、价税与本币金额、税率、币种代码、订单行单位名等），过账与展示以快照为准。图纸为挂接复制（共享不可变文件），非字节复制；行/发货单删除时挂接一并清理。另冗余**已对账数量**（`reconciled_qty`，见同名词条），由销售对账单生效/回退同步加减。
- **报价截止**（`valid_until`）：报价单有效期终点，截止当日仍有效；**已过期**是派生展示态（已审核 且 当前日期 > 截止日），不落库不跑定时任务翻状态。报价延期不改截止日，复制重报一张新单。
- **有效报价条目**：对某张销售订单而言可选的报价条目——报价单已审核，且订单日期落在 [报价日期, 截止日] 区间内，且报价单公司、对手（客户/内部公司一视同仁）、币种均与订单一致；有效区间起点即报价日期，无独立生效日。采购侧对称：有效采购报价条目对采购订单的同口径定义（对手换供应商/内部公司）。
- **采购报价单**（PurchaseQuotation，`pur_quotation`）：供应商/内部公司向本公司承诺价格的单据——全镜像销售报价单：纯价格承诺清单，条目只报单价没有数量，不构成数量/供货承诺（那是采购订单），不派生会计或库存后果；对手限供应商/内部公司（Party 机制，不能是单据公司自身）；一单一币挂头默认公司本币，无汇率无双币；状态机 草稿→已审核→已作废（无「已关闭」，无反审核）；报价截止/已过期口径同销售报价。条目结构（(物料, 单位) 唯一、定价模式、价格档、行快照）与销侧一致；行情挂钩模式同步预留未实现。采购侧不校验客户物料约束，任何物料均可报价。权限码 `purchase.quotation`。
- **采购订单**（PurchaseOrder，`pur_order`）：公司向供应商承诺采购的订货单据，纯业务承诺——审核不派生任何会计或库存后果；实物入库与未开票应付由**采购入库单**承载，进项发票仍由发票模块承载；对手限供应商/内部公司两类（多态引用，同 Party 机制，不能是单据公司自身）；一单一币：货币与汇率挂订单头，默认公司本币，全部条目跟随换算（双币金额链同销售订单）；状态机 草稿→已审核→（已关闭/已作废），仅草稿可改可删，已审核锁死无反审核，关闭不可逆。已关闭订单不可再入库；存在已审核未作废入库引用其条目时订单不可作废（须先作废相关入库）。权限码 `purchase.order`。
- **采购订单类型**（`order_type`）：采购订单的分类——常规/零星两类，必填，保存后锁死不可改（换类型只能作废重开）；两类共用状态机、编号规则与权限码，系统层面不区别对待。
- **常规采购订单**（regular）：条目逐行强制引用**有效采购报价条目**（`quotation_item_id` 非空）；单价与单位随报价锁定——固定价直接带入，数量梯度按行数量自动套档（低于首档起订量视为无报价、保存报错），税率带入后仍可手改；对手/公司/日期/币种有存量条目时不可改（前端改即清空条目），审核时逐行复核报价引用与价格。
- **零星订单**（spot）：采购订单的自由录价类型——条目任选物料、手填单价，不带报价引用（`quotation_item_id` 必空）；单行数量 ≤ 供应链设置的零星上限（`spot_item_max_qty`，默认 100，按行录入数量直接比，不做单位换算），审核复核。采购侧不校验客户物料约束。
- **采购订单条目**（PurchaseOrderItem，`pur_order_item`）：采购订单的商品行（物料、单位、数量、原币含税单价、税率、行备注）；行单位限该物料默认单位或其单位转换单位，金额四列全部系统算不可手改（金额链，见「汇率」），随单级联删除；行保存冻结物料快照（编号/名称/规格/客户料号/单位名＋图纸挂接复制，机制同订单条目快照）；另冗余**已收数量**（默认单位），由采购入库审核/作废同步加减。
- **已收数量**（`received_qty`）：采购订单条目上按物料**默认单位**累计的已履约入库量；事实来源为已审核未作废的采购入库行，条目列是受控投影——入库审核加、作废减；剩余可收 = 订购数量（折默认单位）− 已收数量（容差内超收时可负）。
- **入库超收比例**（`receipt_overreceive_ratio`）：供应链设置中的全局容差（小数，如 0.05=5%），默认 0（禁超收），合法范围 0～100%；入库审核时要求该订单条目「累计已收（含本单）≤ 订购 base × (1 + 比例)」。
- **采购入库单**（PurchaseReceipt，`pur_receipt`）：对供应商/内部公司的**履约入库**单据——审核同一事务内写正向库存分录、按行累加订单条目已收数量，并在本币过账金额大于零时按头上借贷科目生成总账分录（贷方强制「未开票应付」角色科目、借方自选；金额按订单条目快照本币含税金额 × 本行 base 占订购 base 比例汇总）；金额为零（赠品等）则**跳过总账**，但仍要求头上借贷科目已填（录入纪律，不造零金额名义分录），仍出库存与已收。头=公司+对手（供应商/内部公司）+入库日期+过账日期（有金额审核时必填，默认入库日期）+头仓（可空，仅新建行预填）+备注+单号+**借贷科目（草稿保存即必填，表单位于条目表下方；新建/换公司时按该公司默认过账科目整组覆盖代入，无默认则清空；打开已有草稿且未改公司则保留单上科目）**；行=必挂订单条目+物料（须与条目一致）+单位/数量/base_qty+行仓（必填，本公司启用叶子）+行备注+物料快照+订单条目快照。可跨多张订单，但全部行订单的公司、对手、原币必须一致；同一订单条目可在一单多行（如分仓）。状态机 草稿→已审核→（已作废），仅草稿可改可删，无反审核、无红冲、无关闭态；作废回滚库存分录、总账分录与已收数量（作废减库存，仍过负库存校验）。只记买方公司账，不向内部公司对手自动镜像出库。菜单在供应链/库存下，与销售发货同组；列表双 tab（入库条目默认 / 入库单）。权限码 `purchase.receipt`。
- **入库条目**（PurchaseReceiptItem，`pur_receipt_item`）：采购入库单的商品行；单位限物料默认单位或其转换单位，系统折算默认单位 `base_qty`（6 位）供库存与已收累计；行保存冻结物料快照（编号/名称/规格/客户料号/本行单位名，并复制物料 `drawing` 图纸挂接到行）与订单条目快照（订单号、订购量与订购 base、价税与本币金额、税率、币种代码、订单行单位名等），过账与展示以快照为准。图纸为挂接复制（共享不可变文件），非字节复制；行/入库单删除时挂接一并清理。另冗余**已对账数量**（`reconciled_qty`，见同名词条），由采购对账单生效/回退同步加减。
- **参保类型**（InsuranceType，`hr_employees.insurance_types`）：员工参加的原子险种多选，任意组合不设互斥；当前状态快照（变更靠审计日志追溯），纯档案字段不参与工资计算，列表可按「包含/不包含某险种」筛选。
- **原子险种**：参保类型的八个固定枚举值——社保工伤/社保失业/社保医疗/社保养老/社保生育/公积金/商保工伤/商保医疗；「五险一金」「三险」不是系统概念只是勾选组合，商保值不区分保司与保单。
- **行情品种**（MarketInstrument）：可被业务引用的一条**稳定参考价序列**主数据（如「沪铜」「长江电解铜」），全局共享不分公司；不随交易所换月而新建主数据；覆盖交易所序列、现货指数等。必填口径：币种、计量单位、默认价类、来源类型；可选合约代码等仅作补充，不作主数据拆分维度。菜单口语可叫「期货管理」，领域与文档以「行情」为准。
- **行情管理**（模块边界）：基础数据下的行情主数据、时序价与盯盘工作台——只观测、落库、被引用，不持仓、不交割、不记账；报价挂钩与未来库存估值均只读行情。录入通道含手工补录与自动/手动拉取，采集周期不作品种业务必填项；界面入口见**行情页**。
- **行情价点**（MarketPricePoint）：某行情品种在某一观测时刻的一条不可变价格事实——（品种, 观测时刻, 价格, 币种, 计量单位, 价类, 来源）；币种与计量单位继承自品种且不可与品种口径不一致；只追加不覆盖。有效价点在（品种, 观测时刻, 价类）上唯一；录错则作废后重录（或作废+新点一气呵成的更正），已作废不参与取价。录入通道含手工补录与拉取，同模型落库。
- **价类**（`price_kind`）：价点语义分类（如结算价、均价、最新价），枚举可扩；品种钉默认价类，价点可覆盖；同一品种多价类并存时，业务引用须指定价类（或显式用默认）。交易所序列自动拉取时：盘中写**最新价**，日终写**结算价**。
- **来源类型**（行情品种）：外部价源分类——交易所序列 / 现货指数 / 其他；只描述价从哪类市场来，不引入持仓或交易语义。
- **取价**：按（品种, 价类, 时点）从有效价点中解析出一个价格的只读规则；默认语义为「≤ 目标时点的最近一条有效价点」。具体业务（点价日、估值日）指定目标时点与价类（可回落到品种默认价类）。
- **行情菜单**：基础数据下「行情」分组，唯一入口为**行情页**（`/base/market`）；口语「期货管理」可作描述，不以「期货交易」建模。
- **行情页**：行情工作台——上图（勾选同币种+同计量单位的启用品种做多序列对比，附 KPI）+ 下表（价点明细 / 品种维护双 Tab）；图与表弱联动。图与 KPI 走价点时序只读查询；权限仍分 `base.market_instrument` 与 `base.market_price` 两套按能力拼装，不另建合并资源码。
- **行情品种停用**：品种退出主路径的方式——停用后不可再被新业务选为挂钩目标、不作为默认定价源，历史价点与既有引用保留；无任何价点时允许物理删除。编码创建后宜锁死，名称可改。
- **行情拉取**（MarketFetch）：按品种配置从外部价源写入行情价点的通道（`source=fetch`）。**启用拉取**（`fetch_enabled`）的品种才参与定时/手动刷新；映射字段挂品种主数据——**外部最新价代码**（如主连 `CU0`）、**外部品种组**（如上期所日数据 `cu`）。系统级节奏在 `sys_setting`（界面：基础数据→基础设置→行情拉取）：**定时总开关**、**最新价间隔**（仅 30/60/120 分钟）、**结算自动补拉**开关；交易时段按间隔拉最新价，工作日日盘收市后补持仓量最大合约的结算价。手动刷新：立即拉最新价，若已过结算窗口且配置允许则顺带补结算。运行后写**上次拉取摘要**供界面展示。现货指数（如长江铜）默不启用，继续手工。
- **销售对账单**（SalesReconciliation，`sal_reconciliation`）：公司就发货事实与客户/内部公司核对应收的勾稽单据，是「未开票应收 → 应收账款」迁移的业务枢纽——自身确认/结单不过往来重分类，常规单的重分类由发票审核关联触发（见对账类型与 ADR 2026-07-21）。头=公司+对手（客户/内部公司，Party 机制）+对账类型（必填、保存后锁死）+借贷科目槽（建单按「公司默认过账科目」的发货借方/发货贷方整组代入，可改；贷方强制「未开票应收」角色，借方校验口径随类型见对账类型）+单号+备注；条目=必挂发货条目+对账数量（行单位，≤ 该条目剩余可对账量）+行备注，物料/价税口径沿用发货条目快照。行金额=对账数量×快照原币含税单价（2 位）→本币金额=×源订单汇率（2 位），单头不设汇率、合计=Σ 行。条目池=同公司同对手、已审核未作废发货、剩余可对账量>0 的条目，单内同币种；不做发货日期强约束（列表给筛选）。草稿不占量；消耗发生在生效时点（常规单确认时、赠送/样品单结单时逐行校验剩余可对账量并累加已对账数量，撤回/作废回滚）。编号挂编号规则（留空自动取号可手填），两类共用一条系列；权限码 `sales.reconciliation`；菜单供应链销售组，列表双 tab（对账条目/对账单）。分次部分对账的 2 位舍入尾差不强行配平，残留由赠送/样品单核销通道兜底。
- **对账类型**（`reconciliation_type`）：销售对账单的分类——常规/赠送样品两类，手选必填、保存后锁死（换类型只能作废重开），共用编号规则与权限码，状态机与条目约束不同。**常规对账单**：状态机 草稿→客户已确认→已结单；草稿可改可删（客户确认前的多轮修改都在草稿内完成），确认锁死、未关联发票前可撤回回草稿；条目禁勾样品订单来源与零金额行；「已结单」由开票触发——开出方向发票在草稿上选关联对账单，审核时校验（对账单=客户已确认、公司/对手一致、发票价税合计=对账单合计，一对一），过账=发票正常三行（借应收账款/贷金额科目/贷税额科目）+冲回组（借单头借方科目=发货贷方口径/贷单头贷方科目=未开票应收，金额=价税合计），发票作废/红冲自动解除关联并把对账单退回客户已确认；已结单无独立作废入口，纠错走发票侧（红冲/作废→退回已确认→撤回→回草稿改）。**赠送/样品对账单**：状态机 草稿→已结单（审核即结单）；条目不限来源——样品订单来源、任意订单零金额行、正常金额行均可（典型场景：超发尾差核销，发 1020 客户只认 1000，余 20 走此单结清）；结单过账 借单头借方科目（费用/损失类，手选，本公司启用非汇总）/贷单头贷方科目（未开票应收角色），金额=勾选条目价税合计，为零跳过分录（同销售发货先例）；已结单可作废，作废回滚分录组与已对账数量。
- **客户确认**（confirmed）：常规对账单的中间生效态——客户线下认可账单后由内部人员手工翻入，是发票关联的前置闸门；数量消耗（已对账数量累加）在此时点发生，未关联发票前可撤回。
- **已对账数量**（`reconciled_qty`）：发货条目/入库条目上按物料默认单位累计的已对账量；事实来源为生效中的对账单行（常规单自客户/供应商确认起、赠送/样品单自结单起），条目列是受控投影——确认/结单加、撤回/作废减；剩余可对账 = 发货/入库 base − 已对账数量。存在非零已对账数量的发货/入库条目，其发货单/入库单不可作废（须先撤回/作废相关对账单），同「有已审核发货订单不可作废」先例。
- **采购对账单**（PurchaseReconciliation，`pur_reconciliation`）：公司就入库事实与供应商/内部公司核对应付的勾稽单据，是「未开票应付 → 应付账款」迁移的业务枢纽——自身确认/结单不过往来重分类，常规单的重分类由开入发票审核关联触发（全镜像销售对账单，见 ADR 2026-07-21 采购对账单）。头=公司+对手（供应商/内部公司，Party 机制）+对账类型（必填、保存后锁死）+借贷科目槽（建单按「公司默认过账科目」的入库贷方/入库借方整组代入，可改；借方强制「未开票应付」角色，贷方=入库借方口径（常规）/收益类（赠送样品））+单号+备注；条目=必挂入库条目+对账数量（行单位，≤ 该条目剩余可对账量）+行备注，物料/价税口径沿用入库条目快照。行金额=对账数量×快照原币含税单价（2 位）→本币金额=×源订单汇率（2 位），单头不设汇率、合计=Σ 行。条目池=同公司同对手、已审核未作废入库、剩余可对账量>0 的条目，单内同币种；常规单禁勾零金额行（采购订单无样品类型，无"样品来源"禁用），赠送/样品单不限来源（兼任超收尾差核销通道）。状态机不对称：常规单 草稿→供应商已确认→已结单（确认可撤回（未关联发票前），已结单无独立作废入口、由发票作废/红冲退回确认态）；赠送/样品单 草稿→已结单→（已作废），结单审核即过账 借未开票应付(带对手)/贷收益类科目、零金额跳过分录。草稿不占量；消耗发生在生效时点（常规单确认时、赠送/样品单结单时逐行校验剩余可对账量并累加已对账数量，撤回/作废回滚）。编号挂编号规则（留空自动取号可手填），两类共用一条系列；权限码 `purchase.reconciliation`；菜单供应链交易组「采购对账」，列表双 tab（对账条目/对账单）。术语切割：业务域「对账」并称**销售对账单**与**采购对账单**（入库与收票、发货与开票之间的勾稽单据），与银行流水勾稽的**银行对账记录**相切割。
- **供应商确认**（confirmed）：常规采购对账单的中间生效态——供应商线下认可账单后由内部人员手工翻入，是开入发票关联的前置闸门；数量消耗（已对账数量累加）在此时点发生，未关联发票前可撤回。
- **费用角色**（报销类型）：差旅费 / 办公费 / 业务招待费 / 交通费 / 其他费用五个科目角色，枚举写死（同往来角色先例），新增类别=加枚举值；不进应收应付报表，其科目分录不要求带对手。报销类单据上「选报销类型」即选费用角色——金额/费用科目=本公司挂该角色的唯一启用非汇总科目，缺挂或多挂则报错改手选；角色只作带科目的选择器，单据上不存角色值、只存科目。
- **其他应付款**（`other_payable`）：往来角色，员工报销挂账专用——费用报销发票审核贷记（带员工对手）、报销单审核借记核销；纳入净应付（见 ADR 2026-07-21 费用报销）。
- **费用报销发票**：增值税发票第四类型——方向开入、对手限员工、不关联任何对账单；选报销类型自动带出金额科目与往来科目（税额科目手选）；审核即完成含税支出挂账（借费用/税额、贷其他应付款-员工），不生成下游单据；被报销单挂票行引用（含草稿报销单）后不可作废/红冲，须先移除行或作废报销单。
- **报销单**（ExpenseReport，`acc_expense_report`）：对员工付款的核销单据——费用报销发票审核完成挂账（第一步）后，给员工打款是第二步。头=公司+员工+报销日期+过账日期+付款科目（贷方，本公司启用非汇总本币科目）+单号（挂编号规则，留空自动取号）+备注；状态机 草稿→已审核→（已作废），仅草稿可改可删，无反审核、无红冲；审核过账=挂票行借各发票往来科目（带员工对手，金额=各票价税合计）+无票行借费用科目、贷付款科目（Σ 全部行）；作废=分录标记作废+解除全部发票引用（发票回到「已审核未报销」，可被新报销单重新引用或正常作废/红冲）。权限码 `acc.expense_report`，菜单财务组发票旁。
- **挂票行**：报销单行之一，引用本公司该员工「已审核未报销」的费用报销发票；一票一单全额核销（v1 不支持部分付款），行金额=发票价税合计。
- **无票行**：报销单行之一，手填摘要+金额+报销类型（带费用科目），税额恒 0——五金店现金等非税支出通道；无票支出无挂账时点，审核时挂账与付款一步两清（借费用、贷付款科目）。
