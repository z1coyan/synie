# 总账财务

## 概述

总账财务模块是全系统的财务记账主干：维护每个公司的会计科目树，承载手工会计凭证的录入、审核与取消，并把所有业务单据（凭证、增值税发票、承兑交易等）审核产生的财务事实统一沉淀为总账分录，供应收应付报表等财务视图直接查询。服务对象是财务人员与需要账龄/往来口径数据的管理者。本模块不覆盖增值税发票、承兑汇票、银行流水与银行对账本身（各有独立模块，只在本模块留下分录结果）；试算平衡、科目余额表、明细账、期间关账与多币种折算尚在规划中。

## 核心概念

- **总账分录**：全系统唯一的财务事实。只追加、不可改，没有任何人工直接写入口；一切分录由来源单据在审核时派生，财务报表只查分录、不查业务单据。
- **来源单据**：产生分录的业务单据（手工凭证、增值税发票、承兑交易等）。每条分录都记录其来源单据，可在分录查询中点击回溯。
- **会计科目**：每公司一棵科目树，编码公司内唯一。分「汇总科目」与「叶子科目」：只有叶子科目能入账，汇总科目只做层级归集（与物料的「叶子分类」语义相反，见术语表）。
- **科目角色**：标记往来科目在应收应付口径下的用途（六角色），见术语表；应收应付报表按角色圈定科目范围。
- **手工会计凭证**：人工录入的记账凭证（头 + 若干分录行），是地位与其他业务单据平等的来源单据，走 草稿 → 已审核 → 已取消 的生命周期。
- **红字冲销（红冲）**：对已入账分录组的更正方式——不删不改原分录，生成一组等额红字（负金额）分录与之对冲，原组标记「已被红冲」。
- **往来对手**：分录行上的交易对手（供应商 / 客户 / 内部公司），应收应付报表按对手归属余额。
- **应收应付报表**：纯总账口径的时点余额视图，见术语表。

## 功能清单

### 科目表管理

- 每个公司独立一棵科目树，科目编码在公司内唯一；科目有所属公司、余额方向（借/贷）、启用标志、可选币种。
- 新公司可从内置模板整套建账：企业会计准则（6 要素）、小企业会计准则（5 要素）、国际通用精简（5 要素）三选一；会计要素本身是树根。只有尚无任何科目的公司才能初始化，初始化一次成型、不可重复执行。
- 科目可增改，但有下级科目的不能删除；科目创建后不允许换公司。
- 汇总科目与停用科目不能入账：凭证行选科目时即拦截，过账时再校验一遍。
- 往来科目可挂科目角色（六角色之一），供应收应付报表圈定范围；规则：只能挂叶子科目、外币科目不可挂（人民币科目与未设币种的科目可挂）、同公司多个科目可挂同一角色（报表自动合并）。内置模板已为标准往来科目（应收账款、应付账款、预收账款、预付账款）预设角色。

### 手工凭证

- 凭证由头（编号、单据日期、过账日期、备注）与若干分录行（科目、借/贷金额、对手、行备注）构成。
- 编号公司内唯一；留空时按可配置的编号规则自动取号，手工填写则原样保留。
- 状态流转：**草稿 → 已审核 → 已取消**。草稿态头与行都可改可删；已审核、已取消后一律锁死。已取消是终态，不可复审，更正须另开新凭证。
- 草稿行允许借贷两边都为零（逐步录入），但每行至多一边大于零。
- 审核时校验（任一不满足即拒绝）：分录行不少于两行、每行恰有一边大于零、借贷合计配平、科目均属本公司且启用且非汇总、对手类型与对手必须同填同空、挂了科目角色的往来科目必须填对手。审核时必须有过账日期（可在审核时补填）；审核通过即在同一事务内生成总账分录，并记录提交人与提交时间。
- 取消已审核凭证会将其全部分录标记作废；**已被银行对账引用的凭证不可取消**，须先解除对账。
- 凭证行的币种自动取科目币种，不可手改；删除草稿凭证时其分录行一并删除。
- 凭证支持审核、取消两个独立操作权限，可与录入权限分开授予。

### 总账分录查询

- 分录按发生顺序（序号）展示，可按科目、过账日期、来源单据、对手等维度筛选。
- 分录只读：没有修改、删除入口，也没有人工新增入口。
- 查询返回全部分录，包括已作废行与红字行，以标记列区分；作废不删行，仅标记「已作废」，红字行标记「红字冲销行」、被红冲的原组标记「已被红冲」。
- 来源单据与对手渲染为可点击的名称链接，可回溯原始单据。

### 过账与更正机制

- 所有来源单据（凭证、发票、承兑交易等）审核时走同一套过账校验与写入，校验规则与手工凭证审核一致（配平、行数、单边金额、科目可用、角色科目强制对手）。
- 单据作废 = 其分录整组标记「已作废」，报表等消费侧显式过滤。
- 已入账分录组支持红冲：生成等额红字组对冲，原组保留并标记「已被红冲」；红字组与原组在余额计算中自然对冲归零。红冲拷贝原行，故历史遗留的无对手行也能红冲（豁免角色科目的强制对手校验）。

### 应收应付报表

- 纯总账时点余额：取截至日（含）之前未作废的分录，仅限挂了科目角色的科目，按对手 × 角色轧差；没过账就不存在，不区分币种。
- 报表分应收、应付两个视角（tab）：应收列示未开票应收、应收账款、预收账款与净应收，应付对称；净额口径见术语表「净应收/净应付」。各角色按自然方向出正数，反向余额显示负数，不搬列。
- 全零的对手行不出；存量无对手的角色科目分录汇总为「未指定对手」兜底行（见术语表），列合计恒等于圈定科目的账面余额。
- 同公司多个科目挂同一角色时自动合并为一列。
- 报表数字可点击下钻，跳转分录查询页并预置筛选（对手 + 角色科目集 + 截至日）。

## 业务流程

- **建账**：新公司选择准则模板整套初始化科目树 → 按实际核算需要增补/调整科目 → 确认往来科目的角色挂载（模板已预设标准四科目）→ 开始记账。
- **记账**：录入手工凭证（草稿，逐步补行）→ 审核（配平等校验通过、同事务生成分录）→ 分录入账，应收应付报表等视图即时可见 → 发现错误时，已审核凭证可取消（分录整组作废）或经红冲对冲，均不留修改痕迹之外的变更。
- **业务单据入账**：发票、承兑交易等业务单据审核时与手工凭证走同一过账通道，分录统一可查、统一进报表；单据作废对应分录整组作废。
