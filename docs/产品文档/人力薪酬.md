# 人力薪酬

## 概述

人力薪酬模块覆盖工厂计时工资的全链路：员工档案、考勤机打卡导入、日考勤计算与补卡修正、按考勤生成月工资单、工资发放与员工借款抵扣。服务对象是人事与财务经办人。模块为纯计时模型——无班次排班、无迟到早退判定、无工作日历,周末节假日同公式;工资费用凭证与银行代发对接(GL 侧)尚未覆盖。

## 核心概念

- **员工**:全局主数据(不挂公司),持考勤机编号、日薪、月补贴、参保类型等档案字段。
- **打卡记录**:考勤机原始打卡事实(员工+时刻),导入是唯一入口,不可改;(员工, 时刻) 唯一。
- **考勤导入批次**:一次 .dat 导入的留痕单据,存解析摘要;删除批次即整批撤销。
- **[未知] 员工**:导入勾选自动创建的占位员工,身份落实后直接改名,无合并动线。
- **日考勤**:按 (员工, 自然日) 派生的考勤结果行(四段时刻+工时+加班+奖励工日),自动重算,人工不可直改。
- **补卡单**:计算层的虚拟打卡单据(一单一人一天多卡),是修正考勤的唯一入口;原始打卡永不修改。
- **工资单**:(员工, 月) 唯一的薪资快照单据;待发放可改可删,已发放锁死。
- **发放记录**:一次实际发钱的留痕;首笔翻转已发放,后续为补发;金额可负(冲回)。
- **借款台账**:员工借款/归还流水,余额 = Σ借款 − Σ归还(员工级,不做单笔核销);工资发放时按借款抵扣自动生成归还行。

## 功能清单

### 员工主档

维护员工档案,是考勤匹配与工资快照的数据源。

- 员工编号留空自动取号,手填原样保留;编号全局唯一。
- 考勤机编号非空时全局唯一——.dat 打卡行凭它匹配员工,多台考勤机同编号视为同一人。
- 身份证号唯一(可空),按敏感字段处理,审计日志中记 `[FILTERED]`;身份证正/反面照片走统一附件,不占档案字段。
- 日薪、月补贴均可空但不能为负;生成/刷新工资单时快照进单,未填按 0 计。
- 参保类型:八个原子险种(社保工伤/失业/医疗/养老/生育、公积金、商保工伤/商保医疗)多选任意组合,不设互斥;「五险一金」「三险」只是勾选组合,商保不区分保司。记当前状态不记参保区间(变更靠审计日志),纯档案字段不参与工资计算;列表可按「包含/不包含某险种」筛选(补保排查)。
- 员工无在职/离职状态;删除员工不做业务校验(已有考勤、工资数据的员工删除会被数据关联拦住)。

### 打卡导入(.dat)

从考勤机导出的 .dat 文本导入打卡,两段式:先建批次看摘要,再执行导入。

- 行格式:考勤机编号 + 打卡时间(`YYYY-MM-DD HH:MM:SS`),tab 或空白分隔,只取前两列,其余列(机号/状态键/验证方式)忽略;编号原样匹配,不去前导零。本地时间按 UTC+8 固定偏移存储。
- 上传文件建批次即解析出摘要:总行数、坏行数、文件内重复数、已匹配/未匹配行数及未匹配编号清单(编号×行数,最多列 50 个)。解析失败(空文件、全坏行、超 10 万行上限)批次落库为「解析失败」并记原因。
- 同一文件已有非失败状态的批次时拒绝再建,防误重传;确要重导须先删除原批次。
- 执行导入仅限「已解析」状态批次;重新解析文件直写打卡表。已存在的 (员工, 时刻) 静默跳过并计数——区间重叠重导天然幂等。
- 未匹配编号的处理:默认跳过计数;勾选「自动创建不存在的员工」则按编号自动建 [未知] 员工(编号回填、员工编号自动取号),操作人须同时有员工新增权限,否则整笔拒绝。
- 打卡记录本身不可改、不可单条删;删除批次即整批撤销(其打卡全部级联删除),任何状态的批次可删,这是导错文件的唯一纠正口子。
- 执行导入与撤销批次都会自动重算受影响的日考勤。

### 日考勤计算

把当天全部卡(真实打卡+补卡)折算成工时与工日,结果落为 (员工, 日) 唯一的派生行。

- 12:00 切分上/下午两个半天桶,桶内最早卡=上班、最晚卡=下班;连按考勤机的冗余卡被自然吸收。
- 段工时按 30 分钟向下取整,上下午各自取整;上午封顶 4 小时(早来不算),下午超 4 小时部分计加班工时,正常工时日封顶 8 小时。
- 单日加班(取整后)≥ 3.5 小时额外奖励 0.5 工日(日封顶 0.5);它与月末加班补贴并行,是奖励不是小时折算。
- 缺卡:某半天桶内只有一张卡,该段计 0 工时并将当天状态标为「缺卡」;桶内无卡算无出勤,不算异常。全天无卡不生成行。
- 日考勤人工不可直改:导入执行/撤销、补卡增删改都会自动重算受影响的 (员工, 日);另有按日期区间手动重算的兜底入口(区间最长一年)。
- 月汇总视图按员工给出当月出勤天数、缺卡天数、正常/加班工时、奖励工日与月工日(月工日 = Σ正常工时 ÷ 8 + Σ奖励工日),供工资生成取数。
- 不做跨零点处理:晚班最迟 23 点前收工,按自然日切分;整班倒夜班场景本模型不覆盖。

### 补卡单

漏打卡、考勤机故障的修正入口,一单 = 一人一天的若干补卡时刻(本地钟点,1~20 个)。

- (员工, 日期) 唯一:同一天再补须编辑原单,不允许开第二张。
- 补卡时刻与真实打卡合并进桶取最早/最晚,原始打卡永不修改。
- 保存、修改、删除即自动重算当天日考勤;改单换了人或日期,新旧两天都重算。
- 录入的时刻自动截秒、去重、排序;可填备注,记录录入人。
- 无审批流,靠权限与审计留痕管控。

### 工资单

(员工, 月) 唯一的月度薪资快照,两态流转:待发放 → 已发放。

- 应发公式:应发 = 工日 × 日薪(基本工资,四舍五入到分)+ 补贴 + 奖金 − 罚款 − 借款抵扣;基本工资与应发自动计算,不可手填。所有金额字段不能为负。
- 快照来源:工日、出勤天数、缺卡天数、加班工时自考勤月汇总;日薪、补贴自员工档案。加班补贴暂不单列计价,由奖金字段人工承载。
- 按月批量生成:考勤月汇总有行的员工每人一张;无考勤不建单;已存在的 (员工, 月) 跳过不覆盖,重复生成不会冲掉手工调整。另保留手工建单入口,用于无考勤员工的特批发薪。
- 待发放态所有金额字段可手改(月中调薪、考勤机故障手调),也可「重取快照」按当前考勤与员工档案刷新工日/日薪/补贴(奖金、罚款、借款抵扣、备注不动);员工与月份是单据身份不可改,错了删单重建。
- 已发放态全字段锁死,差错走补发,不回头改单。
- 状态不由人改:首笔发放记录自动翻转为已发放,删除「发放」记录即自动翻回待发放(无论是否还有补发记录)。无「部分发放」状态,单上的实发合计(发放记录之和)即发放进度。
- 月度统计视图给出当月工资单数、未发放数、应发合计、实发合计。

### 工资发放与借款抵扣

发放记录是实际发钱的留痕,一张工资单可多条;借款台账记录员工借款与归还。

- 发放类型自动判别不让用户选:工资单待发放时的那笔为「发放」,已发放后追加的为「补发」。金额允许为负(多发冲回)、禁止为零;发放日期必填。
- 一键发放(单笔/批量):金额不收人工输入,按 应发 − 已发 差额发放,差额 ≤ 0 拒绝,防重复发放;自定义金额/负数冲回走表单录入兜底。
- 发放记录不可修改,只可删除重录;删除「发放」记录即自动翻回待发放并删除联动的台账归还行(无论是否还有补发记录)。
- 借款抵扣联动:首笔发放翻转已发放时,若单上借款抵扣 > 0,校验抵扣额不超过该员工当时借款余额(不足则整笔发放失败),并自动生成一条台账归还行(日期取发放日、反挂工资单)。
- 借款台账为借款/归还两类流水,金额必须大于零;手工行增删改自由、不校验余额方向(允许负余额);工资联动生成的归还行禁手改手删,只随发放/回退联动增删。
- 余额 = Σ借款 − Σ归还,员工级汇总,不做单笔核销;台账页提供每员工累计借款/累计归还/余额汇总。

## 业务流程

月度主线:上传 .dat 建批次看摘要 →(必要时先去员工页补考勤机编号)→ 执行导入(可勾选自动建 [未知] 员工)→ 系统自动重算日考勤 → 对「缺卡」天开补卡单修正 → 月底看考勤月汇总 → 批量生成工资单 →(待发放态手工调整或重取快照)→ 一键发放(首笔翻转已发放,借款抵扣自动还台账)→ 差额差错走补发、多发走负数冲回;导错文件则删批次整批撤销并重算。
