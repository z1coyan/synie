# 销售对账

## 概述

销售对账模块承载发货与开票之间的勾稽：公司就发货事实与客户/内部公司核对应收，产出客户认可的对账单，作为后续开票与「未开票应收 → 应收账款」重分类的业务依据。服务销售内勤与财务。

边界：常规对账单自身确认/结单不产生会计分录，应收重分类由发票审核关联触发（见 [发票](./发票.md)）；一张对账单对一张发票，不支持拆票；分次部分对账的分位尾差不配平，由赠送/样品单核销兜底。设计定案见 [ADR：销售对账单](../adr/2026-07-21-sales-reconciliation.md)。

## 核心概念

- **销售对账单**：勾稽单据，头=公司+对手+类型+借贷科目+单号+备注，条目=发货条目+对账数量+行备注。见 [术语表](../glossary.md)。
- **对账类型**：常规 / 赠送样品，手选必填、保存后锁死；两类状态机与条目约束不同。常规单走「客户确认 → 发票结单」；赠送/样品单审核即结单并过账，兼任超发尾差核销通道。
- **客户确认**：常规单的中间生效态，发票关联的前置闸门；数量消耗在此时点发生，未关联发票前可撤回。
- **已对账数量**：发货条目上默认单位累计的已对账量，生效加、回退减；剩余可对账 = 发货数量 − 已对账数量。
- **金额链**：行金额 = 对账数量 × 发货条目快照原币含税单价（2 位）→ 本币 = × 源订单汇率（2 位）；单头合计 = 各行合计，单头不设汇率。

## 功能清单

### 对账录入

- 头：公司（建后不可换）、对账单号（留空按编号规则取号，可手填）、对账类型（保存后锁死）、对手（客户/内部公司）、借方科目、贷方科目、备注。
- 借贷科目**草稿保存即必填**：贷方强制「未开票应收」角色；借方本公司启用非汇总即可（常规单为发货贷方口径，赠送/样品单为费用/损失类）。新建时按该公司默认过账科目的发货借方/发货贷方整组代入，可改。
- 行：必挂发货条目，对账数量按发货条目行单位录入，行备注；物料与价税口径沿用发货条目快照。
- 「导入所有未对账」：条目表工具栏一键导入——按条目池口径拉全部候选（已在清单的自动跳过），数量默认取剩余可对账量（折行单位），导入后仍可逐行改量或删行；清单为空且候选跨币种时需先手工新增一行钉住币种再导入。
- 条目池：同公司、同对手、已审核未作废发货、剩余可对账量大于零的条目；同一对账单内订单原币须一致；不做发货日期强约束。
- 行保存即校验对账数量 ≤ 该条目剩余可对账量（按默认单位折算比较）。
- 条目约束按类型不对称：**常规单**禁勾样品订单来源条目与零金额行；**赠送/样品单**不限来源。
- 草稿可改可删；已有行时对手不可再改（须先删行）。

### 常规单：客户确认 → 发票结单

- 状态机：草稿 → 客户已确认 → 已结单。
- 客户确认：至少一行；逐行复核剩余可对账量并累加已对账数量，确认后单据锁死。
- 撤回确认：未关联任何发票（含草稿发票）前可撤回，回滚已对账数量、退回草稿。
- 已结单由开票触发：开出发票在草稿上选关联对账单，审核校验（对账单=客户已确认、公司/对手一致、发票价税合计=对账单本币合计、一对一）后同一事务翻为已结单并过账冲回组（详见 [发票](./发票.md)）。
- 已结单无独立作废入口，纠错走发票侧：发票作废/红冲自动解除关联并把对账单退回客户已确认。

### 赠送/样品单：审核即结单

- 状态机：草稿 → 已结单 →（已作废）。
- 结单审核：至少一行；逐行复核剩余可对账量并累加已对账数量；按勾选条目价税合计（本币）过账 借单头借方科目（费用/损失）/ 贷单头贷方科目（未开票应收，分录带对手）。金额为零跳过分录，但科目仍必填。
- 过账日期：有金额结单时必填，未填默认结单当日。
- 已结单可作废：回滚总账分录组与已对账数量。

### 与发货交叉

- 任一条目存在非零已对账数量的发货单不可作废，须先撤回/作废相关对账单。
- 列表视图：菜单供应链销售组，双 tab（对账条目 / 对账单）。

## 业务流程

- 常规：发货已审核 → 建对账单勾条目（可部分数量）→ 客户线下认可后确认 → 开出发票关联该单审核（五笔分录 + 结单）→ 票错则发票作废/红冲，对账单自动退回已确认，可重新关联。
- 赠送/样品（含超发尾差）：发货已审核 → 建赠送/样品单勾条目 → 审核即结单过账 → 错则作废回滚。
