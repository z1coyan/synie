# 基础资料

## 概述

基础资料是全系统共享的主数据集合：公司、货币、计量单位，以及交易对手（供应商、客户）。它们不直接产生业务，而是被总账、发票、票据等模块的单据引用，是整个系统的数据地基。维护入口按菜单分散（公司归系统管理、客商归供应链），但口径上都是全局共享、不分公司的主数据；各类主数据的增删改查均按角色权限控制。本模块不覆盖科目体系（归「总账财务」模块）；市场参考价（沪铜等）见独立篇 [行情管理](./行情管理.md)（行情页：图 + 价点/品种维护）。**基础设置**（菜单入口）页内 Tab 承载基础域全局配置，当前仅「行情拉取」Tab（定时节奏存 `sys_setting`，细节见行情管理篇）。客商的开票信息（税号、地址电话、开户行）与编号自动生成功能在规划中。

## 核心概念

- **公司**：独立核算主体，全局共享；支持树形层级（上级公司），用于集团/合并视角。业务单据、科目、账簿都归属某个公司。每个公司必设**本币**（记账货币），是该主体单据双币换算的目标口径（见术语表「本币」）。
- **货币**：币种主数据，以 ISO 4217 三位大写编码标识，全局共享；有启用/停用状态（默认启用）。
- **计量单位**：按单位类型（长度/面积/重量/数量）分组的全局主数据，换算以基准单位为锚。
- **基准单位**：每个单位类型下换算比例恒为 1 的单位，每类型至多一个；同类型其余单位按换算比例折算到基准单位（如 kg 为基准时 g=0.001）。
- **对手**：业务往来对象的统称，三类——供应商、客户、内部公司；单据以「对手类型 + 对手」引用，应收应付报表按对手轧差（见术语表「应收应付报表」）。
- **客商**：供应商与客户的合称；两者分别维护、编号各自唯一，在对手体系中是两个并列类型。

## 功能清单

### 公司维护

- 新增、编辑、删除公司。
- 编号：两位英文字母，手工输入，全局唯一，创建后不可改；名称、简称、本币必填（存量公司已统一初始化为人民币）。
- 层级：可设上级公司形成树；上级不能选自身（多层成环暂未拦截，靠人工避免）。
- 删除：物理删除，不可恢复。已被业务数据引用（科目、凭证、银行账户、用户公司授权等）或有下级公司的，删除会被数据库外键拦截。

### 货币维护

- 新增、编辑、删除货币；启用/停用由列表行操作切换（不进创建/编辑表单），新建默认启用。
- ISO 编码：三位大写字母，全局唯一，创建后不可改；名称必填，符号（如 ¥）可选。
- **启停语义（拦新不拦旧）**：停用后不可再选作新单据币种或公司本币；历史单据与已挂引用不受影响。被任一家公司引用为本币的货币不可停用。
- 初始化向导预置约 20 种常用货币时默认全部停用；用户选定本币后仅本币启用，其余须在本页手动启用后才能用于外币单据等。
- 已被科目、分录、银行账户等引用的货币不能删除。
- 边界：无汇率管理，当前业务口径下金额不做折算（应收应付报表不区分币种，见 ADR）。

### 计量单位维护

- 新增、编辑、删除单位；单位全局共享，不分公司。
- 单位类型四种：长度、面积、重量、数量，创建时必选。
- 符号全局唯一（如 pcs、mm）；换算比例必须大于 0。
- 基准单位：每类型至多一个，其换算比例必须为 1；把另一单位设为基准前需先取消原基准，否则冲突报错。
- 单位的类型、基准标记、符号、换算比例均可修改。
- **初始化种子**（向导完成时幂等，按符号跳过已有）：面向机加工——长度基准 `mm`（另 μm/cm/m/in）、面积基准 `mm²`（另 cm²/m²）、数量基准 `pcs`/件（另只/个/套/台/片/根/支/块/张/箱/包/卷/捆/打/次/项）、重量补 `g`（行情迁移已有重量基准「吨」与千克，不抢基准）。箱/包等计数口径全局 ratio=1，物料级「一箱多少件」走物料单位转换。
- 边界：已被物料/单据引用的单位删除受业务约束拦截。

### 客商维护（供应商 / 客户）

- 供应商、客户分别维护，规则相同：编号手工填写、各自全局唯一（供应商编号与客户编号互不影响）、创建后可改；名称必填，简称可选。
- 编号自动生成功能暂未接入，目前全靠手工维护。
- 删除：物理删除且不做引用检查。客商一旦被凭证、发票、票据引用，删除后历史单据上的对手名称将无法正常显示（应收应付报表中按「未指定对手」兜底），删除前需自行确认无往来数据。

### 往来对手引用

- 总账分录（凭证行）、增值税发票、承兑票据交易等单据都带「对手类型 + 对手」；对手三类：供应商、客户、内部公司（直接引用公司主数据，用于集团内往来）。
- 对手类型与对手必须同时填或同时空；保存时按类型查主数据校验对手真实存在（多态引用，无数据库外键）。
- 挂了科目角色的科目（应收/应付/预收/预付等六角色），其过账分录必须填对手；历史存量无对手的数据在应收应付报表中归入「未指定对手」兜底行。
- 应收应付报表按对手 × 科目角色轧差，出各角色余额与净应收/净应付；口径细节见 ADR 与术语表。

## 业务流程

主数据之间无流转式业务流程，各自独立维护、即建即用。首次初始化建议顺序：先建公司，再建货币与计量单位，再建客商，最后到各业务模块建单引用（科目初始化见「总账财务」模块）。
