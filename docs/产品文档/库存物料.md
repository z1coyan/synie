# 库存物料

## 概述

库存物料模块管理企业的物料分类体系、物料主数据、仓库档案与库存数量账，为采购、库存等供应链业务提供统一的物料与仓储档案，服务全体项目成员。分类与物料均全局共享、不分公司；仓库与库存单据按公司维护。当前已实现物料分类、物料主数据（含单位转换、图纸附件）、仓库树、手工出入库单、手工调拨单、库存盘点单，以及库存分录流水与库存余额两个只读视图；库存金额账（估值）未实现，不在本文范围。

## 核心概念

- **物料分类**：物料的树形分类学，全局共享不分公司，层级深度不限。
- **叶子分类**：可挂物料的末级分类；叶子不能再有子分类。与科目的「汇总」语义相反，勿混。
- **分类编号**：人工定义、全局唯一的编码（如 01=原材料），作为物料编号前缀；可改，已生成的物料编号不追溯。
- **上级分类**：分类自引用构成树；不填即为根分类，允许多个根分类并列。
- **物料**：全局共享的物料主数据，只能挂叶子分类。
- **客户物料**：是否专属某一客户；是则必选一个客户（v1 只挂一个），否则为通用料。
- **物料编号**：物料唯一标识，仅按编号规则自动取号（seed：分类编号 + 可选客户编号 + `-` + 序号；通用料与客户料共用规则、按前缀分桶），不接受手填、创建后不可修改。
- **客户方产品编号**：客户侧料号文本，仅客户物料可填。
- **默认单位**：物料计量与单位转换的基准单位，必填、类型不限。
- **单位转换**：物料级换算事实，一行＝「1 默认单位 = x 该单位」（如 1kg = 518 只）。
- **图纸**：物料的图片附件槽位，仅图片、可多张；其他文件走通用附件。
- **启用/停用**：分类与物料均有启停标记，仅作状态记录，不联动其他约束。
- **仓库**：按公司的树形仓库主数据，名称公司内唯一，无仓库号；新建公司时自动种子三个默认仓库。
- **叶子仓库**：树语义同物料分类——上级只能选非叶子仓，新仓默认叶子；有下级的不能改叶子、不能删；只有叶子仓能发生库存。
- **仓库关联科目**：仓库指向本公司存货科目的可空引用，限本公司、非汇总、本币（未指定币种）科目，为将来出入库记账用；库存 v1 只记数量，此字段继续空转。
- **外协仓**：占位标记，本期纯记录、无业务联动。
- **允许负数库存**：仓库标记，默认否（最严方向）；打开后该仓跳过负库存校验。
- **手工出入库单**：仓管无上游单据直接录入的库存来源单据，入库/出库一体（方向字段区分、创建后不可改）；审核后按行增减库存（入正出负）。
- **手工调拨单**：同公司两仓间移货的单据，经在途仓中转——发货后货在在途仓，收货按行确认实收后入调入仓。
- **库存盘点单**：核对账实并校正库存的单据，单头一仓、行即要盘的物料清单（允许部分盘点）；审核按「实盘 − 账面快照」差异派生库存分录（盘盈正、盘亏负）。
- **库存分录**：库存领域唯一事实表，由来源单据审核时自动派生，只追加不可改；作废不删行、仅标记。
- **库存余额**：分录的只读聚合视图，按仓×物料汇总；在途仓作为普通仓自然呈现「在途库存」。

## 功能清单

### 分类新建与编辑

维护分类的编号、名称、是否叶子、启用状态与上级分类。

- 分类编号必填、全局唯一，重复时报「分类编号已存在」；编号不限格式，不要求继承父级前缀。
- 名称必填，不要求唯一。
- 新分类默认是叶子、默认启用；要建可挂子分类的中间分类，须手动把「叶子」关掉。
- 上级分类可空（即根分类）；不能选自身；只能选非叶子分类——叶子分类不能挂子分类。
- 编号与上级创建后均可改；分类编号拼入物料编号，改码不追溯已生成的物料编号。
- 已有子分类的分类不能改成叶子；分类下存在物料的不能改成非叶子。
- 边界：暂不检测多级成环（如 A→B→A），属已知留跟进项。

### 分类删除

- 有下级分类的不能删，须先移走或删除全部下级。
- 分类下存在物料的不能删，须先移走或删除物料。
- 删除即彻底删除，无作废/归档；操作留审计记录。

### 分类树浏览

- 入口：「供应链 → 物料 → 物料分类」页面。
- 树形展示、逐级懒加载展开，每层按分类编号升序。
- 全局共享：页面无公司选择器，所有公司看到同一棵分类树。
- 增删改查需具备物料分类的相应权限，操作接审计。

### 物料新建与编辑

维护物料的编号、名称、分类、默认单位、规格、是否客户物料、客户、客户方产品编号与启用状态。表单分两个 tab：「基本信息」（字段与图纸、附件）、「单位转换」。

- 编号按启用的编号规则自动取号（内置规则：分类编号 + 客户编号（非客户料省略）+ `-` + 4 位序号），表单中禁用不可手填、创建后不可修改；编号全局唯一。未配置启用规则时保存报错，需先在「系统管理 → 编号规则」配置。
- 名称、分类、默认单位必填；规格选填。
- **是否客户物料**默认否。是则必须选一个客户，并允许填写客户方产品编号；否则客户与客户方产品编号必须为空（改回通用料时清空）。
- 只能挂启用的叶子分类；改挂分类/客户不追溯改号（取号即固化）。
- 名称/规格不做组合防重，重复录入的防线是录入前先搜索。
- **销侧客户约束**：报价、销售订单（及未来发货）只能选用通用料或「客户 = 单据对手」的客户料；内部公司对手只能用通用料。采购/库存本期不校验。选料器按对手过滤。
- **客户字段锁**：物料一旦被报价条目或订单条目引用（含已作废单据上的行），「是否客户物料」与「客户」不可再改；无引用时可改。有物料挂靠的客户不可删除。
- 存在单位转换行时不能修改默认单位，须先删除全部转换行；存在库存分录（含已作废）后同样不可修改——分录只存数量，改单位等于篡改历史分录语义。

### 单位转换维护

在物料抽屉的「单位转换」tab 中维护「1 默认单位 = x 该单位」的物料级换算行，tab 顶部提示当前基准单位（随「基本信息」tab 的默认单位字段联动）。

- 单位从计量单位主数据中选择，类型不限（跨类型如 kg→只，同类型如箱/包等包装单位均可）。
- 同一物料同一单位只能有一行；单位不能选默认单位自身；系数必须大于 0。
- 与计量单位主数据里的同类型全局换算（如 kg↔g）互相独立，系统不做交叉校验。
- 转换行随物料表单一并保存（增删改在点「保存」时统一提交）；删除物料时转换行一并删除。

### 图纸与其他文件

- 物料详情有两个附件区：「图纸」仅收图片、可多张；「其他文件」不限类型。
- 图纸在列表页有缩略图列，点击全屏预览，可循环切换同物料全部图纸。
- 新建物料时即可上传附件（先暂存，保存成功后自动挂接；放弃保存则不可见）；删除物料不级联删附件（与全站附件行为一致）。

### 物料删除与停用

- 删除即彻底删除，转换行级联删除；操作留审计记录。被库存分录引用（含已作废分录）的物料不可删除——作废分录仍是历史引用。
- 停用仅作标记与列表筛选，暂不拦截业务引用（采购/销售单据接入后再挡新引用）；物料与分类的启停均通过列表行操作切换（不在编辑表单里），新建默认启用。

### 物料列表浏览

- 入口：「供应链 → 物料 → 物料管理」页面。
- 全局共享：页面无公司选择器，所有公司看到同一份物料档案。
- 编号、名称、分类、规格、是否客户物料、客户、客户方产品编号、默认单位、启用均可筛可排。
- 增删改查需具备物料的相应权限（单位转换跟随物料权限，无独立权限点），操作接审计。

### 仓库新建与编辑

页面顶部先选公司，再维护其仓库树：名称、上级仓库、是否叶子、关联科目、外协仓与允许负库存两个占位标记。

- 名称必填、公司内唯一，重复时报「仓库名称已存在」；创建后不可换公司。
- 新仓默认叶子、默认启用；要建归集节点（挂子仓）须手动关掉「叶子」。
- 上级仓库可空（多根并列）；只能选同公司的非叶子仓，不能选自身；暂不检测多级成环（同分类先例，留跟进）。
- 关联科目可空，限本公司、非汇总、本币（未指定币种）科目；多仓可共用一科目；被引用的科目不可删除。
- 有下级仓的不能改成叶子；有下级仓的不能删，删除即彻底删除，操作留审计记录。存在库存分录（含已作废）的仓不可删除、不能改为非叶子。
- 启停走列表行操作切换、不进表单（同物料先例），新建默认启用。停用**拦新不拦旧**：新单据用仓（手工出入库单头仓、调拨三仓）保存时必须启用，已发货调拨单的收货不再校验（在途必须能收尾）。

### 默认仓库种子

- 新建公司（初始化向导首个公司、公司管理页新增两个入口）成功后自动种子三个仓库：「{公司编号} - 所有仓库」（非叶子根）及其下「{公司编号} - 默认仓库」「{公司编号} - 在途」两个叶子仓。
- 幂等：公司已有任意仓库时不再补建；种子仓无特殊保护，可改可停可删（受常规约束）。
- 种子失败不阻断公司创建：界面报错提示，仓库可事后手工补建。

### 仓库树浏览

- 入口：「供应链 → 仓储 → 仓库管理」页面。
- 按公司查看：页面顶部公司选择器选定后加载该公司仓库树，逐级懒加载展开，每层按名称升序。
- 列示名称、上级、关联科目（编号-名称）、是否叶子、外协仓、负库存、启用状态。
- 增删改查需具备仓库的相应权限（种子初始化复用「新增」权限，无独立权限点），操作接审计；公司维度数据按授权隔离，未授权公司不可见。

### 手工出入库单

入口：「供应链 → 库存 → 手工出入库单」页面。入库/出库一体，方向新建时选定、创建后不可改。

- 页面顶部先选公司，再按公司列出单据；列示单据编号、方向（入库/出库，可筛）、业务日期、仓库、状态、摘要、审核时间。
- 头字段：方向、单据编号（留空按编号规则自动取号，也可手填）、业务日期（默认当天，可改即补录）、仓库（限本公司启用叶子仓）、摘要、备注；公司取页面顶部所选，创建后不可换。
- 行字段：物料、单位（限该物料默认单位或其转换单位，选物料后默认带默认单位）、数量、行备注；折算数量按物料默认单位口径由系统折算，只读不可手改。行随单保存一并提交，行保存即冻结物料/单位快照。
- 状态机：草稿 → 已审核 →（已作废）。仅草稿可改可删；审核要求至少一行，审核按行派生库存分录（入库为正、出库为负）；已审核只能作废，无反审核、无关闭态。
- 作废同样过负库存校验（作废入库单会减库存，致负则拒绝作废）；纠错＝作废＋重开，不做红冲。
- 负库存校验（审核/作废同一口径）：按仓×物料「现有余额 + 本单变动 ≥ 0」，任一行不过整单拒；标记「允许负数库存」的仓跳过；草稿不校验。
- 摘要带入库存分录，供流水查看；期初建账走入库方向的手工出入库单，不开专用通道。

### 手工调拨单

入口：「供应链 → 库存 → 手工调拨单」页面。同公司调出/调入两仓间移货，经在途仓中转，一单两动作。

- 头三仓：调出、调入、在途，均限本公司启用叶子仓且两两不同；在途仓默认预填该公司种子在途仓（「{公司编号} - 在途」），可改。
- 行字段同手工出入库单（物料/单位/数量/行备注，折算数量系统算）。
- 状态机：草稿 → 已发货 → 已收货（终态）。仅草稿可改可删；已发货不可改不可删、无作废（路上有货不能当单据不存在，纠错走反向调拨）。
- 发货（行操作）：写「调出仓负 + 在途仓正」分录，要求至少一行、三仓启用。
- 收货（行操作，已发货单可用）：弹窗逐行确认实收数量，默认等于已发数量（折算口径），可改小（0 ~ 已发）不可改大；确认后按实收写「在途仓负 + 调入仓正」分录并回写行实收数量；差额留在在途仓，后续用手工出入库单（出库）清理。一次收讫不分批；收货不校验仓启用。
- 限同公司调拨；跨公司调货是买卖，走销售/采购线。

### 库存盘点单

入口：「供应链 → 库存 → 库存盘点单」页面。核对账实并校正库存，一单一仓，允许部分盘点。

- 页面顶部先选公司，再按公司列出单据；列示单据编号、业务日期、仓库、状态、摘要、审核时间。
- 头字段：单据编号（留空按编号规则自动取号，也可手填）、业务日期（默认当天，可改即补录）、仓库（限本公司启用叶子仓）、摘要、备注；公司取页面顶部所选，创建后不可换。
- 行＝本次要盘的物料清单，两种建行方式：「整仓带出」自动列出该仓账面余额非零的全部物料行（可再删行）；或空白手工逐行加（账面为零的物料要盘只能手工加，典型如实物有、账面无的盘盈）。
- 行字段：物料、单位（限该物料默认单位或其转换单位）、实盘数量、行备注；账面数量＝取数时刻的库存余额快照（恒为物料默认单位口径），差异＝实盘 − 账面由系统算；折算口径同手工出入库单行先例。行保存即冻结物料/单位快照。
- 「刷新账面数」（草稿可用）：按最新余额重取全部行的账面数量，已填实盘数保留。
- 状态机：草稿 → 已审核 →（已作废）。仅草稿可改可删；审核要求至少一行、逐行实盘数已填且 ≥ 0（有空行整单拒，补数或删行）；差异非零的行派生一条库存分录（盘盈为正、盘亏为负），零行不落；已审核只能作废，无反审核、无关闭态。
- 审核兜底校验：取快照后该仓库存分录有新增或作废（含业务日期在过去的补录单）则审核拒绝，需刷新账面数后重审。盘点不做冻结，取快照到审核之间该仓尽量不动货。
- 作废照常过负库存校验（撤销盘盈会减库存）；盘点差异分录本身永不致负（审核时点余额恒等于快照、调整后恒等于实盘）。
- 作业纪律：取快照前先确认该仓单据已录齐（审核后才补录的历史单会让账实再次不符，只能再盘或手工出入库调整）；盘点前先收尾相关在途调拨（货已物理到仓未收货会盘出假盘盈）。同一仓可同时开多张盘点单（兜底校验把审核串行化，后到者刷新即可）；在途仓不禁止盘点。

### 库存余额

入口：「供应链 → 库存 → 库存余额」页面。

- 页面顶部先选公司；筛选：截至日期（默认今天）、仓库（限本公司叶子仓）、物料、「显示零余额行」开关（默认隐藏零行）。
- 结果按仓×物料一行，列示仓名、物料编号/名称/规格、单位（物料默认单位）、数量；在途仓作为普通仓自然呈现在途库存。
- 口径为库存分录聚合：未作废、业务日期不晚于截至日；只查分录不查单据。

### 库存分录流水

入口：「供应链 → 库存 → 库存分录」页面，只读。

- 页面顶部先选公司；列示业务日期、仓库、物料、数量（入正出负）、来源单据（链接，点开速览来源手工出入库单/手工调拨单）、来源单号、摘要、是否已作废。
- 各列支持常规筛选与排序；分录由来源单据审核时自动派生，用户无写入口；作废来源单据不删分录行，仅打「已作废」标记。

## 业务流程

建料动线：先在物料分类中建好叶子分类 → 新建物料（选叶子分类与默认单位，编号自动取号）→ 切到「单位转换」tab 录入换算行 → 保存后上传图纸/其他文件（也可建料时一并暂存上传）。调整归属通过编辑「物料分类」完成，编号不变。

建仓动线：新建公司即自动种子三个默认仓库；增补时在仓库管理页先选公司 → 新建仓库（默认叶子，建归集节点先关掉「叶子」再挂子仓）→ 需要存货核算时在「关联科目」挂本公司存货科目。仓库启停走列表行操作。

出入库动线：库存页选公司 → 新建手工出入库单（先定方向，编号留空自动取号，录头选仓）→ 逐行录物料、单位、数量 → 保存草稿 → 审核（派生分录、即时增减库存，负库存整单拒）→ 出错在已审核前改删，已审核后作废重开。

调拨动线：新建手工调拨单（选调出/调入仓，在途仓自动预填）→ 录行 → 发货（货离开调出仓、进入在途仓）→ 到货后收货（逐行确认实收，差额留在在途仓）→ 在途差额用手工出入库单（出库）清理。库存余额页随时按仓（含在途仓）查看数量。

盘点动线：确认该仓单据已录齐、相关在途调拨已收尾 → 新建库存盘点单（选仓，整仓带出或手工建行）→ 实地清点并逐行填实盘数 → 审核（快照后该仓有分录变动会被拒，刷新账面数后重审）→ 差异自动派生分录，库存余额即时校正；出错在已审核前改删，已审核后作废重开。
