# 生产BOM

## 概述

生产 BOM 模块承载制造侧的配方主数据：为物料建立单层 BOM——一张 BOM 由配料、工艺路线、副产品三块内容组成，并配套工序与工艺模板两个全局基础主数据，服务生产与工艺管理人员，是将来生产执行层（工单扣料、报工、完工入库、成本归集）取配方的锚点。BOM 跟随物料全局共享、不分公司。

边界：本期只做到配方主数据——生产工单执行层（扣料/报工/入库/成本归集）、外协加工单据流（发料/入库/加工费）、BOM 版本管理均不在本期范围；工艺路线只描述「怎么做」，不驱动执行、不派生任何库存或财务后果。

## 核心概念

- **BOM**（物料清单）：挂在物料上的单层配方主数据；一物料至多一张、以物料为唯一键、无独立编号。多级结构靠半成品物料各自挂 BOM 嵌套形成，系统不建 BOM 树实体。见 [术语表](../../CONTEXT.md)。
- **配料行**：BOM 的子物料行——子物料 + 单位 + 单位净用量（每 1 默认单位母物料的净消耗）+ 可空损耗率；平铺清单，不记录投料工序。
- **损耗率**：配料行上的可空比率，空即无损耗；理论耗用 = 净用量 ×（1 + 损耗率）。
- **工艺路线**：挂在 BOM 上的工序子表（每 BOM 私有），行 = 工序引用 + 顺序 + 工艺要求文本 + 外协标记，只描述工序顺序与要求。
- **外协标记**：工艺路线行上的布尔标记，表示该工序通常外发加工；不指定供应商、不产生任何单据。
- **副产品行**：BOM 上声明的联产出行（如铜屑、网屑）——物料 + 单位 + 单位产出量（每 1 默认单位母物料）；本期仅作声明。
- **工序**：全局共享的工序主数据（如冲网、分切、焊接成筒、CNC），被路线行与模板行引用。
- **工艺模板**：独立可维护的工艺路线模板主数据；建 BOM 路线时选模板复制带入，此后模板再改不影响已建 BOM（快照语义）。

## 功能清单

### BOM 新建与编辑

- 在物料上建 BOM：选择物料 + 备注，保存即建档；一物料至多一张，重复建报「该物料已存在 BOM」。
- BOM 无独立编号，以物料为唯一键；跟随物料全局共享，不分公司、无公司选择器。
- 创建后不可换物料——换物料 = 删旧 BOM 建新 BOM。
- 单份可改可删，无版本管理：换配方直接改，历史靠审计日志追溯（版本管理待有下游单据引用 BOM 时再议，见 ADR）。
- 删除 BOM 时其配料行、工艺路线行、副产品行一并级联删除；已建有 BOM 的物料不可删除（须先删 BOM）。
- 增删改查需具备 BOM（`mfg.bom`）的相应权限，操作接审计。

### 配料行维护

BOM 的配料清单，逐行维护，行 = 子物料 + 单位 + 单位净用量 + 损耗率 + 行备注。

- 单位净用量必填且大于 0，语义钉死为「每 1 默认单位母物料的净消耗」。
- 损耗率可空、不为负；空即无损耗，理论耗用 = 净用量 ×（1 + 损耗率）。
- 行单位限该子物料的默认单位或其单位转换单位（同销/采单据行单位约束）。
- 子物料不能是 BOM 母物料自身（保存校验）；多级间接成环（A 配料 B、B 配料 A）不硬校验，留待树展开视图检测截断。
- 平铺清单，不记录投料工序（投料时点待执行层需要时再补列）。
- 行维护视为 BOM 维护的一部分，不设独立权限点：读随 BOM 读权限，增删改随 BOM 编辑权限（同物料单位转换先例）。

### 工艺路线维护

BOM 的工序子表，每 BOM 私有，逐行添加，行 = 工序 + 顺序 + 工艺要求文本 + 外协标记。

- 工序从工序主数据中选择；工艺参数与要求走行内自由文本，不在工序主数据上结构化。
- 外协标记默认否；为是仅表示该工序通常外发加工，不指定供应商、不产生任何外协单据。
- 路线不带生产数量、不驱动执行、不派生库存/财务后果。
- **从工艺模板带入**：本 BOM 尚无路线行时，可选一份工艺模板，按其行顺序复制带入为本 BOM 私有路线行（已有路线行则拒绝带入，避免混入两套路线）；复制后与模板脱钩——模板日后再改不影响已建 BOM（快照语义，同系统「改默认不追溯」惯例）。带入动作复用 BOM 编辑权限，不设独立权限点。
- 行权限同配料行：读随 BOM 读、增删改随 BOM 编辑。

### 副产品行维护

BOM 上声明的联产出清单，行 = 物料 + 单位 + 单位产出量 + 行备注。

- 单位产出量必填且大于 0，语义为「每 1 默认单位母物料的联产出量」。
- 行单位限该物料默认单位或其转换单位；副产品物料不能是母物料自身。
- 本期仅作声明，不产生入库或成本后果，是将来执行层扣料/完工入库/成本分摊的锚点。
- 行权限同配料行。

### 工序维护

全局共享的工序主数据，字段 = 编号 + 名称 + 备注。

- 编号必填、全局唯一；留空按编号规则自动取号（初始化预置：`M(O)-` + 4 位补零序号，如 `M(O)-0001`），也可手填；创建后不可修改。
- 名称必填，不强制唯一。
- 被任一 BOM 工艺路线行或工艺模板行引用的工序不可删除，须先移除相关行。
- 增删改查需具备工序（`mfg.operation`）的相应权限，操作接审计。

### 工艺模板维护

全局共享的工艺路线模板主数据，头 = 编号 + 名称 + 备注，行 = 工序 + 顺序 + 工艺要求文本 + 外协标记（结构同 BOM 工艺路线行）。

- 编号必填、全局唯一；留空按编号规则自动取号（初始化预置：`M(T)-` + 4 位补零序号，如 `M(T)-0001`），也可手填；创建后不可修改。
- 模板可改可删，删除时模板行级联删除；模板的改删均不影响已从它带入路线的 BOM（快照语义）。
- 模板行维护不设独立权限点：读随模板读权限、增删改随模板编辑权限。
- 增删改查需具备工艺模板（`mfg.route_template`）的相应权限，操作接审计。

### 权限与菜单

- 权限资源三个：BOM（`mfg.bom`）、工序（`mfg.operation`）、工艺模板（`mfg.route_template`），均为标准增删改查动作；三类 BOM 行与模板行不设独立权限点（口径见上文各节）。
- 菜单入口在「生产」组：BOM、工序、工艺模板三个入口。

## 业务流程

建配方动线：先在工序主数据中维护好所需工序 → （可选）把常用工序组合维护成工艺模板 → 在目标物料上建 BOM → 逐行录入配料行（子物料、单位、净用量、损耗率）→ 从工艺模板复制带入工艺路线（或逐行手工建行、按需标外协）→ 如有联产出再录副产品行。日后换配方直接改 BOM，历史由审计日志追溯。
