# 资金银行

## 概述

资金银行模块管理公司自有银行账户、银行流水台账，以及流水与会计凭证的对账勾稽，服务财务人员的资金档案与银企核对工作：账户是主数据，流水是银行对账单的电子档案（数据以银行为准，不推算、不校验连续性），导入是流水的主要录入路径，对账把流水与已审核凭证按金额勾稽起来。银行流水本身不记账、不产生总账分录；付款审批、资金计划不在本模块，承兑票据的收付虽也使用银行账户但归票据模块；对账的自动匹配建议、批量对账、对账报表尚在规划中。

## 核心概念

- **银行账户**：公司在银行开立的自有资金账户主数据，别名与账号同公司内唯一；可绑定一个会计科目（对账的前提）；退出方式是停用而非删除。流水上的「对方户名/对方账号」只是文本信息，对手方不是主数据。
- **银行流水**：一行对应银行对账单的一行（交易时间、收入/支出、余额快照、对方信息、摘要），数据以银行为准；不参与记账。
- **导入模板**：描述某银行账户导出文件列布局的解析配置（各字段所在列、日期/金额格式、数据起始行）；一个账户可配多份模板。
- **流水导入批次**：一次上传解析的留痕单据，状态为已解析/解析失败/已导入；批次下挂解析出的暂存行，可逐行修正后转正为流水。
- **对账记录**：流水与已审核凭证之间的 m-n 金额勾稽；同一对流水-凭证仅一条记录。
- **对账状态**：流水的派生状态——未对账/部分对账/已对账，按已对账金额合计与流水金额自动推出。

## 功能清单

### 银行账户管理

登记与维护公司的自有资金账户（别名、所属银行、开户支行、户名、账号、币种、绑定科目、启用、备注），可挂开户资料附件。

- 别名与银行账号均同公司内唯一，跨公司可重复；所属公司建后不可改。
- 币种必填；流水的币种随账户。
- 绑定科目选填，填了必须满足：同公司、非汇总、启用；科目指定了币种时须与账户币种一致。
- 名下流水一旦存在对账记录，不允许改绑或解绑科目，须先解除相关对账。
- 停用不删：停用账户不能录新流水、不能发起新导入，存量流水与模板仍可维护；名下已有数据的账户删除会被拒绝，不再使用的账户应停用。

### 银行流水台账

手工登记与维护银行流水，是导入之外的兜底路径；流水可挂附件（汇款凭证截图等）。

- 每行记录交易时间（精确到秒）、收入、支出、余额、对方户名、对方账号、摘要、备注。
- 金额互斥：收入与支出恰填一项且必须大于零；不允许负数，冲正、转出退款等按实际资金方向录入（如转出退回记为收入行）。
- 流水必须挂在同公司账户下；新增时要求账户处于启用状态，修改时不强制（允许更正错录的归属账户）。
- 余额是银行口径快照，只存档不推算。
- 已有对账的流水受保护：不可删除、不可更换账户、不可收支换边，修改金额不得低于已对账金额；金额变化后未对账金额与对账状态自动重算。

### 导入模板管理

为银行账户配置导出文件的解析模板，独立页面、独立权限。

- 一个账户可建多份模板（同一银行不同导出渠道格式不同）；模板名同公司内唯一。
- 列位置用列号字母表示（A–Z、AA–ZZ），录入自动归一为大写。
- 时间配置二选一：日期时间单列（列+格式），或日期+时间双列（日期列与格式必填，时间列可省、缺省按 00:00:00）；两种模式不可混填。
- 金额配置二选一：收入/支出双列（至少配一列），或带符号金额单列（正数=收入、负数=支出，导入时按符号拆列）；两种模式互斥。
- 日期/时间格式从预设清单选择（覆盖常见的年月日、时分秒组合），不支持自由文本。
- 可配置数据起始行（默认第 2 行，跳过表头），以及余额、对方户名、对方账号、摘要、备注等可选列；未配置的列导入时对应字段留空。

### 流水导入

在银行流水页发起：上传银行导出文件，按模板解析成暂存行，确认修正后批量转正为流水。支持 xlsx 与 xls 文件；网页或文本改名的「假 xls」会被拒绝并提示另存。

- 一次导入绑定公司、账户（须启用）、该账户下的一份模板与一个上传文件；提交即解析，结果只有两种：已解析（行落暂存）或解析失败（整批报错留痕，导入历史中可见原因）。
- 防重传：同账户已存在内容相同的非失败导入记录时拒绝再次上传；确需重导先删除原记录。
- 解析规则：Excel 原生日期单元格优先于模板格式，文本单元格才按所选格式解析；文件时间按本地时间理解、统一换算存储（默认按 UTC+8）；金额自动剥离千分位与货币符号；双列模式下 0 或空视为未填、负数报行错，带符号单列按正负拆收支、0 报行错；所配列全空的行静默跳过；字段超长或解析失败只标记该行错误，不阻塞其他行；整批无数据行或超过 5000 行则整批失败。
- 已解析批次可逐行编辑或删除（编辑校验与流水一致：交易时间必填、收/支恰一项大于零，保存通过即清除该行错误）；批次的账户、模板、文件不可改，配错了删除重来。
- 执行导入要求批次内至少一行且全部行无错误；逐行转正为银行流水并回填两者关联，任一行失败整批回滚；执行人除导入权限外还需流水的新增权限。
- 已导入批次只读且不可删除；已解析、解析失败的批次可删除，暂存行随之清除。

### 流水对账

在流水行的对账抽屉中，把流水与已审核凭证按金额勾稽：一笔流水可分次对多张凭证，一张凭证也可对多笔流水。

- 关联已有凭证：凭证须已审核且与流水同公司；流水所属账户必须已绑定科目；方向必须匹配——收入流水要求凭证含该银行科目的借方行，支出流水要求贷方行。对账金额须大于零，且同时不超过流水未对账余额与凭证在该科目该方向上的剩余额度（凭证侧额度按科目×方向独立计算，内部转账凭证的两侧互不挤占）。选中凭证后可查询本组合的剩余可对账额度，用于预填金额。
- 快速新增凭证：在对账抽屉内一步创建一张两行分录的凭证（收入流水：借银行科目、贷对方科目；支出反向），自动审核过账并建立对账；对方科目不能是账户绑定的银行科目；金额默认流水未对账余额、摘要默认取流水摘要、日期默认流水交易日，均可改；操作人还需凭证的新增与审核权限，任一步失败整体回滚不留残单。
- 同一对流水-凭证只存在一条对账记录，调整金额须先解除再重新关联。
- 解除对账即删除对账记录；每次关联或解除后，流水的已对账金额、未对账金额与对账状态自动重算，流水列表以三态标识（未对账/部分对账/已对账）并展示未对账金额。
- 反向约束：存在对账关联的凭证不能取消，须先解除对账；存在对账的流水不可删、不可换账户或换边、金额不得低于已对账额。

## 业务流程

两条主线：

1. **流水入户**：建银行账户（并绑定科目）→ 配导入模板 → 上传文件发起导入 → 解析（已解析/解析失败）→ 修正或删除问题行 → 执行导入，暂存行批量转正为流水；零星流水可手工补录。
2. **对账勾稽**：流水到位后，在对账抽屉中关联已审核凭证（或快速新增凭证自动审核），对账状态从未对账经部分对账走向已对账；此后凭证取消、流水改删均受对账关联约束，须先解除。
