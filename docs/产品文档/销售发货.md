# 销售发货

## 概述

销售发货模块承载对客户/内部公司的**履约出库**：从已审核销售订单条目分批发货，审核后扣减库存、累加订单已发数量，有金额时按头上借贷科目确认未开票应收。服务销售内勤与仓管。

边界：不做存货成本/销项税拆分（税在发票、成本待采购估值）；内部公司对手只记卖方账，不自动镜像买方入库；无反审核/红冲；发货与发票勾稽未实现。

## 核心概念

- **销售发货单**：履约出库单据，状态机 草稿→已审核→（已作废）。见 [术语表](../glossary.md)。
- **发货条目**：必挂订单条目；行仓必填；数量折默认单位；行保存冻物料与订单条目快照（含图纸挂接复制）。
- **已发数量**：订单条目上默认单位累计，审核加、作废减。
- **发货超发比例**：供应链设置全局容差，默认 0%（禁超发）；入口「供应链 → 设置 → 供应链设置」。
- **物料快照**：行保存时冻结编号/名称/规格/客户料号/单位名，并把物料当前图纸（`drawing` 槽位）挂接复制到行；审核锁行即冻结，主数据后续变更不回溯。图纸为挂接复制（共享不可变文件，非字节复制），行/发货单删除时挂接一并清理。口径与订单条目快照一致，见 [术语表](../glossary.md)。

## 功能清单

### 列表视图

- 双 tab：**发货条目**（默认）与**发货单**；共用发货单抽屉。
- **发货条目** tab 展示行快照口径的物料信息，另有图纸列（缩略图，点开可预览该行全部图纸）。
- 菜单：供应链 → 库存 → 销售发货。

### 发货录入

- 头：公司（建后不可换）、发货单号（可留空取号）、发货日期、过账日期、对手（客户/内部公司）、默认仓库（可空，新建行预填）、借方科目（未开票应收角色）、贷方科目、备注。
- 行：订单条目（须已审核订单）、数量、单位（可空=订单单位）、出库仓库（必填）、行备注；保存时同步冻结物料快照与图纸挂接。
- 可跨多张订单，但公司、对手、原币须一致。

### 审核

- 至少一行；行仓须本公司启用叶子仓。
- 超发校验：累计已发 ≤ 订购 base × (1 + 超发比例)。
- 写负向库存分录；有金额时写总账两行（借未开票应收、贷选定科目）；累加已发数量。
- 整单金额为零：跳过总账，科目可空。

### 作废

- 仅已审核可作废：库存/总账分录作废，回滚已发数量（过负库存校验）。

### 与订单交叉

- 关闭订单后不可再发；有已审核发货时订单不可作废；草稿发货须先删再作废订单。

## 业务流程

录入草稿 → 审核算库存与应收 → 纠错作废重开。订单部分发完可关闭剩余。
