# 产品文档

Synie ERP 的功能说明书：面向项目成员与 AI 协作者，回答「每个模块是什么、能做什么、业务规则与边界是什么」。

- **写什么**：用户可感知的功能、业务规则、边界约束；只写代码已实现的行为。
- **不写什么**：实现细节（表结构、字段、权限码、代码路径）、未实现的规划（至多在概述一句话交代边界）、纯技术内容（前端组件、工程基建）。
- **内容基准**：以代码当前行为为准；历史设计文档只作线索。
- **设计理由**：不展开，链接到 `docs/adr/` 对应决策记录。

## 单篇模板

每篇固定五节：

1. **概述** — 一段话：模块干什么、服务谁；末尾一句话交代边界（不覆盖什么 / 什么在规划中）。
2. **核心概念** — 关键概念及相互关系，一句话级，与术语表对齐。
3. **功能清单** — 每功能点一小节：做什么、业务规则（校验 / 状态流转 / 约束）、边界。
4. **业务流程** — 跨功能的串联流程；无则一句话说明。
5. **相关文档** — 链接到相关 ADR / 术语表，只列真实存在的文件。

## 与其他文档的关系

- **术语表**（[../../CONTEXT.md](../../CONTEXT.md)）：领域术语的唯一定义处。产品文档引用术语但不复制定义；新术语随时补入。
- **决策记录**（`docs/adr/`）：难以逆转、需要上下文才能理解的决策。
- **活跃规格与工单**（`.scratch/`）：进行中的 spec 与 implementation tickets（本地 issue tracker）。

## 维护约定

交付新功能或变更业务规则时，同步更新本目录对应篇及根目录 `CONTEXT.md`（已写入根 `AGENTS.md`）。
