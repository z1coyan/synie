# 票据

## 概述

票据模块管理公司**收到**的承兑汇票（银行承兑 / 商业承兑 / 财务公司承兑），服务财务人员：从收票建档、段级持有库存，到转让（背书）、贴现、到期兑付、同公司跨账户调拨的全生命周期；交易审核时自动生成 GL 凭证并过账。边界：只做应收承兑；应付承兑（自己开出汇票）、商票拒付、质押、批量托收、到期提醒等均未实现，不在本文范围。

## 核心概念

- **承兑票据（票据主档）**：一票一条的票面档案，票号全局唯一，跨公司共享——内部公司间流转后仍是同一张票。建档随接收交易完成，无独立新建入口。
- **子票段**：参照新一代票据系统的等分化票据包，1 元 = 100 子票；交易与持有都精确到 `[子票起, 子票止]` 区间，段金额 =（子票止 − 子票起 + 1）÷ 100。
- **承兑交易**：接收 / 转让 / 兑付 / 贴现 / 调拨五类单据，走「草稿 → 已审核 → 已作废」生命周期，是 GL 的凭证来源单据。
- **持有承兑**：按「公司—银行账户—子票段」的库存只读快照，由已审核交易自动推导，无手工维护入口。
- **库存重放**：以票为单位，把全部已审核交易按（发生日期，审核时间）重放、逐笔校验段合法性并整体重建持有段的机制，是持有数据的唯一事实来源。
- **交易对手**：多态往来对象（供应商 / 客户 / 内部公司），仅接收与转让必填，保证应收应付冲抵正确。

## 功能清单

### 票据建档与票面修正

- 建档唯一入口是接收交易：录入时按票号查档，已建档则自动挂接（票面以库内先录为准），未建档则顺带注册。
- 票号必填、全局唯一、建后不可改；不强校验位数（兼容新一代 30 位票号与存量票号）。
- 票据种类必填，三选一：银行承兑汇票 / 商业承兑汇票 / 财务公司承兑汇票。
- 到期日必填（日期校验依赖）；票据包金额可空，有值时作为子票段的上限。
- 出票人、收款人、承兑人各四项（名称、账号、开户行、行号）为文本快照，均可空；另有承兑日期、备注。
- 「能否转让」默认可转让；票面标记不得转让的票据禁止转让与贴现，兑付与调拨不受限。
- 票面修正：票据一旦存在任何交易（含草稿），到期日、票据包金额、能否转让锁死不可改；其余票面字段随时可改。修正在持有承兑页对持有段行内发起。
- 删除：仅当票据无任何交易时可删（即接收草稿删完后遗留的空档案）。
- 可见性：普通用户只能看到「自己可及公司名下有过交易」的票据；超级管理员与全公司权限用户可见全部。

### 承兑交易（五类型）

公共规则：

- 状态流转：草稿可改可删；审核后不可改，纠错走作废；作废为终态。交易类型与所属公司创建后不可改。
- 发生日期必填，允许补录历史；单据编号留空自动取号。
- 每笔交易作用于一个子票段：金额必须大于零，且满足勾稽「子票止 = 子票起 + 金额 × 100 − 1」；票据包金额有值时段不得越出票面范围。
- 本方银行账户必须属于本公司且启用（调拨时该账户为转出账户）。
- 审核：除调拨外必须填写过账日期，并选好票据科目与结算科目（贴现利息大于零时还必须选利息科目）；科目须同公司、启用、非汇总。审核即过账，借贷方向由类型自动决定，用户不指定方向。
- 作废不删分录，而是把原分录标记为已取消（报表按未作废口径统计），同时回滚持有库存。
- 作废有条件：该段已被后续交易动过（被转让 / 兑付 / 贴现 / 调拨消耗）则作废被拒，须先作废后续交易。
- 重放按（发生日期，审核时间）定序：倒填日期的交易插入历史链重新校验，天然拦下「先转后收」类时序错误；同日多笔按审核先后定序。

各类型特定规则：

- **接收**：交易对手必填；发生日期不得晚于到期日；接收段不得与全系统（任何公司任何账户）现有持有段重叠——同一子票段只能被一方持有。
- **转让（背书）**：交易对手必填；发生日期不得晚于到期日；票面「不得转让」禁止；转出段必须是本公司本账户当时持有的，可横跨多个相邻持有段，部分转让自动拆段（余段保留原取得日期与来源）。
- **兑付（到期托收）**：发生日期不得早于到期日（到期日起才能提示付款）。已知边界：兑付后同一段可被再次录入接收，系统暂不拦截，依赖录入纪律（随商票拒付一并建模）。
- **贴现**：发生日期不得晚于到期日且票面须可转让；必填贴现机构（文本，前端预置常见银行下拉可自由输入）、年化利率（≥ 0）、利息（≥ 0）、实收金额（> 0），勾稽「金额 = 利息 + 实收」；前端按「金额 × 利率 × 天数 ÷ 360」自动算利息与实收，均可手改（以银行实扣为准）。
- **调拨**：同公司跨银行账户的段迁移；必须选转入账户（同公司、启用、不同于转出账户）；无日期约束、无交易对手、不生成凭证（科目不变，仅存放账户变化）。

内部公司间转让是两笔单据：转出方录转让（对手 = 对方内部公司），接收方录接收（对手 = 本方内部公司）；同日发生时须先审转让、后审接收，否则接收因段仍在转出方名下重叠被拒。

### OCR 识票（录入辅助）

- 接收录入时可上传票面图片，OCR 识别后回填票面字段与子票段（票号、种类、金额、日期、出票人 / 收款人 / 承兑人信息等），识别结果仅作草稿预填，不落库、可手改；识别后按票号自动查档，命中即挂接已有档案。
- 复用承兑交易的新建权限，不另设权限点。

### 持有承兑（段级库存）

- 只读视图：按「公司—银行账户—子票段」展示当前持有，含持有金额、到期日、取得日期与来源交易；不支持任何手工增删改，也不接审计（交易本身即审计线索）。
- 自动推导：接收在对应（公司，账户）加段；转让 / 兑付 / 贴现消耗段；调拨把段从转出账户迁到转入账户（取得日期记调拨发生日）。
- 取得日期 = 产生该段的交易发生日期；部分消耗拆出的余段保留原取得日期与来源交易。
- 持有页是后续交易的发起入口：对某段发起转让 / 兑付 / 贴现 / 调拨，整段预填，可改小子票起与金额做部分交易（合法性由后端重放兜底）。

### 审核记账（GL 凭证）

按单据上选定的科目自动生成配平分录，借贷方向由类型决定：

- **接收**：借 票据科目（全额）／ 贷 结算科目（应收账款等往来科目，带交易对手）。
- **转让**：借 结算科目（应付账款等往来科目，带交易对手）／ 贷 票据科目。
- **兑付**：借 结算科目（银行存款）／ 贷 票据科目。
- **贴现**：借 结算科目（实收金额）＋ 借 利息科目（贴现利息，利息为 0 时该行省略）／ 贷 票据科目（全额）。
- **调拨**：不生成凭证。

分录全部为正数；GL 分录可回链到来源承兑交易。

## 业务流程

主流程：收票（接收交易，票号未建档顺带注册，可用 OCR 辅助录入）→ 审核（生成凭证过账 + 段入持有库存）→ 持有期间可部分转让、贴现或同公司调拨（均从持有段发起，审核后扣减 / 迁移段）→ 到期兑付（审核后段出清）。全程段级库存由重放机制维护；已审核交易的纠错走有条件作废，作废后凭证标记取消、持有段一并回滚。

页面形态：财务「承兑汇票」单页两 tab——「承兑交易」（列表 + 接收录入 + 审核 / 作废）与「持有承兑」（段级库存 + 发起后续交易 / 票面修正）。
