# 销售订单

## 概述

销售订单模块承载公司与客户之间的订货承诺：录入客户要买的物料、数量与含税价格，审核后单据生效锁死，作为将来发货与开票的源头依据。服务销售内勤与跟单人员。

边界：订单是纯业务承诺——审核不派生任何会计或库存后果；发货、开票对接与行级履行跟踪（已发数量等）均未实现，待下游模块落地。

## 核心概念

- **销售订单**：公司向客户承诺供货的订货单据，状态机 草稿→已审核→（已关闭/已作废）。见 [术语表](../glossary.md)。
- **订单条目**：订单的商品行，记录物料、单位、数量、含税单价、税率；含税金额由系统按 数量×含税单价 计算。
- **对手**：订单的客户方，取客户主数据或内部公司（多态引用）。
- **含税总额**：订单全部条目含税金额的合计，随行自动算出，只读。

## 功能清单

### 订单录入

- 头：订单号、订单日期（默认当天）、公司、对手、交易条款（对客户，自由多行文本）、订单备注（对内）。
- 订单号留空按编号规则自动取号，手填原样保留；订单号全局唯一。
- 对手限客户/内部公司两类；选内部公司时不能是订单所属公司自身。
- 条目行：物料、单位、数量、含税单价、税率、行备注；单位只能选该物料的默认单位或其单位转换单位，选物料后默认带默认单位。
- 条目约束：数量必须大于零；含税单价可为零（赠品/样品）；税率必填，界面按百分比录入（默认 13%，存储为小数 0.13）；含税金额=数量×含税单价（两位小数）由系统计算，不可手改。

### 审核

- 仅草稿单可审核；审核前必须至少有一行条目（其余字段约束在录入期已强制）。
- 审核后单据锁死：头不可改、行不可增删改、单不可删；无反审核。

### 关闭与作废

- 仅已审核单可关闭或作废，两者都是终态、不可逆。
- 作废：单据不该存在（录入错误、未履行即取消）。
- 关闭：单据生效过但提前终止或执行完毕，人工标记不再履行。

### 删除

- 仅草稿单可删除，条目随单一并删除。

## 业务流程

录入（草稿，可反复改）→ 审核（生效锁死）→ 执行完毕或提前终止时点「关闭」；发现单据错误则在审核后直接「作废」，草稿阶段直接删除。变更已审核单没有改单动线，作废/关闭后重新开单。
