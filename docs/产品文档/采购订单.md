# 采购订单

## 概述

采购订单模块承载公司与供应商之间的订货承诺：录入公司要买的物料、数量与含税价格，审核后单据生效锁死，作为将来入库与收票的源头依据。服务采购内勤与跟单人员。

边界：订单是纯业务承诺——审核不派生任何会计或库存后果。实物入库、未开票应付过账与订单已收数量由**采购入库单**承载（见 [采购入库](./采购入库.md)）；进项发票对接仍未实现。

## 核心概念

- **采购订单**：公司向供应商承诺采购的订货单据，状态机 草稿→已审核→（已关闭/已作废）。见 [术语表](../glossary.md)。
- **订单类型**：订单分**常规订单**与**零星订单**两类，新建必选，保存后不可改（换类型只能作废重开）。两类共用状态机、编号规则与权限。
- **常规订单**：条目只能从**有效采购报价条目**里逐行挑选——报价单已审核、订单日期落在报价有效期内、公司/对手/币种均一致。单价与单位随报价锁定（固定价直接带入；数量梯度按行数量自动套档，低于首档起订量视为无报价无法保存），改价必须重报价；仅税率带入后可手改。
- **零星订单**：条目任选物料、手填单价（即无报价约束），但单行数量不得超过供应链设置中的零星上限（默认 100，按行录入数量直接比较，不做单位换算）。
- **订单条目**：订单的商品行，记录物料、单位、数量、原币含税单价、税率；金额列全部由系统按金额链计算（见下）。常规订单行必带报价条目引用，零星订单行必不带。
- **币种与汇率（双币）**：一单一币——币种（原币）与汇率挂订单头，新建默认取单据公司本币，全部条目跟随换算；本币单汇率强制为 1（界面不出汇率字段），外币单汇率必填（大于零，手工填写，无汇率牌价表）。金额链：原币金额=数量×原币单价（2 位）→ 本币金额=原币金额×汇率（2 位）；本币单价=原币单价×汇率（4 位）仅展示参考。落库统一双套（本币单两套同值），报表口径一律读本币列。口径与销售订单一致，见 [术语表](../glossary.md)。
- **订单条目快照**：行保存（新增/修改）时把物料编号/名称/规格/客户料号与单位名称冻结到行上，并把物料当前图纸（`drawing` 槽位文件）复制挂接到行；审核锁行即冻结，主数据后续变更不回溯，订单上下文展示一律以快照为准。图纸为挂接复制（共享不可变文件，非字节复制），行/订单删除时挂接一并清理。机制与销售订单一致，见 [术语表](../glossary.md)。
- **对手**：订单的供应商方，取供应商主数据或内部公司（多态引用）。
- **含税总额**：订单全部条目含税金额的合计（原币/本币各一列），随行自动算出，只读。

## 功能清单

### 列表视图

- 页面分两个 tab（子路由，URL 可直达）：**订单条目**（默认视图，行级明细——每行展示所属订单号、订单日期、对手、订单状态，以及行快照口径的物料编号/名称/规格、单位、收货进度、币种、本币单价、原币单价、本币金额、原币金额、行备注，另有图纸列（缩略图，点开可预览该行全部图纸）；其中**收货进度**为数量/已收/未收三合为一列：进度条＋「已收/数量、未收」按行单位展示（已收/未收存储为物料默认单位口径，展示按行单位换算比折回），该列的筛选与排序均按**未收数量**口径；跨订单混合行恒展示双币全列（本币单两套同值）；默认按订单日期倒序；点订单号或行「查看」开完整订单抽屉，草稿单可经行「编辑」直接改单）与**订单**（整单列表，含币种、原币/本币含税总额列与审核/关闭/作废/删除动作）。
- 两个 tab 共用同一个订单抽屉（新增/查看/编辑三态），「新建订单」入口两个 tab 都有（条目 tab 直接新建整单，与订单 tab 同一表单）；关闭/作废/删除只在订单 tab。
- 菜单：供应链 → 交易 → 采购订单。

### 订单录入

- 头：订单类型（常规/零星，必选，保存后不可改）、订单号、订单日期（默认当天）、公司、对手、币种、汇率（仅外币单）、交易条款（对供应商，自由多行文本）、订单备注（对内）。
- 订单号留空按编号规则自动取号，手填原样保留；订单号全局唯一；两类订单共用同一编号规则（想区分单号可在编号规则里加订单类型字段段）。
- 对手限供应商/内部公司两类；选内部公司时不能是订单所属公司自身。
- 币种：选公司后自动带出公司本币，可手动改为外币；选本币时汇率字段隐藏（系统固定为 1），选外币时汇率必填且大于零。草稿期间改币种/汇率，全部条目的本币金额随之重算；审核后随单冻结。
- 条目子表在订单类型、对手、公司、订单日期选齐之前不可新增；上述四项或币种任一变更加载即清空已录条目（类型保存后锁定，仅新建期可改）。
- 常规订单条目行：从有效采购报价条目选择器逐行挑选（仅列已审核、订单日期在报价有效期内、公司/对手/币种一致的报价条目，可跨多张报价单）；选定后物料、单位、含税单价由报价带入且锁定——数量梯度条目按行数量自动套档，低于首档起订量保存报错；税率从报价带入默认，可手改；另可填行备注。
- 零星订单条目行：物料、单位、数量、原币含税单价、税率、行备注自由录入；单位只能选该物料的默认单位或其单位转换单位，选物料后默认带默认单位；单行数量不得超过供应链设置的零星上限（默认 100）。采购侧不校验客户物料约束，任何物料均可录入。
- 常规订单条目经报价带出的物料在报价侧已无选料约束；行保存与审核仍复核物料与报价引用一致。
- 条目约束：数量必须大于零；原币含税单价可为零（赠品）；税率必填，界面按百分比录入（默认 13%，存储为小数 0.13）；金额列（原币金额、本币金额、本币单价）由系统按金额链计算，不可手改。
- 条目展示随整单币种简化：整单本币时只显示一套「含税单价/含税金额」；外币单展开双套列并在行表单给出本币含税金额只读预览。

### 审核

- 仅草稿单可审核；审核前必须至少有一行条目（其余字段约束在录入期已强制）。
- 审核入口：订单 tab 行「审核」与条目 tab 行「审核整单」都会先弹出整单条目核对确认框（列出该单全部条目），确认后才提交审核；有审核权限的用户在订单抽屉也可直接「保存并审核」（保存成功紧接着审核同一单）。
- 审核时逐行复核：常规订单校验每行报价引用仍有效（已审核、在有效期、公司/对手/币种一致）且单价与报价一致；零星订单校验每行数量不超过当前供应链设置的零星上限。
- 审核后单据锁死：头不可改、行不可增删改、单不可删；无反审核。

### 关闭与作废

- 仅已审核单可关闭或作废，两者都是终态、不可逆。
- 作废：单据不该存在（录入错误、未履行即取消）。
- 关闭：单据生效过但提前终止或执行完毕，人工标记不再履行。

### 删除

- 仅草稿单可删除，条目随单一并删除。

### 供应链设置（采购 Tab）

入口：「供应链 → 设置 → 采购」。维护采侧全局配置（非公司维度；销售项见销售 Tab）：

- **零星订单单行数量上限**（默认 100）：零星订单单个条目的数量不得超过该值，按行录入数量直接比较、不做单位换算；常规订单不受此限。改小不追溯存量草稿，在条目保存与订单审核时校验。
- **入库超收比例**（默认 0%）：入库审核时允许累计已收 ≤ 订购数量 × (1 + 比例)；0% 表示禁止超收。细节见 [采购入库](./采购入库.md)。

## 业务流程

录入（草稿，可反复改）→ 审核（生效锁死）→ 执行完毕或提前终止时点「关闭」；发现单据错误则在审核后直接「作废」，草稿阶段直接删除。变更已审核单没有改单动线，作废/关闭后重新开单。
