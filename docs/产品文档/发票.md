# 发票

## 概述

增值税发票台账：登记公司**开入**（进项）与**开出**（销项）的增值税发票，供财务人员对纸面发票做电子归档与记账。发票是纸面事实的档案——购销双方开票信息、销售清单一律存文本快照，不关联物料、客户、供应商主数据。发票审核后自动生成分录并过账，与手工凭证地位平等；此后只能作废或红冲。支持上传发票原件经 OCR 识别预填表单。本模块不覆盖承兑汇票与进项税抵扣认证；报销发票（员工对手）的付款核销见 [费用报销](./费用报销.md)；部分红冲、批量导入在规划外。

## 核心概念

- **开票方向**：开入（进项票）/ 开出（销项票）；方向决定过账时的借贷安排，用户不直接指定借贷。
- **发票种类**：专用发票、普通发票、电子专票、电子普票、数电专票、数电普票，六种。
- **对手**：发票的对方，支持供应商、客户、内部公司三类，与凭证行、总账分录共用同一套对手口径。
- **发票状态**：草稿 → 已审核 → 已作废 / 已红冲（后两个为终态）。
- **三科目**：往来科目、金额科目、税额科目——审核过账的记账依据，在发票上选定。
- **两层编号**：单据编号是系统内部编号（留空自动取号）；发票代码 + 发票号码是票面真实编号，公司内唯一防重录。
- **对向发票**：内部公司之间互开发票时，同一张纸面发票在双方公司台账中的互链记录（一方开出 ↔ 另一方开入）。
- **关联销售对账单**：开出发票必须关联一张客户已确认的常规销售对账单；审核据此完成「未开票应收 → 应收账款」重分类并把对账单结单（见 [销售对账](./销售对账.md)）。
- **关联采购对账单**：开入发票必须关联一张供应商已确认的常规采购对账单；审核据此完成「未开票应付 → 应付账款」重分类并把对账单结单（见 [采购对账](./采购对账.md)）。

## 功能清单

### 发票登记

手工录入发票，或经 OCR 预填后核对保存（见下节）。

- 新增时先选类型，四选一：销售开出（对手限客户）、内部互开（对手限内部公司，保存后可一键生成对向发票）、采购开入（对手限供应商/内部公司）、费用报销（对手限员工，不关联对账单；选报销类型自动带出金额科目与往来科目，税额科目手选；审核即完成挂账，被报销单引用后不可作废/红冲——详见 [费用报销](./费用报销.md)）。类型决定开票方向与对手类型，表单内不再手改；编辑既有发票时方向沿用记录本身。新增表单按「OCR 优先」动线排布：手填「基本信息」（公司、对手、关联对账单、三科目）→ 上传附件 → 点「上传发票识别」（附件面板与识别按钮就在「票面信息」组正上方）→ 回下方核对「票面信息（OCR 填充，请核对）」与「金额与清单」（识别不出的字段手补）；编辑/查看态按「基本信息 / 票面信息 / 购销双方 / 金额与科目 / 关联」五组展示。
- 按公司登记：公司创建时选定、创建后不可改；只能在自己有权限的公司下登记，发票数据按公司隔离。
- 必填：开票方向（新增时由类型选择决定）、发票种类、对手。对手为内部公司时不能选本公司（不能给自己开票）。
- 编号：单据编号留空自动编号；发票代码 + 发票号码在公司内唯一，重复登记会被拦截。草稿可暂无发票号码（审核前必填）；数电票无发票代码，代码留空即可，判重照常生效。
- 购销双方信息（名称、税号、地址电话、开户行及账号）与销售清单均为文本快照，手填或 OCR 回填，不与主数据联动。
- 销售清单每行记录：名称、规格型号、单位、数量、单价、金额、税率、税额；纯文本记录，不强制行合计等于头部金额（容尾差）。「从明细汇总带出」按钮可按清单汇总回填头部的未税金额、税额、价税合计。
- 价税合计的人民币大写由页面按金额自动显示，不存储。
- 开票人、复核人、收款人是发票纸面栏位（文本），与系统审核人无关。
- 发票可挂附件（如原件扫描件）：新建时即可先上传暂存、保存成功后自动挂接；保存后也可随时补传。附件不受发票写权限锁定的影响。
- 开出发票（销售开出与内部互开）必须关联销售对账单（仅常规单；仅草稿可改），未关联不能保存；关联后审核校验：对账单须为客户已确认、公司/对手一致、发票价税合计 = 对账单本币合计、一票一单。开入发票中的采购开入（对手=供应商/内部公司）必须关联采购对账单（仅常规单；仅草稿可改），未关联不能保存；关联后审核校验口径对称：对账单须为供应商已确认、公司/对手一致、发票价税合计 = 对账单本币合计、一票一单。费用报销发票（对手=员工）不关联任何对账单。
- 草稿可自由修改、可删除；已审核、已作废、已红冲的发票一律锁死，不可改不可删。

### OCR 识别预填

在新建发票时上传发票原件，调阿里云 OCR 识别后自动回填表单，人工核对再保存。

- 入口：新建发票抽屉的「上传发票识别」按钮。
- 文件要求：常见图片格式（png/jpg/jpeg/bmp/gif/tiff/webp）或 PDF，单个不超过 10MB。
- 回填范围：发票代码、号码、开票日期、发票种类、购销双方四项信息、未税金额、税额、价税合计、开票人/复核人/收款人、备注、销售清单行。识别不出的字段不回填，也不会清空已填内容。
- 开票方向不由 OCR 判断，用户自选。
- 识别结果不落库，仅是录入辅助；保存发票后才成为台账记录。
- 留档：发票保存成功后，识别用的原件自动挂为该发票的附件；挂接失败不影响保存，可事后手工补传。
- 前置条件：需先在「系统管理 → 财务设置」配置阿里云 OCR 凭证，未配置时识别按钮不可用（凭证本身的保管归系统管理模块）。
- 安全约束：只能识别本人上传的文件；识别复用发票创建权限，无独立权限点。

### 审核过账

审核使发票生效：生成总账分录并过账，发票同时锁定。

- 审核时填写过账日期（界面默认带出已填的过账日期，未填则取开票日期）。
- 审核前齐全性校验：发票号码、开票日期必填；未税金额、税额、价税合计必填，且未税金额 + 税额 = 价税合计、价税合计大于零、税额不为负；往来科目、金额科目必选；税额大于零时税额科目必选。科目须为本公司启用中的非汇总科目。
- 过账分录由方向与三科目自动决定，借贷配平：
  - 开出（销项）：借 往来科目（价税合计）；贷 金额科目（未税金额）、税额科目（税额）。
  - 开入（进项）：借 金额科目（未税金额）、税额科目（税额）；贷 往来科目（价税合计）。
  - 税额为零时税额科目可空，分录只有两行。
- 仅往来科目那行分录携带对手，与往来挂账实务一致；金额、税额行不带对手。
- 关联销售对账单的开出发票，审核另加**冲回组**两笔：借 对账单头借方科目（发货贷方口径）/ 贷 对账单头贷方科目（未开票应收，该笔带对手），金额 = 价税合计——发货时挂的未开票应收同步清零；同一事务把对账单翻为已结单。
- 关联采购对账单的开入发票，审核同样另加**冲回组**两笔（借贷镜像）：借 对账单头借方科目（未开票应付，该笔带对手）/ 贷 对账单头贷方科目（入库借方口径），金额 = 价税合计——入库时挂的未开票应付同步清零；同一事务把对账单翻为已结单。
- 审核记录审核人与审核时间；审核后发票与生成的分录不可再经发票改动，只能作废或红冲。

### 作废与红冲

已审核发票的两种退出方式，均为终态、互斥（红冲后不可再作废，反之亦然）。

- **作废**：用于实际未开票的场景。原分录不删除，全部打上作废标记，账面与报表（如应收应付）按未作废口径自动剔除。
- **红冲**：用于实际已开票后需冲销的场景。必填红冲过账日期，可登记红字发票号码存档；系统按原分录逐条金额取负（红字、借贷方向不变）生成一组红字分录，原分录组标记「已红冲」。同一发票名下两组分录合计归零，发生额被正确冲减；分录查询中原凭证（已红冲）与红字凭证均可分辨。
- 关联过对账单的发票，作废/红冲都会**自动解除关联**并把对账单退回中间确认态（销售对账单退回「客户已确认」、采购对账单退回「供应商已确认」），可稍后重新关联开票。
- 费用报销发票被报销单挂票行引用（含草稿报销单）期间不可作废、不可红冲；须先在报销单草稿上移除该行，或作废整张报销单（见 [费用报销](./费用报销.md)）。

### 内部公司对向发票

对手为内部公司时，同一张纸面发票需在双方公司各有一条台账（A 开出 ↔ B 开入），系统省去二次录入。

- 触发：新建发票保存成功且对手是内部公司时，提示「是否为对手公司创建对向发票草稿？」；仅新建后提示一次。
- 镜像规则：公司与对手互换、方向取反，票面字段（日期、种类、代码、号码、双方信息、清单、金额、纸面栏位、备注）原样复制；不复制单据编号、三科目、过账日期与附件（对方公司科目体系不同，科目留空待补）。
- 两票创建后互链，可从任一票点开对向发票；对向草稿被删除时，原票的链接自动置空。
- 对向发票是对方公司的普通草稿，由对方经办人补科目后自行审核；当前用户对对方公司无权限时创建失败，需通知对方手工登记。

## 业务流程

- **单票录入动线**：手工登记或「上传原件 → OCR 识别 → 预填表单 → 人工核对」→ 保存为草稿（识别用原件自动留档为附件）→ 备齐号码、日期、金额、三科目后审核，系统在同一事务内生成分录并过账 → 事后若实际未开票则作废，若已开票需冲销则红冲。作废与红冲都只标记、不删数。
- **内部互开动线**：A 公司登记开出发票 → 保存后确认为 B 公司创建开入草稿并互链 → B 公司经办人补本科目体系的三科目 → 双方各自审核过账，形成两条方向相反的分录。
