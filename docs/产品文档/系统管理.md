# 系统管理

## 概述

系统管理模块为整个 ERP 提供账号与安全底座:用户与登录认证、角色-权限分配、公司维度的数据权限、操作审计、单据自动编号、文件与存储接入管理;全新部署的首次初始化由初始化向导完成(建首个超级管理员、首个公司)。服务对象是系统管理员(配置与运维)和全体用户(登录与被授权)。边界:公司、货币、单位等主数据的维护在基础数据模块;部门管理在菜单中预留入口但尚未实现;用户自助修改密码、登录设备管理等能力未提供。

## 核心概念

- **用户**:登录系统的账号,用户名全局唯一且不区分大小写;密码只存哈希,明文从不落库。
- **超级管理员**:用户级旗标,绕过一切功能权限与数据权限检查;仅初始化向导设置,常态界面无入口。
- **角色**:一组权限授权的集合,可停用;停用的角色立即不再为其下用户贡献权限。用户通过「分配角色」获得权限,一人可多角色,权限取并集。**内置角色**随迁移种子(当前仅 admin,持全域通配授权),不可删、不可停用、授权只读,可正常分配给用户。
- **权限点**:按「域.资源:动作」组织的操作能力(如「销售订单-审核」),由代码派生、不入库,因此权限点与系统功能永不漂移。授权支持通配:可整资源授予(该资源全部动作)或整域授予(该域全部动作);另有全域通配(`*`),仅内置角色随迁移种子持有,界面不写通配授权。
- **公司授权**:用户级数据权限,决定能看到哪些公司的业务数据;显式授权、默认不可见(fail-closed),授权挂用户不挂角色。
- **审计日志**:数据变更(创建/更新/删除及审核、作废等业务动作)的字段级留痕,只增不改不删。
- **编号规则**:绑定单据类型的自动取号规则,内容为有序段(固定文本/记录字段/序号)的组合;每种单据至多一条启用规则。
- **打印模板**:绑定资源类型的单据版式 Excel 模板(.xlsx+占位符),同资源可多份、一份默认;打印=填充转 PDF 交浏览器,导出=填充后的 xlsx 下载。
- **存储接入点**:文件字节的存放位置(本地目录或对象存储),全局有一个默认接入点,新上传统一落默认点。

## 功能清单

### 初始化向导

全新部署(库中无用户)首次打开应用即进入初始化向导;已初始化的环境访问 /setup 会被重定向离开。

- 三步:创建首个管理员(用户名+自设口令,直接成为超级管理员并自动登录)→ 创建首个公司(编号/名称/简称/本位币,并按所选科目表模板一键初始化科目;向导已预置约 20 种常用货币供选本币,预置时全部停用,选定本币后仅本币启用,其余须事后在货币管理页手动启用)→ 选择首选语言(仅占位,界面翻译后续版本提供)。
- **起步方式**(仅新建管理员时可选):**空白项目**(只建底座)或**示例数据**(预填演示管理员与公司,完成时为首个公司写入覆盖全业务链的示例数据——客商/物料/员工等主数据、销售与采购全链单据(报价→订单→发货/入库→对账→发票)、库存出入库/调拨/盘点、工序/工艺模板/BOM,以及银行账户与流水、手工凭证、报销单、工资单,便于立刻走通主流程)。示例路径默认预填:管理员 `admin`/`admin123`/管理员;公司 `JT`/台州京泰电气有限公司/台州京泰/人民币;语言简体中文——均可在向导中修改。开发环境亦可 `mix synie.demo` 一键等价初始化(仅 dev/test,已初始化库拒绝覆盖)。
- 是否完成初始化以完成旗标为准(系统设置单行表记录完成时刻);存量环境迁移时自动回填为已完成、永不进向导;落旗后向导入口与其接口永久关闭。
- 向导分步幂等、可断线续做:中途关闭后重开,从第一个未完成步骤继续;已建过管理员则改走「登录后继续」。

### 用户与登录认证

用户管理页维护系统账号。

- 新建用户时系统生成随机初始密码,**明文只在创建响应中显示一次**,需当场转交用户;此后任何人都无法查看密码,只能重置。
- 用户名创建后不可改;可编辑的只有姓名;用户可删除。
- 重置密码由管理员操作,同样生成随机新密码且明文只显示一次;重置复用「编辑用户」的权限,不设独立权限点。用户本人无自助改密入口。
- 登录使用用户名+密码;失败统一提示「用户名或密码错误」,不区分用户不存在与密码错误。
- 登录防暴破:同一用户名+IP 在 5 分钟窗口内失败 10 次即被锁定至窗口结束,期间无论密码对错都拒绝;成功登录即清零计数。
- 登录态有效期 7 天,过期需重新登录。

### 界面外观

全站壳层与业务页支持**外观模式**三选一:**白天**(强制浅色)、**黑夜**(强制深色)、**系统**(跟随操作系统色阶,系统变更时即时跟随)。默认未选过时为系统。

- 偏好仅保存在本机浏览器,不落用户表、不跨设备同步;同源多标签页切换后即时同步。
- 主入口:登录后用户菜单内的三选一控件;登录页与初始化向导右上角提供同款弱化入口。
- 启动幕布、左侧模块图标栏、登录/向导左栏品牌区等**品牌仪式面**保持玄蓝深色,不跟随外观;内容区、二级菜单、顶栏、表单与业务页跟随色阶。
- 列表打印走独立浅色打印窗,不受外观模式影响。

### 角色与权限配置

角色权限页维护角色及其授权,行内「配置权限」打开权限配置宽面板。

- 角色有编码(唯一)、名称与启用状态;编码创建后不可改,启停通过列表行操作切换(不在编辑表单里)。停用角色即收回其全部权限贡献,无需逐用户调整。内置角色(admin)行只读:不可启停、不可删除,权限矩阵仅可查看。
- 宽面板左侧为域导航(带已授权进度徽标),右侧为当前域的权限矩阵:每个资源一行、标准动作(新增/查看/编辑/删除/打印/导入/导出/批量删除/批量编辑/批量打印)平铺为列,资源不具备的动作不可勾;审核/关闭/作废等工作流动作收进行尾「更多」展开区。行头、列头、域导航项均支持三态全选;顶部搜索可跨域过滤资源,命中结果按域分组平铺、可直接勾选。勾选初态含通配覆盖(已被整资源/整域授权覆盖的码显示为已勾)。矩阵的资源清单与中文名随权限目录由后端代码派生并下发。
- 保存为单次事务性同步:前端提交目标权限码集合,后端在事务内对比增删,只同步当前权限目录内的具体码——通配授权行与目录外陈旧码原样保留。界面只逐码授予(全选亦展开为具体码),**不主动写通配授权**——新上线的权限点默认不授予任何角色(fail-closed)。
- 入口门控:无「角色权限-查看」权限看不到「配置权限」入口;编辑需同时具备角色权限的新增与删除两种权限,缺任一则矩阵只读。
- 用户的角色分配与公司授权在用户管理中维护,权限上跟随用户管理(查看跟随「查看用户」,增删跟随「编辑/新建用户」),不设独立权限点。
- 数据权限(公司维度):用户只能看到被授权公司的业务数据,未授权即不可见;读取时无权数据静默不出现在结果中,写入越权则返回校验错误。跨公司管理人员有「全部公司」旗标覆盖(与超级管理员旗标一样无界面入口,仅内部设置)。共享主数据(物料、计量单位等)不分公司,全员可见。

### 操作日志

操作日志页提供全局审计查询;各记录详情页另有按该记录过滤的编辑记录。

- 留痕范围:数据的创建、更新、删除,以及审核、作废、重置密码等业务写动作;登录与读取不留痕。
- 每条日志记录:操作时间、资源、记录名称(记录被删后仍可显示)、操作类型与动作、操作人(冗余存名,人员删除后仍可显示)、字段级变更明细(旧值→新值)。
- 变更明细规则:创建记全部写入值;更新只记实际变化的字段,无实际变更不落日志;删除记删除前的完整快照,作为追溯与手工恢复依据。敏感字段(如密码哈希)一律不落值,标记为已过滤。
- 日志与业务动作同一事务:业务操作失败不会留下脏日志。
- 日志只增不改不删;需「审计日志-查看」权限方可查询。公司数据的日志按授权公司过滤(未授权不可见),全局数据的日志不受公司限制。
- 当前各业务模块的主要单据与主数据(凭证、发票、银行、工资、考勤、客户/供应商、用户角色等)均已接入审计。

### 单据编号规则

编号规则页维护自动取号规则,规则以「绑定单据类型 + 段组装」方式定义,无需书写模板语法。

- 规则内容是有序段列表,段有三种:固定文本、记录字段(本单据字段或其直接关联的字段,如公司编码)、序号。序号段必须恰好一个;位数 0~12,其中 0 表示不补零,1~12 为补零宽度。
- 字段段取值为空时省略该段(不报错),便于物料等「可选前缀」场景用一条规则覆盖多种形态。
- 日期类字段作段时必须选择格式(年/月/日的组合,如「年月」渲染为 202607);非日期字段不能设格式。
- 段可拖拽调整顺序;改序与改规则内容一样只影响此后取号。
- 每种单据至多一条启用规则,启用即生效;规则的绑定单据创建后不可改。
- 「按公司计数」开关(默认开):各公司各自独立计数;关闭则全局共用一组序号。
- 没有独立的重置周期配置:序号按「渲染后的段文本」计数,段里引用的日期一变(如跨月),序号自然从头计。
- 取号发生在单据保存时:编号留空则自动取号,手工填写则原样保留。取号并发安全;单据保存失败会跳号,序号允许有洞、不回收。
- 计数器:规则下可查看各计数范围的当前序号,并可手工调整当前值(调整本身留审计);删除规则连同其计数器一起删除。
- 当前已接入自动编号的单据:会计凭证、增值税发票、承兑交易、费用报销、物料、员工、销售订单/报价/发货/对账、采购订单/报价/入库/对账、手工出入库/调拨/盘点等。
- 初始化向导完成时预置规则:业务单据统一为「主模块字母(子模块字母)-业务日期YYYYMMDD-4 位序号」(如销售订单 `S(O)-20260722-0001`),按公司计数;物料为「分类编号+可选客户编号-不补零序号」(如 `F(P)77-1`);员工为 `H(E)-`+4 位序号。已有规则不覆盖。

### 打印模板

打印模板页维护单据版式模板,供单据的模板打印与模板导出使用;首期接入销售订单与销售发货单。

- 一份模板=名称+资源类型(如销售订单)+上传的 .xlsx 模板文件+备注;模板全局共享不分公司,同一资源可传多份,其中一份可标为默认(打印时预选、可临时换另一份)。
- 模板文件里用占位符取值:单元格写 `${字段名}` 取单据头字段(如 `${order_no}`、`${company_name}`),可与普通文字混排;某一行写 `${items.字段}` 即成为明细模板行,输出时按单据条目数逐行复制展开,`${items._seq}` 给行序号;合计等用 Excel 公式自行书写。日期/数字的显示格式由单元格自身的 Excel 格式控制。可用字段以模板管理页提供的《字段清单》为准。
- 上传即校验:只接受 .xlsx;系统提取模板中全部占位符对照该资源字段清单,有未知名称即拒绝保存并逐个点名,错模板不会活到打印时才暴露。
- 纸张大小、方向、页边距、打印区域等版式设置完全由 Excel 模板的页面设置决定,系统不另设字段、输出时严格按模板页面设置渲染、不自行添加页边距;上传时检测到未做页面设置会给出提示但不拦截。
- 模板可删除(打印是瞬时动作,删模板不影响任何历史单据);更新版式=重新上传替换文件,已输出过的文件不追溯。模板内容暂不支持图片(图纸、公司 Logo 等)。
- 单据侧入口:单据行内「打印」「导出 Excel」与列表勾选后的「批量打印」「批量导出 Excel」,均先弹模板选择(预选默认模板)。打印产出 PDF 由浏览器打开(预览/打印/另存皆用浏览器能力);导出产出填充后的 xlsx 直接下载。批量打印把多份单据版式合进一个 PDF 顺序输出,批量导出为一个 xlsx 内每张单据一个 sheet(以单号命名);单次批量上限 100 条。
- 无可用模板时,单据打印/导出 Excel 入口提示先到本页上传模板;列表工具栏的 CSV 导出是另一回事(导出列表列数据),不受影响。PDF 转换服务异常时打印报错,导出 Excel 不受影响。

### 文件管理

文件管理页查看系统内全部已上传文件;文件上传入口在业务单据的附件面板等各处,本页不提供上传。

- 文件内容不可变:只增只删,不存在编辑;列表展示文件名、存储接入、大小、类型、上传人、上传时间。
- 下载按宿主授权:能看到该文件任一业务挂接(附件)、且同时具备至少一个宿主资源读权限(如员工附件需「员工-查看」)的用户方可下载;没有任何挂接的裸文件仅上传人本人和超级管理员可下载。
- 删除有挂接保护:仍被业务单据引用的文件不可删,需先在业务单据中移除附件。
- 查看抽屉中可看到文件的挂接记录(挂在哪些业务记录、哪个槽位、挂接时间)。

### 存储接入

存储接入页管理文件字节的存放位置,支持本地目录(local)、S3 兼容对象存储(s3)与阿里云 OSS(oss)三种类型。

- 接入名全局唯一,且创建后不可改(已入库文件按接入名引用);存储类型创建后同样不可改。
- 按类型校验必填配置:本地存储必填根目录;S3/OSS 必填服务地址、Bucket 与访问密钥对。
- 全局默认接入点唯一,通过「设为默认」动作切换;新上传的文件统一落默认接入点,已存文件仍在原接入点、不受影响。
- 删除保护:内置接入点(向导完成时种子的本地存储)不可删;当前默认接入点不可删(需先切换默认);仍有文件存放的接入点不可删。
- 访问密钥只写不回读:任何界面与接口都不返回密钥值,列表与详情均不展示;创建时必填,编辑时留空表示不修改。

### 财务设置

财务设置页维护财务模块的全局配置(非公司维度),当前为阿里云 OCR 凭证,供发票/承兑的票面识别调用。

- 凭证为阿里云 RAM 用户的 AccessKey 对,需授权 AliyunOCRFullAccess;仅本页与识别调用使用。
- AccessKey Secret 只写不回读:界面不回显、接口不返回,编辑时留空表示不修改;是否已配置以页面显示的配置状态为准。
- 两项凭证任一留空即视为未配置,业务侧识别入口不可用。

## 业务流程

- **新部署初始化**:迁移(顺带种子 CNY、内置 admin 角色与单行配置)→ 打开应用进入初始化向导(选空白或示例起步 → 建超管 → 建公司 → 选语言)→ 完成时幂等种子内置存储接入、编号规则、物料两级分类与机加工常用计量单位(示例路径另写客商/物料/报价)并落完成旗标,进入日常运行。空库无需再跑 seeds;开发可用 `mix synie.demo`。
- **新用户开通**:管理员新建用户(获得一次性初始密码)→ 在用户抽屉中分配角色、授权公司 → 用户凭用户名+初始密码登录。未授权公司的用户登录后看不到任何公司业务数据。
- **权限调整**:角色权限矩阵保存后,该角色下所有用户的后续请求即时生效(权限每请求实时加载,无需重新登录)。
- **单据取号**:管理员建规则(选单据类型、组装段、启用)→ 用户开单时编号留空 → 保存时按规则自动取号并累计计数 → 需要时管理员在计数器中校正当前序号。
- **文件流转**:业务单据上传附件 → 文件落全局默认存储接入点并建立挂接 → 有权限者从业务侧下载/预览 → 业务侧移除挂接后,文件管理页方可删除文件。切换默认存储接入点只影响此后的新上传,存量文件仍从原接入点读取。
